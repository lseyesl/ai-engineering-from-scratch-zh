# 思维树与 LATS：审慎搜索

> 单条思维链轨迹没有回溯空间。ToT（Yao 等人，2023）将推理转化为树，每个节点上带有自我评估。LATS（Zhou 等人，2024）将 ToT 与 ReAct 和 Reflexion 统一在蒙特卡洛树搜索下。24 点游戏从 4%（CoT）提升到 74%（ToT）；LATS 在 HumanEval 上达到 92.7% 的 pass@1。

**类型：** Build
**语言：** Python（标准库）
**前置知识：** Phase 14 · 01（智能体循环），Phase 14 · 03（Reflexion）
**时间：** ~75 分钟

## 学习目标

- 将推理作为搜索来框架化：节点是"思维"，边是"扩展"，值是"有多有希望"。
- 实现一个标准库 ToT 风格 BFS 树搜索，带有自我评估评分。
- 扩展为一个玩具 LATS MCTS 循环，包含选择/扩展/模拟/反向传播。
- 判断何时搜索值得 token 乘数（24 点游戏、代码生成），以及何时单条轨迹就足够（简单问答）。

## 问题

思维链是线性行走。如果第一步错了，每个后续步骤都在错误的前提下工作。在 24 点游戏上（使用四个数字用加减乘除得到 24），GPT-4 CoT 达到 4% 的准确率。模型过早选择了错误的子表达式，无法恢复。

推理需要的是能够提出多个候选项、评估它们、选择有前途的，并在出现死胡同时回溯的能力。这就是搜索。思维树和 LATS 是两个经典公式。

## 概念

### 思维树（Yao 等人，NeurIPS 2023）

每个节点是一个连贯的中间步骤（"一个思维"）。每个节点可以扩展到 K 个子思维。LLM 使用评分提示自我评估每个节点。搜索探索树——BFS、DFS 或束搜索。

```
                     (根节点: "从 4 6 4 1 得到 24")
                    /               |            \
           ("6 - 4 = 2")    ("4 + 1 = 5")    ("4 * 6 = 24")  <- 分数: 高
              /   \              |                  |
          ...    ...          ...                finish
```

自我评估是承重的部分。论文展示了三种变体：`sure / likely / impossible` 分类、`1..10` 数值评分、以及候选项之间投票。三者都在 24 点游戏上大幅击败 CoT（GPT-4 从 4% 提升到 74%）。

### LATS（Zhou 等人，ICML 2024）

LATS 在 MCTS 下统一了 ToT、ReAct 和 Reflexion。LLM 扮演三个角色：

- **策略**：提出候选下一步行动（ReAct 风格）。
- **值函数**：评分部分轨迹（ToT 风格的自我评估）。
- **自我反思者**：失败时编写自然语言反思（Reflexion 风格），并用它来重新播种未来的 rollout。

环境反馈（观察）混入值函数，使搜索受到真实工具结果的启示，而不仅仅是模型意见。论文时的结果：HumanEval pass@1 92.7%（GPT-4，SOTA），WebShop 平均 75.9（GPT-3.5，接近基于梯度的微调）。

### MCTS，最小化

每次迭代的四个阶段：

1. **选择**——使用 UCT（树的上置信界）从根节点走到叶节点。
2. **扩展**——通过策略生成 K 个子节点。
3. **模拟**——从子节点使用策略 rollout，用值函数（或环境奖励）评分叶节点。
4. **反向传播**——沿路径更新访问计数和值估计。

UCT 公式：`Q(s, a) + c * sqrt(ln N(s) / N(s, a))`。第一项是利用；第二项是探索。根据任务调整 `c`。

### 成本现实

搜索使 token 爆炸。ToT 在 24 点游戏上使用 CoT 的 100-1000 倍 token。LATS 类似。这不是免费的；将搜索保留用于：

- 单条轨迹明显不够的任务（24 点游戏、复杂代码）。
- 墙上时钟时间不如正确性重要的任务。
- 具有廉价、可靠值函数的任务（代码的单元测试、数学的显式目标）。

如果你的任务有单一正确答案和一个嘈杂的评估器，搜索通常会使情况更糟——它会找到一个"高分的"错误答案。

### 2026 年定位

大多数生产智能体不运行 LATS。它们运行带有基于工具验证（CRITIC，第 05 课）的 ReAct。搜索出现在专门的领域中：

- 将测试作为值函数的编码智能体（HumanEval 风格）。
- 探索多个查询路径的深度研究智能体。
- LangGraph 子图中的规划密集型工作流。

AlphaEvolve（第 11 课）是 2025 年的极限：代码上的进化搜索、机器可检查的适应性、前沿收益（56 年来首次 4x4 矩阵乘法改进）。

## 构建

`code/main.py` 实现：

- 在一个程式化的"选择算术操作"任务上的小型 ToT BFS。
- 在相同任务上的玩具 LATS MCTS 循环（选择/扩展/模拟/反向传播），使用 UCT 选择。
- 一个组合符号分数和自我评估分数的值函数。

运行：

```
python3 code/main.py
```

轨迹显示 ToT 使用 BFS 在每个节点扩展三个候选项，与 LATS 通过 MCTS 收敛到最佳 rollout 进行对比。同时打印两者的 token 计数。

## 使用

LangGraph 将 ToT 风格探索作为子图模式提供；LangChain 团队关于 LATS 的博客（2024 年 5 月）是参考教程。LlamaIndex 提供 `TreeOfThoughts` 智能体。对于大多数 2026 年生产智能体，此模式位于 `if task_complexity > threshold: use_search()` 门后——参见第 05 课中的评估器-优化器模式。

## 交付

`outputs/skill-search-policy.md` 根据任务形状、预算和评估器保真度，在线性 ReAct、ToT、LATS 和进化搜索之间进行选择。

## 练习

1. 使用 UCT c=0.1 和 c=2.0 运行玩具 LATS。轨迹中有什么变化？
2. 将值函数替换为更嘈杂的评分器（添加随机抖动）。MCTS 还能找到最佳叶节点吗？它能容忍的最小信噪比是多少？
3. 实现束搜索 ToT（在每层保留 top-k）并与 BFS 比较。在紧张的 token 预算下哪种更好？
4. 阅读 LATS 第 5.1 节。复现 HumanEval 轨迹计数：需要多少次 rollout 才能达到报告的 pass@1？
5. 阅读 LATS 论文关于"何时 LATS 帮助较少"的讨论。编写一段决策规则，将任务形状映射到搜索策略。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|----------------|------------------------|
| 思维树 | "分支 CoT" | Yao 等人——带有自我评估的思维节点树 |
| LATS | "LLM 的 MCTS" | Zhou 等人——在 MCTS 下统一 ToT + ReAct + Reflexion |
| UCT | "上置信界" | 平衡利用（Q）和探索（ln N / n）的选择公式 |
| 值函数 | "这个状态有多好" | 提示 LLM 分数或环境奖励；供给反向传播 |
| 策略 | "行动提议者" | ReAct 风格生成器；发出候选下一步思考/行动 |
| Rollout | "模拟轨迹" | 使用策略从节点走到叶节点，用值评分 |
| 反向传播 | "更新祖先" | 将叶节点的奖励沿路径向上推送，更新访问计数和 Q |
| 搜索成本 | "Token 爆炸" | 在 24 点游戏上达到 CoT 的 100-1000 倍；采用前要有预算 |
