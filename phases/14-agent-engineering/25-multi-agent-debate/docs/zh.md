# 多智能体辩论与协作

> Du 等人（ICML 2024，"Society of Minds"）运行 N 个模型实例独立提出答案，然后通过 R 轮迭代互相批评以达成收敛。提高了事实准确性、规则遵循和推理能力。稀疏拓扑在 token 成本上优于全连接网格。

**类型：** Learn + Build
**语言：** Python（标准库）
**前置知识：** Phase 14 · 12（工作流模式），Phase 14 · 05（自我精炼与 CRITIC）
**时间：** ~60 分钟

## 学习目标

- 解释辩论协议：N 个提议者，R 轮，收敛到共享答案。
- 描述为什么辩论能提高事实准确性、规则遵循和推理能力。
- 解释稀疏拓扑：并非每个辩论者都需要看到其他所有人。
- 使用标准库在脚本化 LLM 上实现全连接网格和稀疏变体的辩论；衡量 token 成本与准确率。

## 问题

自我精炼（第 05 课）是一个模型自我批评——存在群体思维的风险。CRITIC（第 05 课）将批评建立在外部工具之上——并非总是可用。辩论引入了第三种模式：多个实例，交叉批评，通过分歧收敛。

## 概念

### Society of Minds（Du 等人，ICML 2024）

- N 个模型实例独立为同一问题提出答案。
- 在 R 轮中，每个模型阅读其他模型的提议并进行批评。
- 模型根据批评更新自己的答案。
- R 轮后，返回收敛的答案。

原始实验因成本原因使用 N=3、R=2。在困难问题（MMLU、GSM8K、国际象棋走棋合法性、传记生成）上，更多智能体和更多轮次能提高准确率。

跨模型组合优于单一模型辩论：ChatGPT + Bard 一起使用效果优于任意一个单独使用。

### 稀疏拓扑

"通过稀疏通信拓扑改进多智能体辩论"（arXiv:2406.11776，2024-2025）表明全连接网格辩论并非总是最优。稀疏拓扑（星型、环型、中心辐射型）可以在更低的 token 成本下达到相同的准确率。每个辩论者只看到一部分同伴。

影响：

- 全连接网格 N=5，R=3 = 5 × 3 = 15 个提议，每个阅读 4 个同伴 = 60 次批评操作。
- 星型 N=5，R=3（一个中心 + 4 个辐射端）= 15 个提议，辐射端只阅读中心 = 12 次批评操作。

### 辩论何时有帮助

- **事实准确性。** N 个独立提议，交叉检查减少幻觉。
- **规则遵循。** 国际象棋走棋合法性——一个模型遗漏了规则，其他模型发现它。
- **开放式推理。** 多种框架逐步缩小到正确答案。

### 辩论何时有害

- **延迟敏感的体验。** N × R 轮串行是你可能无法承受的延迟。
- **成本敏感的规模。** 每个问题 N × R 个 token。
- **简单事实查询。** 一次查询比五次辩论更便宜。

### 2026 年的实际实例

- **Anthropic 编排者-工作者**（第 12 课）——辩论的一种变体，带有综合步骤。
- **LangGraph 监督者**（第 13 课）——中央路由器 + 专业智能体可以将辩论实现为一个节点。
- **OpenAI Agents SDK**（第 16 课）——智能体之间来回交接以进行迭代批评。
- **多智能体评估**——将辩论与评估者-优化器配对以获取评估信号。

### 这种模式在何处出错

- **收敛崩溃。** 所有智能体收敛到第一个错误答案。通过要求必须存在分歧轮次来缓解。
- **中心故障。** 在星型拓扑中，一个坏的中心会污染所有人。轮换中心或使用多个中心。
- **提示同质化。** 所有智能体使用相同的提示；它们产生相同的答案。使用多样化的提示和/或模型。

## 构建

`code/main.py` 实现了标准库辩论：

- `Debater` 类（脚本化 LLM，带有每个辩论者的意见漂移）。
- `FullMeshDebate` 和 `SparseDebate` 运行器。
- 三个问题：一个事实性问题，一个基于规则的问题，一个推理问题。
- 指标：收敛答案、收敛轮数、总批评操作数。

运行：

```
python3 code/main.py
```

输出：每种协议的准确率和成本；稀疏拓扑在 2/3 的问题上以更低的成本匹配全连接网格。

## 使用

- **Anthropic 编排者-工作者**用于简单的 2-3 工作者辩论。
- **LangGraph**用于有状态的多轮辩论，带有检查点。
- **自定义**用于研究或专门的正确性保证。

## 交付

`outputs/skill-debate.md` 搭建了一个多智能体辩论框架，具有可配置的拓扑、N、R 和收敛规则。

## 练习

1. 实现一个"强制分歧"规则：在第 1 轮，每个辩论者必须提出不同的提议。衡量对收敛速度的影响。
2. 添加置信度加权聚合：辩论者返回（答案，置信度）；聚合器按置信度加权。这有帮助吗？
3. 将一个"智能体"替换为具有不同意见的不同脚本化 LLM。异质性是否提高了准确率？
4. 衡量你的 3 个问题上全连接网格与稀疏拓扑的 token 成本。绘制成本与准确率的关系图。
5. 阅读 Society of Minds 论文。将你的玩具扩展到 N=5、R=3。什么会出问题？什么会变得更好？

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|------|----------------|------------------------|
| Debate | "多智能体批评" | N 个提议者，R 轮交叉批评，收敛 |
| Full mesh | "每个人阅读每个人" | 每个辩论者每轮阅读每个同伴 |
| Sparse topology | "有限的同伴视图" | 辩论者只阅读一部分同伴 |
| Hub-and-spoke | "星型拓扑" | 一个中央辩论者，N-1 个辐射端只阅读中心 |
| Convergence | "达成一致" | 辩论者收敛到共享答案 |
| Society of Minds | "Du 等人的辩论论文" | ICML 2024 多智能体辩论方法 |
