# 顶点课程 38：通过换头进行分类器微调

> Track B 的第一个顶点项目。预训练语言模型是一个以 token 预测头结尾的自注意力块堆叠。当你想要垃圾邮件 vs 非垃圾邮件时，头是错误的但主体基本正确。这节课把头拆掉，将一个二类线性层粘合到池化表示上，并以两种不同方式训练分类器：仅最后一层和完整微调。评估是精确率、召回率和 F1 在留出分割上。你学习每种策略给你带来什么以及它的成本。

**类型:** Build
**语言:** Python (torch, numpy)
**前置要求:** Phase 19 第 30-37 课（NLP LLM track：分词器、嵌入表、注意力块、transformer 主体、预训练循环、检查点、生成、困惑度）
**时间:** ~90 分钟

## 学习目标

- 替换语言模型头为分类头，不重新初始化主体。
- 实现两种训练模式：冻结主体（仅头部）和完整微调，共享一个训练循环。
- 构建一个识别分词器的数据管道，进行填充、掩码填充和池化注意力输出。
- 从原始 logits 计算精确率、召回率、F1 和混淆矩阵。
- 推理参数数量、训练时间和提升空间之间的权衡。

## 问题

你在通用语料库上预训练了一个小型 transformer。输出头将最后一个隐藏状态投影到 1000 个 token 的词汇表。你现在有 800 条标记为垃圾邮件或非垃圾邮件的 SMS 消息，你想要一个二元分类器。三个选项存在。

错误的选项是在 800 个示例上从零训练一个全新的分类器。预训练模型的主体已经编码了有用的结构：词身份、位置、简单共现。扔掉它浪费了构建它的计算。

两个正确的选项是冻结主体换头，以及可训练主体换头。仅头部训练快速，内存几乎免费，并且在此少量数据上很少过拟合。完整微调较慢，可能在少量数据上过拟合，但当下游领域偏离预训练语料库时达到更高的准确率。

这节课构建了两者，以便你可以在同一个夹具上比较它们。

## 概念

模型是一个函数 `f_theta(tokens) -> hidden_states`。头是一个函数 `g_phi(hidden) -> logits`。换头意味着保持 `theta` 并替换 `g_phi`。主体的参数是昂贵的部分。头是一个单一的线性层。

两个可训练参数集重要：

- `theta`（主体）：每个注意力块中数万个权重。
- `phi`（头）：`hidden_dim * num_classes` 权重加上一个偏置。

在仅头部训练中，你计算针对 `phi` 的梯度并针对 `theta` 将其置零。PyTorch 通过设置主体参数的 `requires_grad=False` 让你做到这一点。然后优化器只看到头，主体保持冻结。

在完整微调中，你让梯度通过整个堆叠流回。主体的权重漂移以适应分类目标。风险是在小数据上灾难性遗忘：主体的预训练被过拟合噪声冲走。

## 池化问题

分类器需要每个序列一个向量，而不是每个 token 一个向量。三个常见选择：

- **均值池化**：对隐藏状态按序列求平均，由注意力掩码加权。
- **CLS 池化**：预先添加一个特殊 token 并只使用其输出。这就是 BERT 的做法。
- **最后 token 池化**：使用最后一个非填充 token。这就是 GPT 类分类器的做法。

这节课使用带有显式注意力掩码加权的均值池化。它是最简单的，在序列长度上给出稳定的信号，并且不需要预训练 CLS token。

## 数据

八百条 SMS 消息，平衡 400 条垃圾邮件和 400 条非垃圾邮件，在 `code/main.py` 中确定性生成。生成器使用固定种子，选择模板并替换槽填充符，发出 5 到 25 个 token 长的消息。真实数据集有这个夹具没有的噪声。夹具的重点是可重现性。

数据分割 80/20：640 训练，160 测试。分割是分层的，因此测试集保持 50/50 平衡。具有已知平衡的留出集让精确率和召回率可以作为诚实的数字读取。

## 指标

二元分类，类别 1 为正类（垃圾邮件）。计数为：

- `TP`：预测为垃圾邮件，实际是垃圾邮件。
- `FP`：预测为垃圾邮件，实际是非垃圾邮件。
- `FN`：预测为非垃圾邮件，实际是垃圾邮件。
- `TN`：预测为非垃圾邮件，实际是非垃圾邮件。

三个主要指标：

- `precision = TP / (TP + FP)`。在标记为垃圾邮件的消息中，实际比例是多少？
- `recall = TP / (TP + FN)`。在实际垃圾邮件中，模型标记的比例是多少？
- `F1 = 2 * P * R / (P + R)`。两者的调和平均。

混淆矩阵将四个计数打印为 2x2 网格。演示将这对两种训练体制写入标准输出。

## 架构

主体是一个故意微小的 transformer：词汇表 260、隐藏 64、4 头、2 块、最大序列 32。它足够小，两种体制在 CPU 上 90 秒内都能收敛。这节课中不是预训练的；相反，`pretrain_quick` 辅助函数在相同夹具的文本上做五个 epoch 的 LM 训练，给主体一个非平凡的起点。这保持课程自包含。

## 你将构建什么

实现是一个 `main.py` 加上一个测试模块。

1. `ByteTokenizer`：映射字节到 ID，保留一个填充 ID。
2. `Block`：带多头注意力和前馈层的 transformer 块。Pre-norm。
3. `LMBody`：token + 位置嵌入加上块堆叠。返回隐藏状态。
4. `MeanPool`：掩码加权的序列轴平均。
5. `Classifier`：主体、池化、线性头。主体在体制间是相同的实例。
6. `freeze_body` 和 `unfreeze_body`：切换主体参数的 `requires_grad`。
7. `train_classifier`：一个共享循环。接受模型和针对任何可训练参数组配置的优化器。
8. `evaluate`：运行测试集并返回 `Metrics(precision, recall, f1, confusion)`。
9. `run_demo`：短暂预训练主体，然后训练和评估仅头部，然后完整微调，打印两个报告，并以零退出。

## 为什么比较重要

仅头部体制通常训练更快，更优雅地欠拟合。在这个夹具上，你通常看到经过二十个 epoch 的仅头部训练，精确率接近 0.9，召回率接近 0.85。完整微调大约花费三倍时间，根据随机种子落在任一方向几个点内。

这节课不选择赢家。它教你读取数字和成本。在 800 个示例和一个小主体上，仅头部是正确的选择。在 80,000 个示例和一个更大的主体上，完整微调开始产生回报。你从这节课带走的契约是 API：相同的 `train_classifier` 函数处理两者，切换是一个调用。

## 扩展目标

- 添加第三种体制，仅解冻最后一个块。这有时称为部分微调。它比完整 FT 成本低，比仅头部学得多。
- 添加学习率调度器。头上的余弦调度加上主体上的较小常数率是常见的生产设置。
- 将均值池化替换为学习注意力池化：一个带一个学习查询的小注意力层。这在较长序列上通常优于均值池化。
