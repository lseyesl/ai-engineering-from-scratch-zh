# 分块策略对比

> 分块决定了检索器能返回什么。切错边界，任何嵌入模型、重排器或 LLM 都无法在下游修复损失。

**类型：** 构建
**语言：** Python
**前置课程：** 第 11 阶段课程 04（嵌入）、06（RAG）、07（高级 RAG）；第 19 阶段 Track B 基础（课程 20-29）
**时长：** ~90 分钟

## 学习目标
- 从零实现五种分块策略：固定窗口、句子、递归分割、语义聚类和结构化 Markdown 标题。
- 在带有金标准答案标注的固定语料库上测量 recall@k，并解释为什么一种策略在散文类文档上胜出，而另一种策略在技术文档上胜出。
- 阅读分块长度分布，识别每种策略注入的失败模式：孤立句子、符号中间截断、仅标题块、语义漂移。
- 通过检查三个属性为新语料库选择默认策略，无需运行基准测试：文档类型、平均段落长度，以及格式是否携带显式结构。

## 问题所在

每条 RAG 流水线都从将源文档切分成片段开始——片段要足够小以适配嵌入模型，又要足够大以承载一个自足的语义单元。切分位置的选择不是超参数，它是检索器能返回内容的上限。

一个查询"预算中止阈值是什么"只有在包含中止阈值的块可达时才能成功。如果固定窗口分割器将阈值从上下文中切离，嵌入向量会漂移到另一个聚类，BM25 分数下降，重排器看到噪声，LLM 生成的答案就是错的。2024 年的论文"LongRAG: Enhancing Retrieval-Augmented Generation with Long-context LLMs"测量到仅因分块选择就造成了 35 个百分点的检索召回率绝对差距。2025 年关于上下文块标题的后续工作缩小了差距，但并未消除。

本课程并排构建五种策略，在带有金标准答案标注的固定语料库上运行，让你亲自阅读召回率数据。

### 固定窗口

暴力基线。每 N 个字符切一次。可选重叠，这样在位置 N 被切断的句子会完整出现在从位置 N - overlap 开始的块中。快速、确定性、边界效果极差。用作对照组，不要用作默认值。

### 句子

在句子边界处用正则表达式或简单状态机分割。将一个或多个句子打包到目标字符预算的块中。不会在单词中间切断。但仍会在段落和章节中间切断。许多早期 RAG 流水线的默认选择，对于没有其他结构的散文类文档是合理的选择。

### 递归分割

2023 年代库流行的层级策略。先尝试在最强分隔符（双换行、段落）处分割，回退到下一个（单换行），再到句子，再到字符。当块适配预算时递归终止。对结构不一致的文档表现强劲，因为它按区域自适应。

### 语义聚类

嵌入每个句子。聚类共享主题质心的连续句子。当与质心的运行相似度低于阈值时切割。边界反映语义，而非字符。构建较慢且依赖嵌入模型，但对段落内切换主题的文档具有韧性。

### 结构化 Markdown 标题

对于携带显式结构的文档（Markdown、reStructuredText、RFC 风格的编号章节），在标题边界处切割。每个块成为标题加上其下方直到同级或更高级标题的所有内容。每个主题的块最小，但仅在语料库格式良好时可用。

### recall@k 如何衡量边界选择

金标准标注的查询携带源文档中答案区间的精确字符偏移量。分块后，你问：检索器返回的 top-k 块中是否有任何一个与金标准区间重叠？如果是，该查询的 recall@k 为 1。如果不是，为 0。对查询集取平均。对每种策略运行相同评估，差距显示哪种边界策略在你拥有的语料库上存活。

## 构建它

`code/main.py` 实现了：

- `fixed_window(text, size, overlap)` - 基线。
- `sentence_chunks(text, target)` - 简单句子打包器。
- `recursive_split(text, separators, target)` - 层级递归。
- `semantic_chunks(text, similarity_threshold)` - 基于确定性模拟嵌入的质心聚类。
- `structural_markdown(text)` - 标题感知分割器。
- `mock_embed(text, dim)` - 基于哈希的嵌入，使循环可离线运行。
- `DenseIndex` - 与第 19 阶段 Track B 混合检索课程中使用的相同结构。
- `eval_recall(strategy, corpus, queries, k)` - 对比循环。
- 一个 `main()` 函数，在固定语料库上运行每种策略并打印 recall@k 表格。

运行：

```bash
python3 code/main.py
```

输出是一个小表格，每种策略一行，每个 k 一列。句子策略在结构化固定数据上失败。结构化 Markdown 在 Markdown 固定数据上胜出。递归分割在混合固定数据上表现不错，因为递归会自适应。语义聚类在没有有用结构线索的散文固定数据上胜出。

## 表格不会隐藏的失败模式

**孤立句子。** 句子打包产生的块缺少主题句。嵌入向量随后指向错误的聚类。

**符号中间截断。** 固定窗口在代码或 YAML 内部会将标识符切成两半。两半嵌入为噪声。

**仅标题块。** 结构化 Markdown 生成只包含 `## Title` 的块。过滤掉这些或将下一个块的第一段附加过来。

**语义漂移。** 语义聚类在语料库整体围绕同一主题时会切割不足。一个 5000 字符的块将许多具体答案打包成一个模糊的嵌入。将语义聚类与硬字符上限结合使用。

**嵌入过期。** 语义聚类使用嵌入模型。如果更换模型，块也会改变。将分块模型与检索模型分开固定，或一起重建索引。

## 不运行基准测试选择默认策略

三个属性决定新语料库的默认分块器。

| 属性 | 值 | 默认策略 |
|----------|-------|---------|
| 文档类型 | 无结构的散文 | 递归分割，目标 800 |
| 文档类型 | Markdown / RFC / API 文档 | 结构化 Markdown |
| 文档类型 | 代码 | AST 感知（超出范围；见第 19 阶段课程 02） |
| 段落长度 | 长，单一主题 | 句子，目标 500 |
| 段落长度 | 短，混合主题 | 语义，阈值 0.6 |

拿不准时，选递归分割。它是最强的单一策略基线。

## 使用它

生产模式：

- 在发布新流水线之前运行评估；不要信任库的默认策略。
- 每当更改嵌入模型或语料库组合时重新运行评估；胜者取决于语料库。
- 在每个块的元数据中持久化策略名称，以便后续归因回归。

## 发布它

Track F 的端到端 RAG 系统（课程 69）使用此处选择的分块器作为其第一阶段。课程 68 的评估工具读取与 `eval_recall` 返回的相同结构的 recall@k。选择在语料库上胜出的策略并传递下去。

## 练习

1. 添加第六种策略：使用 `tiktoken` 的 token 窗口而非字符计数。在同一固定数据上与固定窗口对比。
2. 在散文固定数据中注入 30% 的代码块。重新运行表格。解释为什么除结构化 Markdown 外每种策略的召回率都下降。
3. 将确定性嵌入替换为项目实际提供商的嵌入。测量语义聚类召回率差异。报告策略间差距是扩大还是缩小。
4. 为每个块添加 `summary` 字段：一句质心描述。将摘要附加到块体后重新运行评估。测量召回率提升。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------------|------------------------|
| Recall@k | "我们拿到正确的块了吗？" | top-k 块中任何一个与金标准答案区间重叠的查询占比 |
| 块重叠 | "滑动窗口" | 在下一个块中重新包含上一个块的最后 N 个字符 |
| 结构化分割器 | "标题感知块" | 在 H1/H2/H3 边界处切割；标题文本是块的一部分 |
| 语义分块器 | "主题感知块" | 嵌入句子，按质心相似度聚类，在漂移处切割 |
| 质心漂移 | "主题转换" | 运行均值与下一个句子之间的余弦相似度降至阈值以下 |
