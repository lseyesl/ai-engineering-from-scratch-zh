# 流水线并行与气泡分析

> 张量并行将矩阵乘法拆分到各 rank。流水线并行将模型拆分到各 rank，每个 rank 一个阶段。微批次流过流水线。开头和结尾的空闲时间是气泡；最小化气泡就是全部技艺。

**类型：** 构建
**语言：** Python
**前置课程：** 第19阶段 C 轨道 第42-49课
**时长：** ~90 分钟

## 学习目标

- 将顺序模型拆分为 N 个阶段，模拟跨 N 个 rank 的前向流水线。
- 使用 GPipe 调度（仅前向填充，然后反向）将 M 个微批次调度通过流水线，计算气泡比例。
- 将气泡与 Megatron-LM 和 PipeDream 使用的交错 1F1B 调度进行比较。
- 论证阶段分配：每阶段等计算量比每阶段等参数量更重要。

## 问题所在

一个 700 亿参数的 fp16 模型仅参数就需要 140 GB。没有消费级 GPU 能容纳。ZeRO-3 跨 rank 分片参数，但每个 rank 仍需为每次前向步 allgather 完整层，每层付出 log(N) 跳。流水线并行走不同路线：将模型切成 N 个阶段，每个阶段放在一个 rank 上。第 1 层的前向在 rank 0 上完成，将激活张量交给 rank 1；rank 1 运行第 2 层并交给 rank 2；以此类推。反向以相反方向流动。内存线性下降，因为每个 rank 只持有一个阶段；计算是顺序的，这就是气泡问题。

气泡是流水线开头的空闲时间（等待第一个微批次到达最后阶段）和结尾的空闲时间（等待最后一个微批次反向流回）。M 个微批次、N 个阶段时，每阶段气泡比例为 (N-1)/(M+N-1)。M=8、N=4 时为 27%。M=64、N=4 时为 4.5%。当你有很多微批次每步时气泡缩小，这意味着每微批次的批次大小要小，这是驱动微批次设计的约束。

## 核心概念

```mermaid
flowchart LR
  R0[rank 0: stage 0 / layer 0] --> R1[rank 1: stage 1 / layer 1]
  R1 --> R2[rank 2: stage 2 / layer 2]
  R2 --> R3[rank 3: stage 3 / loss]
  R3 -.backward.-> R2
  R2 -.backward.-> R1
  R1 -.backward.-> R0
```

### GPipe 调度

先用所有 M 个微批次前向填充流水线，然后才开始任何反向；再以逆序排空反向。每个微批次的激活必须保持到其反向时，所以内存随 M 线性增长。前向需要 M+N-1 个周期，反向需要另外 M+N-1 个周期。每阶段有用工作为 2M 个周期；每阶段气泡为 2(N-1) 个周期。当每个前向和反向各占一个时间单位时，气泡比例为 (N-1)/(M+N-1)。选择 M 远大于 N 可隐藏气泡。

### 1F1B 调度

交错：微批次的前向一到达最后阶段，就启动其反向并让其流回。调度在每个阶段交替一个前向和一个反向。气泡仍为 N-1，但激活内存受流水线深度限制，而非微批次数。生产流水线使用 1F1B（Megatron、PipeDream）。本课先实现 GPipe 因为它更简单，1F1B 作为练习。

### 为什么每阶段等计算量很重要

如果阶段 0 耗时 50 ms，阶段 1 耗时 100 ms，每个周期都受阶段 1 门控。其他阶段每个周期空闲 50 ms 等待阶段 1 释放。等参数数是错误的轴：transformer 的计算由每层的注意力加 MLP 主导，嵌入层参数多但计算少。阶段分配应使每阶段 FLOPs 相等，而非每阶段权重相等。

### 微批次与批次

流水线运行 M 个大小为 B 的微批次。有效批次大小为 M*B。流水线步结束时的梯度是 M*B 个样本上的梯度。气泡比例取决于 M；优化器看到 M*B。调优 M 意味着在气泡（高 M 时更低）和每微批次内存（GPipe 高 M 时激活内存更高）之间权衡。

## 生产中的模式

三种模式使流水线并行足够健壮以投入生产。

**激活检查点与流水线配对。** GPipe 上有 M 个微批次在途时，激活内存是 M 倍一个微批次。激活检查点在反向时重新计算前向，用计算换内存；组合使流水线对长序列可行。

**阶段平衡是测量的，不是假设的。** 生产团队运行性能分析，测量目标硬件上实际的每层计算（FLOPs 和挂钟时间），然后按测量值分区。Megatron-LM 的 `--num-layers-per-stage` 标志接受列表以允许阶段每层成本不同时的不均匀层数。

**发送-接收调度必须避免死锁。** 每个阶段先发送再接收的流水线在线路上死锁。标准修复是交错：偶数 rank 阶段先发送再接收，奇数 rank 阶段先接收再发送。本课显式调度 rank 使模式可见。

## 使用它

生产模式：

- **Megatron-LM。** 大规模流水线并行的参考。使用 1F1B，支持张量+流水线+数据并行组合。
- **DeepSpeed Pipeline。** 与 ZeRO 集成；ZeRO-1 + 流水线是最大开源模型的常见组合。
- **PyTorch Pipe。** PyTorch 原生流水线包装器，基于 `torch.distributed.pipeline.sync.Pipe`。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|----------------|------------------------|
| 流水线 | "沿深度的模型并行" | 每个 rank 一个阶段，激活在阶段间流动 |
| 气泡 | "流水线空闲时间" | 开头+结尾的 (N-1) 步中某些阶段没有工作 |
| 微批次 | "批次的切片" | 一个前向/反向单元；气泡随 M 增大而缩小 |
| GPipe | "先填再排" | 所有 M 个前向先于任何反向；高激活内存 |
| 1F1B | "交错调度" | 每阶段一个前向一个反向；有界激活内存 |
