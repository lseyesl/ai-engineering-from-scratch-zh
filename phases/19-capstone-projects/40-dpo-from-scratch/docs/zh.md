# 顶点课程 40：从零实现直接偏好优化

> 奖励模型和 PPO 是经典的 RLHF 堆栈。DPO 将该堆栈折叠为单个监督损失，直接针对偏好对拟合策略。这节课从奖励差异恒等式推导 DPO 损失，提供工作参考模型加策略模型，计算每个 token 的对数概率，并在选择性和拒绝性补全的偏好夹具上训练一个小型 transformer。测试固定了损失数学和梯度方向，让你知道实现与论文匹配。

**类型:** Build
**语言:** Python (torch, numpy)
**前置要求:** Phase 19 第 30-37 课（NLP LLM track：分词器、嵌入表、注意力块、transformer 主体、预训练循环、检查点、生成、困惑度）
**时间:** ~90 分钟

## 学习目标

- 将 DPO 损失推导为对缩放的对数比率差异的 sigmoid，并将其连接到隐式奖励。
- 构建一个参考模型 + 策略模型对，带冻结的参考和可训练的策略。
- 在两个模型下计算序列级对数概率，掩码提示 token。
- 在 `(prompt, chosen, rejected)` 三元组上训练策略，观察选择性的对数概率相对于拒绝性的上升。
- 用测试固定损失数学、梯度符号和参考不变性。

## 问题

你有一个 SFT 模型。它遵循指令，但其输出不均衡；一些补全清晰，一些冗长或错误。你还有一个小型的偏好对数据集：对于相同的提示，一个人标记一个补全为选择性，另一个为拒绝性。

经典的 RLHF 答案是一个两阶段管道。在偏好上训练奖励模型。用 PPO 针对奖励优化策略。这有效但昂贵：PPO 期间两个模型在内存中，KL 控制保持策略接近参考，当奖励模型脆弱时奖励黑客攻击。

DPO 用单个监督损失替换两个阶段。奖励模型从未显式存在。策略直接在偏好对上训练，带有朝向 SFT 参考的显式 KL 惩罚。在 Bradley-Terry 偏好模型下相同的最优解，代码少得多。

## 概念

从 Bradley-Terry 模型开始。给定一个提示 `x` 和两个补全 `y_w`（选择性）和 `y_l`（拒绝性），人类偏好 `y_w` 的概率是

```text
P(y_w > y_l | x) = sigmoid( r(x, y_w) - r(x, y_l) )
```

其中 `r` 是一些潜在奖励函数。RLHF 首先从偏好拟合 `r`，然后训练策略 `pi` 以最大化 `r` 并带有一个 KL 锚：

```text
max_pi   E_{x, y~pi} [ r(x, y) ] - beta * KL(pi || pi_ref)
```

DPO 推导观察到该目标下的最优策略 `pi*` 以 `r` 有一个闭式：

```text
pi*(y | x) = (1/Z(x)) * pi_ref(y | x) * exp( r(x, y) / beta )
```

重新排列为 `r`：

```text
r(x, y) = beta * ( log pi*(y | x) - log pi_ref(y | x) ) + beta * log Z(x)
```

`log Z(x)` 项对 `y_w` 和 `y_l` 是相同的（它依赖于 `x`，而不是 `y`），因此当计算偏好差异时它抵消：

```text
r(x, y_w) - r(x, y_l) = beta * ( log pi_theta(y_w|x) - log pi_ref(y_w|x)
                                - log pi_theta(y_l|x) + log pi_ref(y_l|x) )
```

代入 Bradley-Terry sigmoid 并对偏好对取负对数似然：

```text
L_DPO(theta) = - E_{(x, y_w, y_l)} [
  log sigmoid( beta * ( log pi_theta(y_w|x) - log pi_ref(y_w|x)
                       - log pi_theta(y_l|x) + log pi_ref(y_l|x) ) )
]
```

这就是损失。它是每个示例一个标量上的 sigmoid，从四个对数概率计算。没有单独的奖励模型。没有 PPO。损失中没有 KL 项；KL 约束内置于闭式推导中。

## 梯度的符号

在任何训练运行前一个有用的健全性检查。取关于 `log pi_theta(y_w | x)` 的梯度：

```text
d L_DPO / d log pi_theta(y_w | x) = - beta * (1 - sigmoid(z))
```

其中 `z` 是 sigmoid 的参数。这对所有 `z` 是负的，这意味着：增加策略对选择性补全的对数概率降低损失。对称地，关于 `log pi_theta(y_l | x)` 的梯度是正的：增加拒绝性补全的对数概率增加损失。训练推动选择性上升，拒绝性下降。参考是冻结的；它不动。

## 数据

课程附带十二个偏好三元组。每个是 `(prompt, chosen, rejected)`。选择性补全简短而精确。拒绝性补全冗长、偏离主题或错误。对覆盖与第 39 课相同的任务家族（首都、算术、列表），因此从 SFT 基础开始的策略有一个合理的起点。

夹具故意很小。DPO 在生产中工作在数万个对上；在这里，重点是损失数学和循环在一个小型数据集上端到端运行，并且选择与拒绝的对数概率差距可见地增长。

## 参考不变性

DPO 实现必须小心处理参考模型。参考是冻结在原位的 SFT 模型。三个属性必须保持：

- 参考参数从不接收梯度。
- 参考对数概率在 epoch 之间从不改变。
- 策略从与参考相同的权重开始。（最优 `theta` 是参考加上学习到的更新；将策略初始化为参考的副本是定义良好的起点。）

实现通过以下方式强制执行这些：

- 在前向传播期间将参考包装在 `torch.no_grad()` 中。
- 在每个参考参数上设置 `requires_grad=False`。
- 在参考构建后通过 `policy.load_state_dict(reference.state_dict())` 构造策略。

## 架构

模型是与第 39 课相同的 TinyGPT（解码器专用、因果、字节分词器）。参考和策略共享架构；策略的权重在训练下从参考漂移，而参考保持固定。

## 你将构建什么

实现是一个 `main.py` 加测试。

1. `InstructionTokenizer`：带 `INST` 和 `RESP` 特殊标记的字节分词器。与第 39 课相同的形状。
2. `TinyGPT`：解码器专用 transformer。与第 39 课相同的形状，使课程即使你跳过了第 39 课也自包含。
3. `make_preferences`：返回十二个 `(prompt, chosen, rejected)` 三元组。
4. `sequence_log_prob`：给定模型、提示前缀和一个补全，返回补全上的下一个 token 对数概率之和（没有提示位置贡献）。
5. `dpo_loss`：取四个对数概率和 `beta`，返回每个示例的损失张量和用于日志的隐式奖励差。
6. `train_dpo`：每 epoch 循环，计算策略和参考下的选择性和拒绝性对数概率，应用损失，并步进 Adam。
7. `evaluate_margins`：返回策略下任何时刻的平均选择性-拒绝性对数概率边距。
8. `run_demo`：从小的预热预训练构建参考和策略，复制权重，训练三十步，打印每步损失和边距，成功时以零退出。

## 为什么 DPO 有效

DPO 在 Bradley-Terry 偏好模型下数学上等价于 RLHF，直到奖励的参数化。隐式奖励 `r(x, y) = beta * (log pi(y|x) - log pi_ref(y|x))` 从偏好中可识别，最多一个 `x` 的函数，在差中抵消。闭式策略让你跳过显式奖励模型。KL 约束被结构性地强制执行：`pi` 与 `pi_ref` 的任何偏差使对数比率更大，sigmoid 饱和，当策略移动太远时抑制梯度。参考是你的安全网。

## 扩展目标

- 添加长度归一化到对数概率和：除以补全长度。长度偏差是已知的 DPO 故障模式，模型优先选择较短的补全，因为它们的对数概率在绝对项中更大。
- 添加损失的 IPO 变体：用 `(z - 1)^2` 替换 sigmoid + log。在夹具上比较收敛。
- 添加标签平滑参数，在硬选择性-拒绝性标签和均匀 0.5 之间插值。
- 将参考替换为更小更便宜的模型（知识蒸馏风格）。
