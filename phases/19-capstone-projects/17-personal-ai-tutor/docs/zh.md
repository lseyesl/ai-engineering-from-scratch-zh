# 顶点项目 17——个人 AI 导师（自适应、多模态、带记忆）

> Khanmigo（可汗学院）、Duolingo Max、Google LearnLM / Gemini for Education、Quizlet Q-Chat 和 Synthesis Tutor 在 2026 年都大规模推出了自适应多模态辅导。共同的形态是苏格拉底式教学策略（从不直接给出答案）、每次互动后更新的学习者模型（贝叶斯知识追踪风格）、语音 + 文本 + 拍照数学输入、课程图谱检索、间隔重复调度，以及针对适龄内容的严格安全过滤器。这个顶点项目是交付一个学科特定的导师（K-12 代数或 Python 入门），运行一个为期两周的、有 10 名学习者的效果研究，并通过内容安全审计。

**类型:** Capstone
**语言:** Python（后端、学习者模型）、TypeScript（Web 应用）、SQL（通过 Postgres + Neo4j 的课程图谱）
**前置要求:** Phase 5（NLP）、Phase 6（语音）、Phase 11（LLM 工程）、Phase 12（多模态）、Phase 14（智能体）、Phase 17（基础设施）、Phase 18（伦理安全）
**涉及阶段:** P5 · P6 · P11 · P12 · P14 · P17 · P18
**时间:** 30 小时

## 问题

自适应辅导曾经是教育科技的研究领域。到 2026 年它已经成为消费产品。Khanmigo 部署在美国大部分学区。Duolingo Max 达到了数千万月活用户。Google 的 LearnLM / Gemini for Education 为 Google Classroom 中的辅导提供支持。Quizlet Q-Chat 与闪卡并存。Synthesis Tutor 凭借面向好奇孩子的导师走红。共同要素：多模态输入（打字、说话、拍摄方程）、苏格拉底式教学法（先提问，后解释）、每次互动后更新的学习者模型，以及严格的适龄安全措施。

你将为一个特定群体构建这样一个系统。衡量标准是一项实际的效果研究：两周内 10 名学习者的前测和后测分数。语音循环必须感觉自然（复用顶点项目 03 的子栈）。记忆必须尊重隐私。安全过滤器必须通过针对 K-12 的 COPPA 感知红队测试。

## 概念

四个组件。**导师策略**是一个苏格拉底式循环：当学习者询问答案时，策略会反问一个引导性问题；当他们答对时，它会进入下一个概念；当他们卡住时，它会提供一个支架式提示。**学习者模型**是贝叶斯知识追踪（或一个简单变体），每次互动后更新每个课程节点的掌握概率。**课程图谱**是一个 Neo4j 图，包含概念及其先决条件边；策略遍历该图以选择下一个概念。**记忆**是一个情景 + 语义存储（agentmemory 风格），保存过去的互动、错误和偏好。

用户体验是多模态的。文本输入用于打字答案。通过 LiveKit + Whisper 的语音输入（复用顶点项目 03）。通过 dots.ocr 或 PaliGemma 2 对数学题进行拍照输入。通过 Cartesia Sonic-2 的语音输出。安全性使用 Llama Guard 4 加上一个适龄过滤器（阻止成人内容、暴力、自残）和一个 COPPA 感知的记忆保留策略。

效果研究是交付物。10 名学习者，前测和后测，两周。报告学习增益差值和置信区间。与非自适应基线（相同内容，线性呈现，没有导师策略）进行比较。

## 技术栈

- 学科选择：K-12 代数或 Python 入门（选择一个深入）
- 导师策略：基于 Claude Sonnet 4.7 的 LangGraph（带提示缓存）
- 学习者模型：贝叶斯知识追踪（经典）或用于间隔重复的 FSRS
- 课程图谱：Neo4j 的概念图 + 先决条件边 + OER 内容
- 记忆：agentmemory 风格的持久化向量 + 情景 + 语义存储
- 语音：LiveKit Agents 1.0 + Cartesia Sonic-2（复用顶点项目 03 子栈）
- 拍照数学：dots.ocr 或 PaliGemma 2 用于方程识别
- 安全性：Llama Guard 4 + 自定义适龄过滤器
- 评估：Bloom 级别问题生成、前/后测工具、效果研究工具

## 构建它

1. **课程图谱。** 构建一个包含 50-150 个概念节点的 Neo4j 图（例如，K-12 代数从"数轴"到"二次公式"），包含先决条件边。为每个节点附加 OER 内容（Open Textbook、OpenStax）。

2. **学习者模型。** 初始化贝叶斯知识追踪，设定先验：猜测、失误、学习率。每次互动后更新每个概念的掌握度。为每个学习者持久化。

3. **导师策略。** LangGraph 包含节点：`read_signal`（学习者的答案是否正确/部分正确/卡住？）、`select_concept`（遍历课程图谱，选择优先级最高的概念）、`scaffold`（苏格拉底式提示）、`update_mastery`。

4. **记忆。** 每次互动写入情景存储。错误和偏好提升到语义记忆。COPPA 感知保留策略：1 年后自动删除，家长可访问。

5. **语音路径。** LiveKit Agents worker 连接到导师策略。ASR 通过 Whisper-v3-turbo。TTS 通过 Cartesia Sonic-2。支持打断（复用顶点项目 03 的机制）。

6. **拍照数学路径。** 上传或拍摄图像；运行 dots.ocr 或 PaliGemma 2 识别方程；作为结构化输入提供给导师。

7. **安全性。** 每个模型输出通过 Llama Guard 4 + 适龄过滤器（阻止自残、成人内容、暴力）。记忆访问按学习者 ID 范围限定；家长访问界面用于删除。

8. **效果研究。** 10 名学习者，前测（标准化 30 题基线），两周导师互动（每周 3 次），后测。与同等规模的非自适应基线对照组进行比较。

9. **每周进度报告。** 为每个学习者自动生成 PDF 摘要，包含探索主题、掌握度轨迹和推荐的下一步。

## 使用它

```
学习者："我不明白为什么 3x + 6 = 12 意味着 x = 2"
[signal]   卡住
[concept]  '隔离变量'（先决条件：加减等式性质）
[scaffold] "你从两边减去什么数来开始呢？"
学习者："6"
[signal]   正确
[mastery]  加减等式性质：0.62 -> 0.77
[concept]  继续 '隔离变量'
[scaffold] "很好。那么 3x / 3 等于什么？"
```

## 练习

1. 在有和没有自适应学习者模型（随机概念顺序）的情况下运行效果研究。报告差值。期望自适应胜出，但差异的大小才是有趣的数据。

2. 添加多模态探测：相同的概念问题以文本、语音和拍照方式呈现。测量学习者是否通过他们偏好的模态更快收敛。

3. 构建家长仪表板：练习主题、掌握度轨迹、即将到来的概念、安全事件（任何防护命中）。符合 COPPA 标准。

4. 添加语言切换模式：导师接受西班牙语输入并用西班牙语教学。测量 X-Guard 覆盖范围。

5. 压力测试记忆隐私：验证学习者 A 即使通过语音片段重放攻击也无法看到学习者 B 的数据。记录尝试的访问并发出告警。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|---------|---------|
| 苏格拉底式策略 | "先提问，不直接给出答案" | 导师问一个引导性问题，而不是给出答案 |
| 贝叶斯知识追踪 | "BKT" | 每个概念掌握概率的经典学习者模型方程 |
| FSRS | "自由间隔重复调度器" | 2024 年的间隔重复调度器，优于 SM-2 |
| 课程图谱 | "概念 DAG" | Neo4j 的概念图，包含先决条件边 |
| 情景记忆 | "每次互动的日志" | 每次互动存储供后续检索 |
| 语义记忆 | "学习到的模式存储" | 从情景记忆中提升的压缩错误和偏好 |
| COPPA | "儿童隐私法" | 限制从 13 岁以下儿童收集数据的美国法律 |
