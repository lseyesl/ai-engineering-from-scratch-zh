# LLM 的差分隐私

> DP-SGD 仍然是标准——注入噪声的梯度更新提供了形式化的 (ε, δ) 保证。在计算、内存和效用上的开销是显著的；参数高效的 DP 微调（LoRA + DP-SGD）是 2025 年的常见配置（ACM 2025）。两组证据之间存在张力：基于金丝雀（canary）的成员推理（Duan 等人，2024）报告了对语言模型的有限成功；训练数据提取（Carlini 等人，2021；Nasr 等人，2025）恢复了大量的逐字记忆。解决方案（arXiv:2503.06808，2025 年 3 月）：差距在于测量对象——插入的金丝雀 vs"最可提取的"数据。新的金丝雀设计实现了无需影子模型的基于损失的 MIA，并首次对在真实数据上训练、具有实际 DP 保证的 LLM 进行了有意义的 DP 审计。替代方案：PMixED（arXiv:2403.15638）——通过在下一 token 分布上使用专家混合的推理时私密预测；DP 合成数据生成（Google Research 2024）。新兴攻击：通过 LLM 反馈的差分隐私逆转——置信度分数泄漏。

## 学习目标

- 定义 (ε, δ)-差分隐私并陈述 DP-SGD 的通用方案。
- 解释 2024-2025 年的张力：金丝雀 MIA 与训练数据提取给出了不同的图景。
- 描述 PMixED 以及为什么推理时私密预测是 DP 训练的替代方案。
- 描述通过 LLM 反馈的差分隐私逆转攻击。

## 问题

LLM 会记忆。Carlini 等人 2021 年表明生产语言模型会按需逐字重现训练文本。DP 是形式化的防御：训练使得输出对任何单个训练示例都可证明地不敏感。2024-2025 年的证据表明 DP-SGD 是必要的，但部署的 ε 值可能与威胁模型不匹配。

## 概念

### (ε, δ)-差分隐私

随机化算法 M 是 (ε, δ)-DP 的，如果对于任意两个相差一个示例的数据集和任意事件 S：
P(M(D) ∈ S) <= e^ε * P(M(D') ∈ S) + δ。

解释：输出分布足够接近（由 ε 参数化），使得任何单个个体的贡献不能被可靠地推断，除概率 δ 外。

### DP-SGD

Abadi 等人 2016。标准方案：
1. 采样一个小批次。
2. 计算每个示例的梯度。
3. 将每个示例梯度裁剪到阈值 C。
4. 对裁剪后的梯度求和，并添加标准差为 σ * C 的高斯噪声。
5. 使用有噪声的和更新参数。

隐私成本由会计员（Moments Accountant、Rényi DP 会计员）跟踪。LLM 文献中报告的 ε 值因威胁模型、数据敏感性和效用目标而有很大差异；没有普遍"安全"的默认 ε。已发表的示例在 LLM 训练设置中大致跨越 ε ≈ 1–10，但这些只是示例——并非推荐的默认值。

### LoRA + DP-SGD

前沿模型的完整 DP-SGD 是不可行的。LoRA（Hu 等人 2022）将梯度更新限制在一个小型适应器上，减少了每个示例梯度存储。LoRA + DP-SGD 是 2025 年的常见配置。DP 保证适用于适应器；基础模型保持不变。

### 2024-2025 年的张力

两条证据线：

- **金丝雀 MIA（Duan 等人 2024）。** 将唯一的金丝雀插入训练数据，测量成员推理攻击者能否识别它们。报告了对语言模型的有限成功。表明 MIA 是困难的。
- **训练数据提取（Carlini 2021、Nasr 等人 2025）。** 用前缀提示模型；测量是否从训练中恢复逐字文本。报告了大量记忆。表明 MIA 在相关意义上是容易的。

2025 年 3 月的解决方案（arXiv:2503.06808）：两者测量的是不同的事物。MIA 在插入的金丝雀上问"示例 e 在 D 中吗？"。提取问的是"我能从 D 中恢复什么？"。"最可提取的"示例才是隐私所关心的；金丝雀低估了这一点，因为它们不是为了可提取性而优化的。

新的金丝雀设计。无需影子模型的基于损失的 MIA。首次在真实数据上对具有实际 DP 保证的 LLM 进行的有意义的 DP 审计。

### DP 训练的替代方案

- **PMixED（arXiv:2403.15638）。** 推理时的私密预测。下一 token 分布上的专家混合；每个专家看到训练数据的一个分片；聚合添加噪声以实现 DP。完全避免了 DP 训练。
- **DP 合成数据生成（Google Research 2024）。** 使用 DP-SGD 进行 LoRA 微调，采样合成数据，在合成数据上训练下游分类器。

两者以不同的威胁模型为代价，绕过了完整 DP 训练的效用成本。

### 通过 LLM 反馈的差分隐私逆转

2025 年新兴攻击。使用 DP 训练模型的置信度分数作为预言机来重新识别个体。即使输出不泄露，置信度分布也可能泄露。

防御：不暴露置信度，或在暴露前截断/量化它们。这是超越 (ε, δ)-DP 训练的额外要求。

## 使用它

`code/main.py` 在一个玩具二分类数据集上模拟 DP-SGD。你可以调整噪声乘数 σ 和裁剪范数 C，并跟踪 (ε, δ) 预算和准确率成本。一个"金丝雀攻击"插入一个独特的训练示例，并测量在 DP 前后对数损失测试能否检测到它。

## 交付物

本课程产出 `outputs/skill-dp-audit.md`。给定一个语言模型部署上的 DP 声明，它会审计：(ε, δ) 值、使用的会计员、MIA 评估协议、以及置信度暴露向量是否已被评估。

## 练习

1. 运行 `code/main.py`。在 σ ∈ {0.5, 1.0, 2.0} 范围内调整并报告 (ε, δ)-准确率权衡。确定效用崩溃的点。
2. 实现金丝雀插入和对数损失测试。在 σ = 1.0 的 DP-SGD 前后测量检测率。
3. 阅读 Nasr 等人 2025 年关于训练数据提取的论文。为什么在中等 ε 下提取成功不会崩溃？这对 MIA 作为评估意味着什么？
4. 设计一个完全在推理时运行的使用 PMixED（arXiv:2403.15638）的部署。PMixED 解决的是 DP-SGD 没有解决的哪种威胁模型？
5. 草拟通过 LLM 反馈的 DP 逆转攻击。设计一个限制置信度分数泄漏的对策，并估算其部署成本。

## 关键术语

| Term | What people say | What it actually means |
|------|-----------------|------------------------|
| DP | "(ε, δ)-差分隐私" | 形式化隐私：邻接数据集变化下输出分布接近 |
| DP-SGD | "注入噪声的 SGD" | 梯度裁剪 + 高斯噪声添加；标准 DP 训练 |
| LoRA + DP-SGD | "高效的私密微调" | 在低秩适应器上的 DP-SGD；2025 年标准配置 |
| MIA | "成员推理" | 确定某示例是否在训练数据中的攻击 |
| Canary | "插入的水印示例" | 用于测量 DP 泄漏的独特训练示例 |
| PMixED | "私密推理混合" | 通过下一 token 分布上的专家混合进行推理时 DP |
| DP Reversal | "置信度泄漏攻击" | 使用模型置信度作为预言机进行重新识别的攻击 |
