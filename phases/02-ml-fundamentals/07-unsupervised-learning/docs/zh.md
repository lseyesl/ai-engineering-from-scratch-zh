## 问题

到目前为止的每个 ML 课程都假设有标签数据："这是输入，这是正确的输出。"在现实世界中，标签是昂贵的。一家医院有数百万患者记录，但没有人手动为每条记录标记疾病类别。一个电商网站有数百万用户会话，但没有人手工标注客户细分。一个安全团队有网络日志，但没有人标记每个异常。

无监督学习在没有人告诉它找什么的情况下发现模式。它对相似的数据点进行分组，发现隐藏的结构，并发现异常。如果说监督学习是用有答案的教科书学习，那么无监督学习就是盯着原始数据直到模式自己显现出来。

关键在于：没有标签，你无法直接衡量"对"或"错"。你需要不同的工具来评估你的算法发现的结构是否有意义。

## 概念

### 聚类：将相似的事物分组

聚类将每个数据点分配到一个组（簇），使得同一组内的点比与其他组的点更相似。问题始终是："相似"是什么意思？

### K-Means：主力算法

K-Means 将数据划分成恰好 K 个簇。每个簇有一个质心（它的重心），每个点属于最近的质心。

Lloyd 算法：
1. 随机选择 K 个点作为初始质心
2. 将每个数据点分配给最近的质心
3. 将每个质心重新计算为其分配点的均值
4. 重复步骤 2-3 直到分配不再改变

目标函数（惯性）衡量每个点到其分配质心的总平方距离。K-Means 最小化这个值，但只找到局部最小值。不同的初始化可能给出不同的结果。

### 选择 K

两种标准方法：

**肘部法：** 对 K = 1, 2, 3, ..., n 运行 K-Means。绘制惯性 vs K。寻找"肘部"，即增加更多簇不再显著降低惯性的位置。

**轮廓系数：** 对每个点，衡量它与其自身簇的相似度（a）与最近的其他簇的相似度（b）。轮廓系数 = (b - a) / max(a, b)，范围从 -1（错误簇）到 +1（良好聚类）。在所有点上取平均值以获得全局分数。

### DBSCAN：基于密度的聚类

K-Means 假设簇是球形的，并要求你提前选择 K。DBSCAN 不作任何假设。它将簇发现为由稀疏区域分隔的密集区域。

两个参数：
- **eps**：邻域半径
- **min_samples**：形成密集区域所需的最小点数

三种类型的点：
- **核心点**：在 eps 距离内至少有 min_samples 个点
- **边界点**：在核心点的 eps 范围内，但本身不是核心点
- **噪声点**：既不是核心也不是边界。这些是离群点。

优势：能发现任意形状的簇，自动确定簇的数量，识别离群点。弱点：难以处理密度变化的簇。

### 高斯混合模型（GMM）

K-Means 给出硬分配：每个点只属于一个簇。GMM 给出软分配：每个点有属于每个簇的概率。

GMM 假设数据由 K 个高斯分布的混合生成，每个高斯分布有自己的均值和协方差。期望最大化（EM）算法在以下步骤之间交替：

- **E 步**：计算每个点属于每个高斯的概率
- **M 步**：更新每个高斯的均值、协方差和混合权重，以最大化数据的似然

GMM 可以建模椭圆形的簇（不仅仅是 K-Means 那样的球形），并能自然地处理重叠的簇。

### 何时使用哪种方法

| 方法 | 最适合 | 避免当 |
|------|--------|--------|
| K-Means | 大数据集、球形簇、已知 K | 不规则形状、存在异常值 |
| DBSCAN | 未知 K、任意形状、异常检测 | 密度变化大、维度非常高 |
| 层次聚类 | 小数据集、需要树状图、未知 K | 大数据集（O(n^2) 内存） |
| GMM | 重叠簇、需要软分配 | 非常大的数据集、维度过高 |

## 关键术语

| 术语 | 含义 |
|------|------|
| 聚类 | 将数据划分为子集，使得组内相似性超过组间相似性 |
| 质心 | 分配给一个簇的所有点的均值；K-Means 用它作为簇的代表 |
| 惯性 | 每个点到其分配质心的平方距离之和。越低越紧凑 |
| 轮廓系数 | 对每个点 (b - a) / max(a, b)，其中 a 是平均簇内距离，b 是平均最近簇距离 |
| 核心点 | 在 DBSCAN 中，在 eps 距离内至少有 min_samples 个邻居的点 |
| EM 算法 | 期望最大化：迭代计算隶属度概率（E 步）和更新分布参数（M 步） |
