# AI Scientist v2 — 研讨会级别的自主研究

> Sakana 的 AI Scientist v2（Yamada 等人，arXiv:2504.08066）运行完整的研究循环：假设、代码、实验、图表、论文撰写、投稿。它是第一个生成的论文通过 ICLR 2025 研讨会同行评审的系统。独立评估（Beel 等人）发现 42% 的实验因编码错误而失败，文献综述经常将已有概念误标为新颖。Sakana 自己的文档警告说，代码库会执行 LLM 编写的代码，并建议使用 Docker 隔离。这幅图景的两面都是重点。

**类型：** 学习
**语言：** Python（标准库，研究循环状态机玩具）
**前置要求：** Phase 15 · 03（AlphaEvolve），Phase 15 · 04（DGM）
**时间：** ~60 分钟

## 问题

研究是一项开放式的任务。与 AlphaEvolve 的算法搜索或 DGM 的基准受限自我修改不同，研究结果没有机器可检查的正确性标准。论文由评审者评判，而不是单元测试。这使得循环更难闭合——但如果闭合了，价值也更大，因为研究是复合进步所在。

AI Scientist v1（Sakana，2024）通过从人类编写的模板开始来闭合循环。LLM 在固定的脚手架内填充实验。AI Scientist v2（Yamada 等人，2025）通过使用带有视觉语言模型（vision-language model）批评循环的智能体树搜索（agentic tree search）来移除模板要求。该系统生成想法、实施实验、制作图表、撰写论文，并根据评审反馈进行迭代。

同行评审结论：一篇 v2 生成的论文被 ICLR 2025 研讨会接收（已披露）。独立评估结论：该系统远不可靠。两者都是事实。

## 概念

### 架构

1. **想法生成。** LLM 根据主题和先前文献提出研究想法。v1 使用模板；v2 使用在假设空间上的智能体搜索。
2. **新颖性检查。** 文献检索步骤检查该想法是否已发表。这是 Beel 等人的评估中发现误标的步骤——已有方法经常被归类为新颖。
3. **实验计划。** 智能体起草实验方案并编写代码。
4. **执行。** 代码在沙箱中运行。失败被反馈到重试循环中。在 Beel 等人的测量中，42% 的实验在此阶段因编码错误而失败。
5. **图表生成。** 视觉语言模型读取生成的图表并重写以提高清晰度。这是 v2 的关键技术新增。
6. **论文撰写。** LLM 起草论文，与内部评审者迭代。
7. **可选：投稿。** 论文提交到会议。

### 研讨会接收结果意味着什么

一篇 v2 生成的论文通过了 ICLR 2025 研讨会的同行评审。作者向项目委员会披露了论文的起源。这个接收是一个数据点；它并不是声称该系统"做研究"的许可。

重要背景：研讨会论文的标准低于主会议论文。同行评审是有噪声的；在任何一天，只有一小部分投稿被接收。一次成功是概念验证，而不是可靠性声明。Nature 2026 论文记录了端到端循环，并且本身是由人类研究人员共同撰写的；它并不是"系统写了一篇 Nature 论文"。

### 独立评估发现了什么

Beel 等人（arXiv:2502.14297）进行了外部评估。主要发现：

- **实验失败。** 42% 的实验因编码错误（错误的导入、形状不匹配、未定义的变量）而失败。重试循环捕获了一部分，但不是全部。
- **新颖性误标。** 文献检索步骤经常将已有概念标记为新颖。这是研究领域的幻觉（hallucination）等价物。
- **呈现质量差距。** 视觉语言图表批评产生了出版级别的视觉效果，掩盖了底层的实验弱点。

最后一个发现对本阶段很重要。一个能产生令人信服的输出却没有做令人信服的研究的系统，比一个明显失败的系统更危险，而不是更安全。评估必须触及底层声明，而不是停留在图表层面。

### 沙箱逃逸问题

Sakana 自己的仓库 README 警告：

> 由于本软件的性质——它执行 LLM 生成的代码——我们无法保证安全性。存在危险包、不受控制的网络访问以及产生意外进程的风险。请自行承担风险，并考虑使用 Docker 隔离。

这是在未验证领域中自主性的操作形态。LLM 编写代码；代码运行；代码可以做进程被允许做的任何事情。如果没有一个硬性限制文件系统、网络和进程操作的沙箱，任何自我导向的研究智能体都可以泄露数据、消耗计算资源或重写自身。

AlphaEvolve 的沙箱故事更容易，因为它的评估器是严格的。AI Scientist v2 的循环运行开放式代码和开放式目标。这就是为什么它需要更强的隔离（最低 Docker；首选 seccomp/gVisor）以及在每个输出离开系统之前进行人工审查。

### v2 在前沿堆栈中的位置

| 系统 | 目标 | 输出类型 | 评估器 | 已知失败 |
|---|---|---|---|---|
| AlphaEvolve | 算法 | 代码 | 单元测试 + 基准 | 受评估器严格性限制 |
| DGM | 智能体脚手架 | 代码 | SWE-bench | 奖励黑客（reward hacking） |
| AI Scientist v2 | 研究论文 | 文本 + 代码 + 图表 | 同行评审（弱） | 实验失败、误标、修饰掩盖弱点 |

v2 是三者中自动评估器最弱的，输出面最广的，通向公开产物的路径最短的。操作控制（沙箱、审查、披露）承担了大部分安全工作。

## 使用它

`code/main.py` 将 v2 循环模拟为一个状态机：想法 → 新颖性检查 → 实验 → 图表 → 论文撰写 → 评审 → 接收或迭代。每个状态都有一个可配置的失败概率，取自 Beel 等人的发现。运行 N 次循环的模拟器并统计：

- 有多少想法到达投稿阶段。
- 有多少投稿会有一个被精美论文掩盖的关键实验缺陷。
- 重试预算如何在质量和产出之间权衡。

## 交付它

`outputs/skill-ai-scientist-sandbox-review.md` 是一个双门审查清单，用于在离开沙箱之前检查任何由研究循环智能体产生的内容。

## 练习

1. 使用默认参数运行 `code/main.py`。有多大比例的循环运行产生了"干净的"论文？有多大比例产生了带有实验失败缺陷但被图表批评修饰过的论文？

2. 默认值已经使用了 Beel 等人的 42%/25%。分别用 `--experiment-failure 0.20 --novelty-mislabel 0.10` 和 `--experiment-failure 0.60 --novelty-mislabel 0.40` 重新运行。两次运行之间，被修饰但有缺陷的论文比例如何变化？

3. 阅读 Sakana 的 AI Scientist v2 仓库 README 中关于沙箱要求的内容。列出你在多天自主运行中会应用的（除 Docker 之外的）两个额外限制。

4. 阅读 Beel 等人关于呈现质量差距的第 4 节。设计一个额外的评估器，能够捕获看起来精美但实验上有缺陷的论文。

5. 提出一个比"让博士读每篇论文"更具可扩展性的研究智能体输出的人工审查协议。识别瓶颈并围绕它进行设计。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|---|---|---|
| AI Scientist v1 | "Sakana 的模板化研究智能体" | 将实验填入固定脚手架 |
| AI Scientist v2 | "无模板研究智能体" | 带有 VLM 图表批评的智能体树搜索 |
| 智能体树搜索（Agentic tree search） | "分支研究智能体" | 并行扩展多个实验计划；由内部批评者剪枝 |
| 视觉语言批评（Vision-language critique） | "VLM 修饰图表" | 多模态模型读取图表并重写以提高清晰度 |
| 文献检索（Literature retrieval） | "新颖性检查" | 搜索先前工作以确认想法新颖性——有记录表明会误标 |
| 修饰掩盖（Polish masking） | "漂亮的论文，破碎的研究" | 呈现质量超过实验质量；隐藏弱点 |
| 沙箱逃逸（Sandbox escape） | "LLM 代码突破" | 智能体执行的代码做了循环设计者未预期的事情 |
