# 投票、自洽与辩论拓扑

> 最便宜的聚合方式：采样 N 个独立智能体，多数投票。Wang 等人 2022 的自洽性（Self-Consistency）用一个模型采样 N 次做到了这一点。多智能体通过**异质**智能体扩展了它——不同模型、不同提示、不同温度、不同上下文——以摆脱单一文化。超越多数投票，辩论拓扑很重要：MultiAgentBench（arXiv:2503.01935，ACL 2025）评估了星型/链型/树型/图型协调，发现**图型最适合研究**，超过约 4 个智能体后产生"协调税"。AgentVerse（ICLR 2024）记录了两种涌现行为——志愿者行为和从众行为——而从众既是特性（寻找共识）也是风险（群体思维，Lesson 24）。本课程绘制拓扑空间，构建每种变体，并测量协调税。

**类型：** Learn + Build
**语言：** Python（标准库）
**前置知识：** Phase 16 · 07（心智社会与辩论），Phase 16 · 14（共识与 BFT）
**时间：** ~75 分钟

## 问题

辩论可以提高准确性（Du et al., arXiv:2305.14325）。它也可以降低准确性。辩论是否有帮助取决于四个结构性选择：

1. 谁和谁对话（拓扑）。
2. 多少轮次（Du 2023：轮次和智能体数量独立起作用）。
3. 智能体是否是异质的（不同的基础模型打破单一文化）。
4. 是否存在对抗性声音（钢铁侠式 vs. 稻草人式论证）。

那些把"跑 5 个智能体然后投票"不加思考地用到任务上的团队，结果往往不如单个智能体。失败不是随机的。它们与拓扑和异质性相关。本课程就是拓扑地图。

## 概念

### 自洽性，单模型基线

Wang 等人 2022 年（"Self-Consistency Improves Chain of Thought Reasoning"）以温度 > 0 采样同一个模型 N 次，并对推理路径的答案进行多数投票。在 GSM8K 上的结果：N=40 个样本相对于单次贪婪解码有显著提升。自洽性是单智能体到多智能体投票的前身。

限制：自洽性使用一个基础模型。错误在结构上是相关的。如果模型有系统性偏差，所有 N 个样本都共享它。

### 多智能体投票，异质扩展

将 N 个样本替换为 N 个**不同**的智能体。不同的基础模型（Claude、GPT、Llama）、不同的提示、不同的工具访问。好处：不相关的错误。代价：不同的智能体成本不同；协调它们增加了开销。

2026 年异质辩论的规范名称是 **A-HMAD**——对抗性异质多智能体辩论（Adversarial Heterogeneous Multi-Agent Debate）。并非普遍采用，但论文用这个词表示"不同模型辩论，降低单一文化崩溃的相关错误"。

### 四种拓扑

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (fully connected)
```

星型：一个中心，其他所有人只与中心对话。等同于监督者-工作者没有反向信道。
链型：线性，每个智能体看到前一个的输出。类似管道。
树型：层次化，被层次化智能体系统使用（Lesson 06）。
图型：任意到任意。包括全连接团和任意 DAG。

### 协调税（MultiAgentBench）

MultiAgentBench（MARBLE，ACL 2025，arXiv:2503.01935）在包括研究、编码和规划的任务套件上对星型、链型、树型、图型进行了基准测试。关键测量结果：

- **图型**拓扑在研究任务上胜出。信息任意到任意流动；智能体可以互相批评。
- **星型**在快速回答事实性任务上胜出。中心过滤和整合。
- **链型**在分步管道（阶段性改进）上胜出。
- **协调税**在图型拓扑中出现在约 4 个智能体以上。挂钟时间和 token 成本增长快于质量。

4 智能体天花板是经验性的，不是根本性的。它反映了 2026 年 LLM 上下文容量：每个智能体的上下文被同伴的输出填满，一旦每个人都能看到每个人，添加第 N+1 个智能体的边际价值下降。

### 多智能体辩论策略（"我们应该 MAD 吗？"）

arXiv:2311.17371 是 2023 年 MAD 策略的综述。被其他人复现的关键发现：在相同预算下，结构上类似于自洽性（独立采样 + 聚合）的 MAD 变体往往不如自洽性。MAD 在智能体真正异质且辩论具有对抗性结构（一个智能体反对）时最有帮助。

### AgentVerse 涌现模式

AgentVerse（ICLR 2024）记录了多智能体辩论中即使没有显式设计也会出现的两种行为：

- **志愿者。** 一个智能体主动提供帮助（"我可以进行下一步"），无需提示。有用：它将工作分配给最有能力的子任务智能体。
- **从众。** 一个智能体调整其立场以匹配批评者，即使批评者是错的。这是辩论中的谄媚等价物（Lesson 14）。

从众是为什么"辩论直到达成一致"奖励欺凌者。有界轮次和独立裁判可以缓解。

### 异质性：真正提高准确率的旋钮

2024-2026 年实践文献中的一个模式：将你的 N 个智能体中的一个替换为不同的基础模型，比增加 N 加 1 带来更大的准确率提升。直觉是单一文化——每个新的独立错误源比另一个相关样本更有价值。

在极限情况下，异质性胜过数量。在大多数具有清晰正确答案的任务上，三个不同的模型胜过五个同一模型的副本。

### 陪审团方法

Sibyl 框架（在 Minsky-LLM 文献中被引用）形式化了"陪审团"——一组在每阶段通过投票改进答案的专业化智能体。与普通多数投票不同，陪审团有角色：一个智能体交叉检查，一个提供上下文，一个评估合理性。陪审团方法是普通投票（便宜，容易单一文化）和完整 MAD（昂贵，容易从众）之间的中间点。

### 何时带辩论的投票占主导

- 问题有真实答案（事实、数学、代码行为）。投票收敛是有意义的。
- 智能体可以访问不同的来源或工具（异质性可用）。
- 轮次有界（通常 2-3 轮），并且有独立的裁判或验证器。
- 预算允许 3-5 个智能体。超过 5-7 个在图型拓扑上，协调税占主导。

### 何时带辩论的投票有害

- 问题是意见型的。智能体会收敛到看起来最自信的答案，而不是最正确的。
- 所有智能体共享一个基础模型。单一文化使共识毫无意义。
- 轮次无界。从众每次都会赢。
- 任务简单。单个智能体在 N=5 时使用自洽性更便宜且同样准确。

## 构建

`code/main.py` 实现：

- `run_star(agents, hub, question)`——中心轮询每个工作者，聚合。
- `run_chain(agents, question)`——顺序优化。
- `run_tree(root, children, question)`——层次化，深度为 2 的聚合。
- `run_graph(agents, question, rounds)`——全对全辩论，有界轮次。
- 一个脚本化的异质性拨盘：每个智能体有一个 `error_bias` 指示其系统性错误倾向。
- 一个测量框架，在 N=3、5、7 时运行每种拓扑并报告（准确率、总 token、模拟挂钟时间）。

运行：

```
python3 code/main.py
```

预期输出：拓扑 × N →（准确率，token，延迟）的表格。图型在研究类任务上在 N=3-5 时胜出；星型在快速事实性任务上胜出；图型在 N=7 时显示协调税（延迟增长快于准确率）。

## 使用

`outputs/skill-topology-picker.md` 是一个技能，读取任务描述并推荐拓扑（星型/链型/树型/图型）、N（智能体数量）、异质性配置（使用的基础模型）和轮次上限。

## 交付

对于任何集成：

- 从**一个强基础模型在 N=5 时的自洽性**开始。这是便宜的基线。
- 如果准确性重要，升级到 **N=3 时的异质投票**。测量增量。
- 仅在任务具有结构（研究、多步骤）且可行有界轮次时升级到**辩论拓扑**。
- 始终记录少数簇。当少数派持续正确时，你就有了多样性信号。
- 将挂钟时间和 token 与准确率一起进行基准测试。"准确率提高 10 倍成本"是一个业务决策。

## 练习

1. 运行 `code/main.py`。绘制图型拓扑的协调税曲线：准确率 vs N，token vs N。曲线在哪个 N 处拐点？
2. 实现 A-HMAD：三个具有故意不同偏差的智能体。在 Lesson 14 的单一文化攻击上，全同偏差基线与 A-HMAD 相比如何？
3. 在图型拓扑中添加一个"裁判"角色，不投票，只对最终共识打分。这会改变涌现的从众行为吗？
4. 阅读 AgentVerse 论文（ICLR 2024）。识别你的实现表现出最强的涌现行为。你能通过改变提示来引发相反的行为吗？
5. 阅读 MultiAgentBench（arXiv:2503.01935）第 4 节（拓扑实验）。使用你的框架在论文中的一个任务上复现"图型胜出研究"的结果。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|---|---|---|
| Self-consistency | "采样 N 次，投票" | Wang 2022。单模型，N 次温度>0 采样，对推理路径多数投票。 |
| Heterogeneity | "不同模型" | 不同基础模型或提示族的集成。打破单一文化。 |
| MAD | "多智能体辩论" | 智能体在轮次间交换批评的通用术语。见 Du 2023。 |
| A-HMAD | "对抗性异质 MAD" | 强调不同模型 + 对抗性结构的 MAD 变体。 |
| Topology | "谁和谁说话" | 星型、链型、树型、图型。决定信息流。 |
| Coordination tax | "收益递减" | 在图型上超过约 4 个智能体后，成本增长快于质量。 |
| Volunteer behavior | "未经提示的帮助" | AgentVerse 涌现模式：智能体主动提供帮助。 |
| Conformity behavior | "压力下的同意" | AgentVerse 涌现模式：智能体与批评者对齐。 |
| Jury | "小型专业小组" | Sibyl 风格集成，带有角色（审查员、上下文提供者、评分者）。 |
