# 生成式智能体与涌现模拟 (Generative Agents and Emergent Simulation)

> Park 等人 2023 年（UIST '23, arXiv:2304.03442）在 **Smallville** 沙盒中填充了 25 个智能体，采用三部分架构：**记忆流**（memory stream，自然语言日志）、**反思**（reflection，智能体对其自身流生成的高层次综合）和**计划**（plan，日级行为，再分解为子计划）。标志性结果是情人节派对涌现：一个智能体被植入"想办一场情人节派对"的种子，无需额外脚本，邀请函就在群体中传播开来，协调了日期，派对如期举行——而其他 24 个智能体最初对此一无所知。消融实验表明，所有三个组件对可信度都是必需的。已记录的问题包括空间规范错误（进入已关门的商店、共用单人卫生间）。这是 2026 年智能体模拟和多智能体社会评估的参考架构。

**类型：** Learn + Build
**语言：** Python (stdlib)
**前置知识：** Phase 16 · 04（原始模型），Phase 16 · 13（共享内存）
**时间：** ~75 分钟

## 问题 (Problem)

大多数多智能体系统是紧密编排的团队：规划者规划、编码者编码、审查者审查。这对定义明确的任务有效，但无法捕捉当智能体拥有记忆、优先级和开放世界时产生的非脚本化涌现行为。研究、社会模拟以及日益增长的游戏 AI 都需要这第二种类型。

Smallville 架构是这方面的标杆。在 Park 2023 之前，最好的智能体模拟只是浅层的脚本跟随者；在此之后，该模式成为开放世界中生成式智能体的默认选择。如果你在 2026 年构建智能体模拟，要么使用 Smallville 的三个组件，要么明确说明为什么不用。

## 概念 (Concept)

### 三个组件 (The three components)

**记忆流 (Memory stream)。** 一个仅追加的日志，记录观察、行动、反思和计划。每条记录包含时间戳、类型、描述（自然语言）以及派生元数据：**新近度**（recency）、**重要性**（importance，智能体自评 1-10）和**相关性**（relevance，与当前查询的余弦相似度）。

```
[2026-02-14 09:12:03] observation: Isabella Rodriguez asked me if I like jazz
[2026-02-14 09:14:22] reflection:   I enjoy long conversations about music
[2026-02-14 10:05:00] plan:         Attend Isabella's Valentine's Day party tonight
```

记忆检索结合三个分数：`score = w_recency * e^(-decay * age) + w_importance * importance + w_relevance * cos_sim`。Top-k 条记录进入当前提示。

**反思 (Reflection)。** 定期（每 N 条记忆或在重要事件发生时），智能体从近期记忆中生成更高层次的综合。反思条目回到记忆流中，像其他记忆一样可检索。这就是智能体构建"理解"的方式——该架构中相当于长期信念的机制。

**计划 (Plan)。** 自上而下的分解。首先，日级计划的粗略轮廓（"去上班，和 Klaus 吃晚饭"）。然后是小时级计划。然后是行动级计划。计划是可修订的：当观察与计划矛盾时，智能体重新规划受影响的部分。

### 为什么三者都重要（消融实验）

Park 等人进行了消融实验，分别去掉观察、反思和计划。每个消融都会损害可信度：

- 没有**观察**，智能体会错过上下文，基于过时的信念行动。
- 没有**反思**，智能体无法形成高层次信念；交互停留在表面。
- 没有**计划**，行为变成反应性噪声；目标消散。

人类评分者的可信度分数在三者齐全时最高；去掉任何一个都会产生可测量的退化。

### 情人节涌现 (The Valentine's Day emergence)

一个智能体 Isabella Rodriguez 被植入目标"想在 2 月 14 日下午 5 点在 Hobbs Cafe 办一场情人节派对。"其他 24 个智能体没有收到这样的种子。经过模拟天数：

1. Isabella 的计划包括邀请他人。
2. 每份邀请成为邻居记忆流中的一条观察。
3. 邻居的反思生成信念："Isabella 要办派对。"
4. 邻居的计划包含"参加 2 月 14 日的派对。"
5. 邻居告诉其他邻居。邀请在没有中央协调的情况下传播。
6. 2 月 14 日下午 5 点，多个智能体聚集在 Hobbs Cafe。

这就是技术意义上的涌现：系统级行为（派对）源于局部交互（双边邀请 + 个人规划），没有中央协调者。

### 已记录的失败模式 (The documented failure modes)

Park 等人明确记录了：

- **空间规范错误 (Spatial norm errors)。** 智能体走进已关门的商店。智能体试图使用同一个单人卫生间。智能体在不适合用餐的房间吃饭。模型无法仅从环境推断社会物理规范。
- **记忆溢出 (Memory overflow)。** 深度模拟运行导致记忆检索成本增长。实际补救措施：定期记忆压缩（总结并修剪）和低重要性条目的衰减。
- **反思幻觉 (Reflection hallucination)。** 反思可能发明记忆流中不存在的关系。缓解措施：在反思提示中包含源记忆 ID，并在检索时验证。

这些是与生产相关的失败模式：任何 2026 年的智能体模拟都会继承它们。

### 三组件实现规则 (Three-component implementation rules)

1. **记忆是仅追加的。** 永远不要修改记忆条目。修正是新条目。
2. **重要性评分是廉价的。** 在写入时调用 LLM 评分 1-10。缓存分数。
3. **检索是排序的，不是过滤的。** 按组合分数取 top-k；不要使用硬过滤（会丢失上下文）。
4. **反思定期运行。** 当未处理记忆的重要性总和超过阈值（例如 150）时触发。
5. **计划是可修订的。** 当新观察与计划矛盾时，只重新生成受影响的部分，而不是整个计划。

### Smallville 之外的生成式智能体

2024-2026 年的后续文献扩展了该架构：

- **用于政策/市场研究的多智能体社会模拟。** 类似 Smallville 的群体模拟用户对功能的行为反应。比 A/B 测试更快；准确性存在争议。
- **游戏 NPC AI。** 使用 Smallville 智能体的 RPG 产生涌现式故事情节，而非脚本化任务。
- **生成式智能体评估基准。** 评估指标不再是任务准确率，而是长时间运行的可信度和行为连贯性。

该架构是参考。扩展会替换组件（向量存储用于记忆、检索增强反思、神经符号计划），但保持三部分结构。

### 这对多智能体工程为何重要

Smallville 证明了当组件正确时，多智能体涌现是廉价的。该架构现在已在开源模型上复现（较小的 LLM 会优雅地降低可信度，而非急剧下降）。任何需要**涌现式社会行为**的生产系统都使用这种形态。任何需要**紧密任务执行**的系统则使用本阶段早期的监督者/角色/原语模式。

## 构建 (Build It)

`code/main.py` 使用 stdlib Python 实现了三个组件，采用脚本化智能体策略（无真实 LLM）。该演示以微型方式重现了情人节派对涌现：

- `MemoryStream` — 仅追加日志，支持新近度/重要性/相关性检索。
- `reflect(stream)` — 对近期高重要性记忆的脚本化反思。
- `plan(agent_state)` — 基于当前信念的日级和小时级计划。
- 场景：5 个智能体。智能体 1 以"下午 5 点办派对"开始。经过模拟滴答，邀请传播，智能体聚集。

运行：

```
python3 code/main.py
```

预期输出：逐滴答的追踪。到最后一个滴答时，至少 3/5 的智能体在其计划中显示派对，并在派对地点聚集。单个种子在没有协调者的情况下产生了协调到达。

## 使用 (Use It)

`outputs/skill-simulation-designer.md` 设计一个生成式智能体模拟：智能体数量、记忆模式、反思节奏、计划周期和评估指标。

## 交付 (Ship It)

生产模拟的规则：

- **记忆就是数据库。** 在规模下选择一个真正的存储（向量数据库、Postgres）。内存中的 stdlib 只适合原型。
- **记录检索追踪。** 对每个行动，记录驱动它的 top-k 记忆。这是你的调试能力。
- **预算每个智能体的 token。** 每个智能体每滴答的检索 + 反思 + 计划是 O(k) 次 LLM 调用。N 个智能体 × T 个滴答 × 每次调用的调用次数可能远超你的预算。
- **定期压缩记忆。** 总结并修剪低重要性条目。保留策略是一个设计决策，不是细节。
- **显式检测空间/社会规范违反。** 该架构不会学习它们。

## 练习 (Exercises)

1. 运行 `code/main.py`。确认 3+ 个智能体在派对聚集。将智能体增加到 10 个——涌现还会发生吗？
2. 移除反思步骤。行为看起来如何？对照 Park 2023 中的消融发现。
3. 引入一个竞争性种子目标（"Klaus 想在下午 5 点做一个研究讲座"）。智能体会分裂，还是一个目标占主导？由什么决定？
4. 添加空间约束：Hobbs Cafe 最多容纳 4 个智能体。模拟能优雅处理溢出吗，还是会遇到"单人卫生间"失败模式？
5. 阅读 Park 等人（arXiv:2304.03442）第 6 节（涌现行为实验）。找出一个在你的微型模拟中无法复现的行为。你需要增强架构的哪个组件？

## 关键术语 (Key Terms)

| Term | What people say | What it actually means |
|------|----------------|------------------------|
| Memory stream (记忆流) | "智能体的日记" | 观察、行动、反思、计划的仅追加日志。 |
| Recency (新近度) | "记忆有多新" | 按年龄的指数衰减分数。 |
| Importance (重要性) | "智能体有多在意" | 写入时自评 1-10。已缓存。 |
| Relevance (相关性) | "与当前查询的相关程度" | 余弦相似度（基于嵌入）。 |
| Reflection (反思) | "高层次信念" | 从近期记忆生成的综合，作为新记忆重新摄入。 |
| Plan (计划) | "日/小时/行动分解" | 自上而下的计划树。当观察矛盾时可修订。 |
| Smallville | "Park 2023 的沙盒" | 产生情人节涌现的 25 智能体模拟。 |
| Believability (可信度) | "质量指标" | 行为是否像合理智能体的人类评分者分数。 |
