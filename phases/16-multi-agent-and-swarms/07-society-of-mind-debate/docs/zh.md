# 心智社会与多智能体辩论 (Society of Mind and Multi-Agent Debate)

> Minsky 1986 年的前提——智能是专家组成的社会——每十年被重新发现一次。2023 年，Du 等人将其转化为一个具体的算法：多个 LLM 实例提出答案，阅读彼此的答案，进行批评，然后更新。经过 N 轮，它们收敛到一个共识，在六个推理和事实性任务上超越了零样本 CoT 和反思。两个发现很重要：**多个智能体**和**多轮次**各自独立地做出贡献。社会胜过单智能体独白；多轮交流胜过一次性投票。

**类型：** 学习 + 构建 (Learn + Build)
**语言：** Python（标准库）
**前置知识：** 阶段 16 · 04（原语模型）
**时间：** ~60 分钟

## 问题 (Problem)

自洽性（Self-consistency）——对一个模型多次采样并取多数答案——是你能够添加的最便宜的推理改进。它有效，但很快饱和。你可以将样本数量翻倍，却看不到另一个有意义的提升。

辩论打破了这种饱和。不是从一个模型获取 N 个独立样本，而是 N 个智能体阅读彼此的推理并修订。样本之间的相关性降低了（它们不再是独立同分布的），收敛点往往在独立同分布投票自信地犯错时却是正确的。

## 概念 (Concept)

### Du 等人 2023 年的算法 (The Du et al. 2023 algorithm)

来自 arXiv:2305.14325（ICML 2024）：

1. N 个智能体各自对问题产生初始答案。
2. 对于轮次 r = 2..R：每个智能体被展示其他智能体在 r-1 轮的答案，并被要求"考虑到这些，给出你更新后的答案。"
3. 经过 R 轮后，对最终答案进行多数投票。

论文在 MMLU、GSM8K、传记、MATH 和事实性基准上进行了测试。辩论始终优于 CoT 和自我反思。

### 两个独立旋钮 (Two independent knobs)

同一篇论文的消融实验：

- **仅智能体数量**（1 轮，N 个的多数投票）在大多数任务上优于单智能体，但会平台化。
- **仅轮次数量**（1 个智能体看到自己之前的推理）几乎没有帮助——这是反思的已知弱点。
- **两者结合**产生了大的跳跃。多个智能体之间的多轮交流驱动了增益。

### 为什么有效 (Why it works)

两种机制：

1. **暴露于分歧 (Exposure to disagreement)。** 当一个智能体看到另一个智能体的推理链得出不同结论时，它必须要么证明自己正确，要么更新。无论哪种方式，第 r+1 轮的上下文都比第 r 轮更丰富。
2. **相关错误减少 (Correlated error reduction)。** 在自洽性中，所有样本来自同一个模型，因此错误是相关的——你平均出一个自信的错误答案。不同的模型或不同的种子去相关。不同的*辩论观点*进一步去相关。

### 异构辩论 (Heterogeneous debate)

A-HMAD 及相关的后续工作为不同的智能体使用*不同的基础模型*。Llama + Claude + GPT 进行辩论减少了单一文化崩溃（第 26 课），因为一个模型家族的相关错误不会被其他模型共享。

缺点：参与辩论的弱模型可能将共识拖向它的错误答案（参见"Should we be going MAD?"，arXiv:2311.17371）。

### NLSOM——129 智能体扩展 (NLSOM — the 129-agent extension)

Zhuge 等人（"Mindstorms in Natural Language-Based Societies of Mind，" arXiv:2305.17066）将这个想法扩展到 129 个成员的社会。结果是：专业化和自组织随着规模出现，系统在视觉问答等任务上优于单智能体。

### 失败模式 (Failure modes)

- **谄媚级联 (Sycophancy cascade)。** 所有智能体都顺从听起来最自信的那个智能体。辩论崩溃为最大声的声音。提示对抗性角色（"一个智能体必须论证反方立场"）有所帮助。
- **主题漂移 (Topic drift)。** 多轮辩论会偏离原始问题。缓解措施：每轮重新注入问题。
- **计算爆炸 (Compute blowup)。** N 个智能体 × R 轮 = N·R 次 LLM 调用，每次调用的上下文都在增长。5 个智能体、5 轮的辩论是 25 次调用，且上下文不断增长。每个问题的成本可能超过单次 CoT 调用的 10 倍。

## 构建 (Build It)

`code/main.py` 在一个数学问题上运行 3 个智能体 × 3 轮的辩论，每个智能体从不同的（可能错误的）答案开始。智能体是脚本化的——每个"更新"通过按脚本化的置信度加权平均邻居的答案。收敛在逐轮日志中可见。

演示展示了两个关键效果：

- 单轮交流使智能体更接近正确答案。
- 第 2 轮之后的额外轮次显示出递减的回报（与 Du 等人的平台化现象一致）。

运行：

```
python3 code/main.py
```

## 使用 (Use It)

`outputs/skill-debate-configurator.md` 为新任务配置辩论：智能体数量、轮次数量、异构性（相同模型 vs 混合）、角色分配（对称 vs 一个对抗性）。它还在运行前估算 token 成本。

## 交付 (Ship It)

如果你交付辩论系统：

- **轮次上限为 3 (Cap rounds at 3)。** Du 等人显示 3 轮捕获了大部分增益。更多是成本，而非质量。
- **智能体上限为 5 (Cap agents at 5)。** 超过 5 个，上下文膨胀和成本占主导。
- **默认异构 (Heterogeneous by default)。** 池中至少有两个不同的基础模型。
- **对抗性槽位 (Adversarial slot)。** 一个智能体被提示无论何时都持反对意见。打破谄媚。
- **记录每一轮 (Log every round)。** 隐藏中间轮的辩论系统无法调试或审计。

## 练习 (Exercises)

1. 运行 `code/main.py`，然后将轮次设置为 5 并观察递减回报。从哪一轮开始额外的收敛停止？
2. 添加第四个智能体，赋予对抗性角色：始终与当前多数意见持反对意见。这会破坏还是改善收敛？
3. 绘制（打印）每轮的共识分数（处于多数答案的智能体比例）。何时达到 1.0，这是否等同于"正确"？
4. 阅读 Du 等人论文第 4 节的消融实验。使用此代码复现"仅智能体" vs "仅轮次" vs "两者结合"的结果。
5. 阅读"Should we be going MAD?"（arXiv:2311.17371）并列出轮询之外的两种辩论变体——例如，裁判主导、辩论链、对抗性辩论。

## 关键术语 (Key Terms)

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 心智社会 (Society of Mind) | "Minsky 的想法" | 智能作为交互的专业化组件；1986 年的框架现在通过 LLM 辩论实现。 |
| 多智能体辩论 (Multi-agent debate) | "智能体争论" | N 个智能体提出答案、互相批评、在 R 轮中修订、多数投票。 |
| 共识 (Consensus) | "他们达成一致" | 不是认识论真理——只是多数答案上的比例。可能自信地错误。 |
| 轮次 (Rounds) | "交流步骤" | 一轮 = 每个智能体阅读其他智能体的答案并更新一次。 |
| 异构辩论 (Heterogeneous debate) | "混合模型家族" | 使用不同的基础模型来去相关错误。 |
| 谄媚级联 (Sycophancy cascade) | "每个人都同意最大声的那个" | 辩论失败模式，智能体顺从最自信的智能体，无论其正确性。 |
| NLSOM | "129 智能体社会" | 自然语言心智社会；Zhuge 等人的规模化版本。 |
| 相关错误 (Correlated error) | "相同模型，相同错误" | 自洽性饱和的原因；跨不同观点的辩论去相关。 |
