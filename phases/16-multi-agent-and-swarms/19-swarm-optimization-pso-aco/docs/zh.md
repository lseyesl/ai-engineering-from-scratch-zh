# LLM 的群体优化——PSO 与 ACO (Swarm Optimization for LLMs)

> 生物启发优化正在 LLM 领域卷土重来。**LMPSO**（arXiv:2504.09247）使用 PSO，其中每个粒子的速度是一个提示，LLM 生成下一个候选；在结构化序列输出（数学表达式、程序）上效果良好。**Model Swarms**（arXiv:2410.11163）将每个 LLM 专家视为模型权重流形上的 PSO 粒子，报告在 9 个数据集上**平均提升 13.3%**，超过 12 个基线，仅需 200 个实例。**SwarmPrompt**（ICAART 2025）将 PSO + Grey Wolf 混合用于提示优化。**AMRO-S**（arXiv:2603.12933）是基于 ACO 的信息素专家，用于多智能体 LLM 路由——**4.7 倍加速**、可解释的路由证据、质量门控异步更新将推理与学习解耦。本课在提示参数空间上实现 PSO，在智能体路由上实现 ACO，测量这些经典算法为何适合 LLM 时代，以及何时不适合。

**类型：** Learn + Build
**语言：** Python (stdlib)
**前置知识：** Phase 16 · 09（并行群体网络），Phase 16 · 14（共识与 BFT）
**时间：** ~75 分钟

## 问题 (Problem)

你有一个在任务评估上得分 62% 的提示。你想改进它。朴素的做法是无梯度手动调整，这很难规模化。强化学习需要奖励信号和足够的 rollout 来训练。通过提示进行反向传播实际上是不可能的——提示是一个离散字符串，不是可微分参数。

经典的生物启发优化——PSO 用于连续搜索空间，ACO 用于路径选择——正是为此场景设计的：无梯度、基于群体、每次评估成本低廉。将它们与 LLM 配对用于无梯度搜索步骤，你就得到了一个出乎意料实用的优化器。

同样的模式适用于多智能体系统中的智能体*路由*。ACO 风格的信息素轨迹记录哪个智能体在哪种任务类型上表现最好，让路由器利用轨迹，并通过衰减信息素使路由可以被重新发现。

## 概念 (Concept)

### PSO 复习 (Kennedy & Eberhart 1995)

粒子群优化 (Particle Swarm Optimization)：在连续搜索空间中的粒子群体。每个粒子有位置 `x_i` 和速度 `v_i`。每次迭代：

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

其中 `p_best` 是粒子自身的最佳，`g_best` 是群体的最佳，`w, c1, c2` 是惯性 + 认知 + 社会权重，`r1, r2` 是随机因子。

### LLM 输出上的 PSO——LMPSO

arXiv:2504.09247 将 PSO 适配用于 LLM 生成的结构化输出（数学表达式、程序）。每个粒子是一个候选输出。速度是一个*提示*，描述如何将当前输出修改向个人/全局最佳。LLM 从速度提示生成新输出。速度的"惯性"是一个类似"做小的增量修改"的提示。

这在以下情况下效果良好：
- 输出是结构化的（可解析、可评估）。
- 适应度是自动的（测试运行、算术评估）。
- 群体较小（~10-30 个粒子），因此 LLM 调用总数可控。

当适应度需要人工审查时效果不佳——每次迭代的成本变得过高。

### Model Swarms

arXiv:2410.11163 将 PSO 从输出层移到*模型*层。每个"粒子"是一个专家 LLM（参数）。群体通过无梯度更新将参数移向集体最佳。报告：在 9 个数据集上平均提升 13.3%，超过 12 个基线，每次迭代仅需 200 个实例。

关键洞察是 LLM 专家模型已经在共享的参数流形中相邻（适配器权重、LoRA 增量）。在这个低维子空间上的 PSO 既廉价又有效。

### ACO 复习 (Dorigo 1992)

蚁群优化 (Ant Colony Optimization)：蚂蚁遍历一个图；每条路径有信息素轨迹。蚂蚁移动概率按信息素强度加权。完成任务的好蚂蚁按解决方案质量比例沉积信息素。信息素随时间衰减。

### AMRO-S——用于智能体路由的 ACO

arXiv:2603.12933 使用 ACO 进行多智能体路由。每个任务类型是一个"目的地"；每个智能体是一条可能的路线。信息素加强产生良好输出的路线。关键贡献：

- **可解释的路由证据。** 信息素强度是人类可读的信号。
- **质量门控异步更新。** 信息素仅在质量检查通过后更新，将推理与学习解耦。
- **在多智能体路由基准上实现 4.7 倍加速。**

质量门控很重要：没有它，快速但错误的智能体会积累信息素，系统锁定在错误的路由上。

### 何时为 LLM 使用 PSO / ACO

**使用 PSO 当：**
- 搜索空间是连续的或映射到连续参数（提示嵌入、LoRA 权重、数值生成参数）。
- 适应度廉价且自动。
- 群体可以很小（10-30）。

**使用 ACO 当：**
- 你有路由或路径选择问题。
- 决策随时间强化（相同的任务类型反复出现）。
- 你需要路由决策的可解释证据。

**两者都不使用当：**
- 适应度需要人工审查（每次迭代成本太高）。
- 搜索空间是离散和组合的，PSO 无法覆盖（改用遗传算法）。
- 实时决策需要严格的延迟（PSO/ACO 相对于单次启发式方法收敛较慢）。

### 为什么生物启发仍然胜出

基于梯度的方法需要可微分信号。LLM 输出和路由决策不是平凡可微分的。伪梯度方法（强化学习路由器、DPO 风格的提示调优器）有效但需要昂贵的训练。

PSO 和 ACO 只需要一个*评估器*函数。如果你能对候选输出或路由决策评分，你就可以在该空间上进行优化。这使得适用门槛低得多。

### 实际限制 (Practical limits)

- **群体预算。** N 个粒子 × T 次迭代 × 每次评估成本。对于每次 LLM 评估约 $0.02，20 个粒子的 PSO 运行 50 次迭代约需 $20。据此规划。
- **探索 vs 利用。** 信息素衰减率和 PSO 惯性需要权衡；衰减太快→忘记解决方案；太慢→卡在早期局部最优。
- **灾难性漂移 (Catastrophic drift)。** 如果适应度景观发生变化（新数据分布），两种算法都可能收敛然后发散。监控最佳适应度稳定性。

## 构建 (Build It)

`code/main.py` 实现：

- `LMPSO` — 在数值提示参数（temperature, top_k 权重）上的 PSO。每个粒子的"LLM 生成"被模拟为脚本化适应度函数。运行算法 30 次迭代并显示 g_best 收敛。
- `AMRO_S` — ACO 风格路由。3 个智能体，4 种任务类型，信息素矩阵，100 个路由任务。打印（task_type → agent choices）随时间分布以显示轨迹形成。
- 比较：随机路由 vs ACO 路由在同一任务流上。测量质量和延迟。

运行：

```
python3 code/main.py
```

预期输出：
- LMPSO：g_best 适应度在 30 次迭代内从随机提升到接近最优。
- AMRO-S：信息素表稳定在每个任务类型的正确智能体上；ACO 路由在质量上比随机好约 30-40%，并且延迟也更低（更少重试）。

## 使用 (Use It)

`outputs/skill-swarm-optimizer.md` 帮助在 PSO、ACO、遗传算法和基于梯度的优化器之间为 LLM / 智能体优化问题做出选择。

## 交付 (Ship It)

- **从小开始。** 10-20 个粒子，20-50 次迭代。只有在收敛曲线显示明显增益时才扩大规模。
- **记录每次迭代的信息素或 g_best。** 没有轨迹的情况下调试群体优化器是很痛苦的。
- **质量门控更新。** 特别是对于 ACO 路由：快速但错误的智能体不能积累信息素。
- **在分布漂移时重置衰减。** 当你的评估分布发生变化时，老化的信息素已经过时；重置或暂时加倍衰减率。
- **限制每次迭代的成本。** 发出每次迭代的成本指标。每次迭代花费 $500 但只提升 0.5% 的 PSO 是不可交付的。

2. 实现一个"灾难性漂移"实验：在第 30 次迭代后改变适应度函数。PSO 适应速度有多快？重置 `p_best` 有帮助吗？
3. 为 AMRO-S 添加质量门控：仅在评估分数 > 0.7 的运行上沉积信息素。这与无门控版本相比如何改变收敛？

5. 阅读 AMRO-S（arXiv:2603.12933）。实现解耦的"推理快速路径"与异步信息素更新。这在持续负载下如何改变系统延迟？

## 关键术语 (Key Terms)

| Term | What people say | What it actually means |
|------|----------------|------------------------|
| PSO | "粒子群优化" | Kennedy-Eberhart 1995。基于群体的无梯度优化器。 |
| ACO | "蚁群优化" | Dorigo 1992。通过信息素轨迹的路径/路由优化。 |
| LMPSO | "使用 LLM 生成的 PSO" | arXiv:2504.09247。速度是一个提示；LLM 产生候选。 |
| Model Swarms | "专家权重上的 PSO" | arXiv:2410.11163。模型参数子空间上的无梯度更新。 |
| AMRO-S | "用于智能体路由的 ACO" | arXiv:2603.12933。任务类型 × 智能体的信息素矩阵。 |
| p_best / g_best | "个人/全局最佳" | 每个粒子和整个群体迄今找到的最佳解决方案。 |
| Pheromone (信息素) | "路由记忆" | 边上的强度；随时间衰减；在质量上沉积。 |
| Quality-gated update (质量门控更新) | "只从好的运行中学习" | 信息素沉积以质量检查为条件。 |
| Catastrophic drift (灾难性漂移) | "分布漂移" | 适应度景观变化；旧的 p_best 和信息素变得过时。 |
