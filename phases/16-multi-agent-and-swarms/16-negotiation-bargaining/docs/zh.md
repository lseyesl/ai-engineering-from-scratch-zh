# 谈判与讨价还价

> 智能体就资源、价格、任务分配和条款进行谈判。2026 年的基准测试集很清晰：NegotiationArena（arXiv:2402.05863）显示 LLM 可以通过角色操纵（"绝望"）将收益提高约 20%；"Measuring Bargaining Abilities"（arXiv:2402.15813）显示买方比卖方更难，且规模无济于事——他们的 **OG-Narrator**（确定性出价生成器 + LLM 叙述者）将成交率从 26.67% 推至 88.88%；大规模自主谈判竞赛（arXiv:2503.06416）进行了约 180k 次谈判，发现**隐藏思维链的智能体**通过向对手隐藏推理而获胜；Bhattacharya 等人 2025 年关于哈佛谈判项目指标的论文将 Llama-3 评为最有效、Claude-3 最具攻击性、GPT-4 最公平。本课程实现 Contract Net 协议（FIPA 的前身，Lesson 02），连接 LLM 风格的买方/卖方，运行 OG-Narrator 风格分解，并测量成交率如何随每个结构性选择而变化。

**类型：** Learn + Build
**语言：** Python（标准库）
**前置知识：** Phase 16 · 02（FIPA-ACL 遗产），Phase 16 · 09（并行群体网络）
**时间：** ~75 分钟

## 问题

两个智能体需要就价格达成一致。仅靠纯语言提示，2024-2026 年的 LLM 以令人惊讶的低成交率达成交易（在 arXiv:2402.15813 的严格参数化讨价还价中约 27%）。规模不能解决这个问题：GPT-4 在讨价还价方面并不比 GPT-3.5 更好；它只在讨价还价的*语言*方面更好。

根本问题是 LLM 混淆了两项工作——决定出价和叙述出价。OG-Narrator 将它们分开：一个确定性出价生成器计算数值移动；LLM 只负责叙述。成交率跃升至约 89%。

这反映了经典的多智能体发现：将机制与通信层解耦获胜。Contract Net 协议（FIPA，1996；Smith，1980）是参考任务市场机制。将 LLM 插入叙述槽，你就得到了一个现代的 LLM 驱动的任务市场。

## 概念

### Contract Net，一段话概括

Smith 1980 年的 Contract Net 协议：一个**管理者**广播**提案请求（cfp）**；**投标者**用包含报价的**propose**消息响应；管理者选择获胜者并向获胜者发送**accept-proposal**、向失败者发送**reject-proposal**。获胜者执行工作。可选消息：**refuse**（投标者拒绝提案）。FIPA 将其编为 `fipa-contract-net` 交互协议。

### 为什么 OG-Narrator 获胜

"Measuring Bargaining Abilities of Language Models"（arXiv:2402.15813）观察到：

- LLM 经常违反谈判规则（以无意义的价格出价，忽略对方的 ZOPA）。
- 它们锚定不佳（接受糟糕的首次出价；以象征性而非战略性金额还价）。
- 仅靠规模不能解决这些问题。更大的模型产生更合理的语言，但有类似的战略错误。

OG-Narrator 的分解：

```
           ┌──────────────────┐        ┌──────────────────┐
  state  → │ 出价生成器       │ price → │  LLM 叙述者      │ → message
           │  （确定性的）    │        │  （撰写人类风格  │
           │                  │        │  的附带说明）    │
           └──────────────────┘        └──────────────────┘
```

出价生成器是一个经典的谈判策略：Rubinstein 讨价还价模型、Zeuthen 策略或简单的以牙还牙式价格策略。LLM 负责叙述。消息包含确定性价格和自然语言框架。

成交率跃升是因为：
- 价格保持在谈判区内。
- 锚点是战略性的，而非情绪化的。
- LLM 做它擅长的事情：写作。

### NegotiationArena 的发现

arXiv:2402.05863 提供了规范的基准测试。主要发现：

- LLM 可以通过采用角色（"我必须在周五之前卖掉这个"）将收益提高约 20%——角色操纵是一种真实的策略。
- 公平/合作的智能体会被对抗性智能体利用；防御需要明确的反制姿态。
- 对称配对的场景中约 40% 收敛到不公平的结果。

这不是"LLM 是糟糕的谈判者"。而是"LLM 谈判得太像人类了，包括那些容易被利用的部分。"

### 思维链隐藏

大规模自主谈判竞赛（arXiv:2503.06416）跨越许多 LLM 策略进行了约 180k 次谈判。获胜者向对手隐藏了他们的推理：

- 如果一个智能体将"我只出到 75 美元；我的保留价是 70 美元"打印到公开可见的草稿中，对手会读取它。
- 获胜者私下计算策略；输出通道只包含出价和最低限度的叙述。

这是对经典博弈论的 2026 年呼应（Aumann 1976 关于理性与信息）：暴露你的私人估值会损害收益。LLM 凭直觉做不到这一点，并愉快地将保留价键入变得对对手可见的推理轨迹中。

工程要点：分离私人草稿上下文和公共消息上下文。不是可选的。

### Bhattacharya 等人 2025——模型排名

在哈佛谈判项目指标（原则性谈判、BATNA 尊重、利益互惠）上：

- **Llama-3** 在达成交易方面最有效（成交率 + 收益）。
- **Claude-3** 是最激进的谈判者（高锚点、晚让步）。
- **GPT-4** 最公平（配对间收益差异最小）。

这是 2025 年的快照。重点不是哪个模型在 2026 年 4 月获胜——而是不同的基础模型有持久的谈判风格。异质集成（Lesson 15）将此作为多样性来源。

### 通过 Contract Net + LLM 进行任务分配

Contract Net 在 LLM 多智能体中的现代复用：

1. 管理者智能体将任务分解为单位。
2. 向工作者智能体广播带任务描述的 `cfp`。
3. 每个工作者返回出价：`(price, eta, confidence)`，其中 price 可以是 token、计算单元或美元。
4. 管理者选择获胜者（单个或多个，取决于任务）并分配。
5. 被拒绝的工作者可以自由竞标其他任务。

这可以很好地扩展到超过 100 个工作者，因为协调是广播和响应，而不是同步聊天。用于生产：Microsoft Agent Framework 的编排模式，一些 LangGraph 实现。

### LLM-Stakeholders 交互式谈判

NeurIPS 2024 引入了具有**秘密分数**和**最低接受阈值**的多方可评分游戏。每个利益相关者有私有效用；LLM 必须从消息中推断它们。这是两方讨价还价到 N 方联盟形成的推广。适用于具有异质工作者能力的生产任务市场。

### 叙述 vs 机制规则

在 2024-2026 年所有谈判基准测试中，一致的工程规则是：

> 让 LLM 叙述。不要让 LLM 计算出价。

如果出价需要是一个数字（价格、ETA、数量），从谈判状态确定性地生成它，并让 LLM 产生框架。如果出价需要是一个提案结构（任务分解、角色分配），让 LLM 起草，但在发送前对照模式进行验证和约束检查。

## 构建

`code/main.py` 实现：

- `ContractNetManager`、`ContractNetTask`、`Bid`——管理者 + 投标者，广播 cfp，收集提案，分配。
- `og_narrator_bargain(state, rng)`——OG-Narrator 买方：确定性 Zeuthen 风格向中点让步。
- `seller_response(state, rng)`——确定性卖方还价策略（两种风格的结构性真相）。
- `naive_llm_bargain(state, rng)`——模拟全 LLM 谈判者：高方差选择价格，经常超出 ZOPA。
- 测量：在 1000 次试验中的成交率，每次试验采样新的保留价。

运行：

```
python3 code/main.py
```

预期输出：naive-LLM 成交率约 65-75%；OG-Narrator 成交率约 85-95%；15-25 个百分点的差距是将出价生成与叙述分离的结构性优势。外加一个带有三个投标者和一个任务的 Contract Net 任务市场分配示例。

## 使用

`outputs/skill-bargainer-designer.md` 设计谈判协议：谁生成出价（确定性还是 LLM）、谁叙述、私人草稿如何与公共消息分离、以及如何监控成交率。

## 交付

生产谈判清单：

- **分离草稿。** 私人状态永远不会到达对方的上下文。这是不可协商的。
- **确定性出价生成。** 价格、数量、ETA：计算，不要提示。
- **验证所有传入出价**的模式。在协议边界拒绝 ZOPA 之外的出价。
- **限制轮次。** 最多 3-5 轮；在死锁时升级到调解人。
- **持续测量成交率和收益方差。** 下降的成交率是一个症状——通常是提示漂移或对方攻击。
- **记录所有被拒绝的提案**及确定性理由。对于 Contract Net 管理者，失败的投标者需要知道原因。

## 练习

1. 运行 `code/main.py`。确认 OG-Narrator 在成交率上击败 naive-LLM。差距有多大？
2. 实现**基于角色的收益提升**（arXiv:2402.05863）——买方仅在叙述中采用"这周迫切要买"的角色，出价生成器不变。成交率或收益有变化吗？
3. 实现思维链**隐藏**：维护一个不传递给对方的私人草稿字符串。如果你意外泄漏它（通过交换通道模拟）会发生什么？
4. 将 Contract Net 扩展到 N 投标者拍卖，带有底价。当所有出价超过底价时，管理者如何在最低价和最高质量之间决定？你选择哪种分配规则，为什么？
5. 阅读 Bhattacharya 等人 2025 年关于哈佛谈判项目指标的论文。实现两个不同风格（激进 vs 公平）的谈判者。测量对称和非对称配对下的收益方差。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|---|---|---|
| Contract Net | "任务市场" | Smith 1980, FIPA 1996。cfp + propose + accept/reject。规范的任务市场。 |
| ZOPA | "可能协议区" | 买方最高价和卖方最低价之间的重叠。此区间外的出价无法成交。 |
| BATNA | "谈判协议的最佳替代方案" | 如果这笔交易失败，你的后备方案。设定你的保留价。 |
| OG-Narrator | "出价生成器 + 叙述者" | 分解：确定性出价，LLM 叙述。 |
| Zeuthen strategy | "风险最小化让步" | 基于风险限制让步的经典出价生成器。 |
| Rubinstein bargaining | "交替出价均衡" | 无限期折扣讨价还价的博弈论模型。 |
| CoT concealment | "隐藏你的推理" | arXiv:2503.06416 中的获胜者保持私人草稿；公共通道只显示出价。 |
| Persona manipulation | "情绪姿态" | arXiv:2402.05863：约 20% 的收益提升来自绝望/紧迫性角色。 |
