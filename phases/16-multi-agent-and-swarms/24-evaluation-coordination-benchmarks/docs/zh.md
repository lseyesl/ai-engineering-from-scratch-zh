# 评估与协调基准测试

> 五个 2025-2026 年的基准测试覆盖了多智能体评估空间。**MultiAgentBench / MARBLE**（ACL 2025，arXiv:2503.01935）评估星型/链型/树型/图型拓扑，使用里程碑 KPI；**图型最适合研究**，认知规划增加约 3% 的里程碑达成率。**COMMA** 评估多模态非对称信息协调；包括 GPT-4o 在内的最先进模型难以击败随机基线。**MedAgentBoard**（arXiv:2505.12371）涵盖四个医疗任务类别，经常发现多智能体并不优于单 LLM。**AgentArch**（arXiv:2509.10769）对结合工具使用 + 内存 + 编排的企业智能体架构进行基准测试。**SWE-bench Pro**（[arXiv:2509.16941](https://arxiv.org/abs/2509.16941)）有跨 41 个仓库的 1865 个问题，涵盖商业应用、B2B 服务和开发者工具；前沿模型在 Pro 上得分约 23%，而在 Verified 上得分 70%+——这是对数据污染的警示。Claude Opus 4.7（2026 年 4 月）在 Pro 上报告为 **64.3%**，使用显式智能体团队协调（没有 Anthropic 主来源发表——视作初步）；Verdent（智能体脚手架）在 Verified 上达到 **76.1% pass@1**（[Verdent 技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。**AAAI 2026 Bridge Program WMAC**（https://multiagents.org/2026/）是 2026 年的社区焦点。本课程基于 MARBLE 的指标，运行拓扑对指标的扫描，并钉住"仅仅通过 SWE-bench Verified 不是通用性的证据"这一规则。

**类型：** Learn
**语言：** Python（标准库）
**前置知识：** Phase 16 · 15（投票与辩论拓扑），Phase 16 · 23（故障模式）
**时间：** ~75 分钟

## 问题

当一篇论文声称"我们的多智能体系统更好"时，问题是：比什么好，在什么上，如何测量的？2023-2024 年多智能体评估的时代是混乱的——每个人都选择自己的指标、自己的基线、自己的任务集。2025-2026 年的基准测试施加了结构。

没有共享的基准测试，你无法有意义地比较两个多智能体系统。更糟糕的是，没有留出集基准测试，前沿模型可能会产生数据污染。SWE-bench Verified 到 2025 年中在训练语料中部分被污染；前沿得分膨胀；Pro 被设计为无污染的现实检测。

本课程列举五个规范的 2026 年基准测试，命名每个测量什么，并教你怀疑地阅读基准测试主张。

## 概念

### MultiAgentBench（MARBLE）— ACL 2025

arXiv:2503.01935。在研究、编码和规划任务上评估四种协调拓扑（星型、链型、树型、图型）。基于里程碑的 KPI 跟踪部分进展，而非仅最终成功。

测量结果：

- **图型**拓扑最适合研究场景；支持任意到任意批评。
- **链型**最适合分步优化编码。
- **星型**最适合快速事实整合。
- **协调税**在图型拓扑上出现在约 4 个智能体以上。
- **认知规划**跨拓扑增加约 3% 的里程碑达成率。

何时使用：你想在苹果对苹果的基础上比较协调拓扑。MARBLE 仓库（https://github.com/ulab-uiuc/MARBLE）提供评估器。

### COMMA— 多模态非对称信息

涵盖智能体具有不同观察模态且必须在没有完全信息共享的情况下进行协调的任务。报告的结果令人不安：包括 GPT-4o 在内的前沿模型在 COMMA 中难以在智能体间协作上击败**随机基线**。信号是多模态智能体训练不足且评估不足——LLM 处理单模态协调尚可；多模态协调崩溃。

何时使用：你的系统具有多模态或非对称信息协调。COMMA 的空结果是衡量之前不要声称的警告。

### MedAgentBoard— 领域压力测试

arXiv:2505.12371。四个医疗任务类别：诊断、治疗规划、报告生成、患者沟通。比较多智能体 vs 单 LLM vs 传统基于规则的系统。

发现：多智能体在大多数类别上**不**优于单 LLM。多智能体的优势很窄——当子任务清晰可分离（诊断 + 治疗）时任务分解有帮助；当协调开销超过专业化收益时（报告生成），它有害。

何时使用：你的领域有清晰明确的单 LLM 基线。如果 MedAgentBoard 的教训能够推广，许多提出的多智能体系统是过度设计的。

### AgentArch— 企业架构

arXiv:2509.10769。企业环境，结合工具使用、内存和编排。基准测试隔离每个层的贡献：添加工具有多大帮助？添加内存？添加多智能体编排？

何时使用：你在设计企业智能体栈需要证明每一层的合理性。AgentArch 帮助你避免购买你无法衡量价值的功能。

### SWE-bench Pro— 现实检查

arXiv:2509.16941。跨 41 个仓库的 1865 个问题，涵盖商业应用、B2B 服务和开发者工具。设计为**无污染**，使用较晚的训练截止日期。前沿模型在 Pro 上得分约 23%，而在 Verified 上得分 70%+。差距就是污染信号。

2026 年 4 月的得分：
- Claude Opus 4.7 在 Pro 上：**64.3%**（报告使用显式智能体团队协调；没有 Anthropic 主来源发表——视作初步）。
- Verdent（智能体脚手架）在 Verified 上：**76.1% pass@1**（[技术报告](https://www.verdent.ai/blog/swe-bench-verified-technical-report)）。
- 没有智能体脚手架的前沿原始得分在 Pro 上：约 23-35%（[SWE-bench Pro 论文](https://arxiv.org/abs/2509.16941)）。

结论："我们击败了 SWE-bench Verified"不再是能力的证据。Pro 是当前的入门测试。智能体团队脚手架在 Pro 上产生可测量的提升（约 30-40 个百分点的增量），这是 2026 年多智能体协调最有力的实证论据之一。

### AAAI 2026 WMAC

AAAI 2026 Bridge Program——多智能体协调研讨会（https://multiagents.org/2026/）。2026 年多智能体 AI 研究的社区焦点。录用的论文和研讨会论文集是评估新方法的规范场所；在生产决策中优先采用 WMAC 录用的主张而非 arXiv 预印本。

### 怀疑地阅读基准测试主张——2026 清单

当有人声称多智能体结果时：

1. **哪个基准测试，哪个分割？** SWE-bench Verified vs Pro 区别很大。在错误分割上报告的数字毫无价值。
2. **污染检查。** 基准测试是在模型的训练截止日期之后发布的吗？如果不是，谨慎对待。
3. **基线比较。** 对比单 LLM 基线、对比随机、对比先前的多智能体工作。不是"对比同一系统的未调优版本。"
4. **统计显著性。** N 次试验、p 值、置信区间。前沿模型方差很高；单次运行会误导。
5. **任务多样性。** 一个任务还是许多？通用性对生产很重要。
6. **成本披露。** 每个任务的 token、挂钟时间。一个 90% 的解决方案以 20 倍成本是业务决策，而不是能力主张。

### 所有基准测试都不擅长测量的内容

- **长期协调。** 数天的挂钟交互。所有当前基准测试运行时间短。
- **对抗性韧性。** 当一个智能体是恶意的或受损时会发生什么？
- **部署下的漂移。** 基准测试是静态的；生产分布会变化。
- **成本归一化性能。** 大多数基准测试报告原始准确率，而非每美元准确率。

为你真正关心的轴构建你自己的内部基准测试通常是正确的做法。

## 构建

`code/main.py` 是一个非交互式演练：

- 在玩具任务上模拟 3 个多智能体系统。
- 为每个计算 MARBLE 风格的里程碑指标。
- 通过从"训练"集中扣留任务来运行污染检查。
- 显式与随机基线比较。
- 打印基准测试主张评分卡。

运行：

```bash
python3 code/main.py
```

预期输出：系统评分卡，包含原始准确率、里程碑达成率、每任务成本、与随机基线的增量以及污染检查备注。

## 使用

`outputs/skill-benchmark-reader.md` 读取任何多智能体基准测试主张并应用审查清单。输出：等级和注意事项。

## 交付

生产评估纪律：

- **构建反映你实际生产分布的内部基准测试。** 公共基准测试提供信息但不替代。
- **在每次比较中包含随机基线。** 如果你在协调任务上不能以很大优势击败随机，任务可能定义不当。
- **报告成本与准确率并行。** Token 成本和挂钟时间。运维团队两者都需要。
- **每季度重建基准测试。** 生产分布会变化；陈旧的基准测试会误导。
- **避免对已发布基准测试的过拟合。** 如果你的团队特别优化 SWE-bench Pro 的数字，你会在生产上回归。

## 练习

1. 运行 `code/main.py`。识别三个模拟系统中哪个具有最佳的成本每里程碑。它匹配原始准确率最高的系统吗？
2. 阅读 MultiAgentBench（arXiv:2503.01935）。为你自己的任务领域，决定 MARBLE 会推荐四种拓扑中的哪一种。从论文结果中证明。
3. 阅读 SWE-bench Pro 论文。什么特别使它抗污染？同样的技术可以应用到其他你关心的基准测试上吗？
4. 阅读 COMMA 关于多模态协调的发现。设计一个你可以添加到内部基准测试的简单多模态协调任务。什么算作有用的信号？
5. 将基准测试主张清单应用于一篇最近多智能体论文的标题结果。你会给这个主张什么等级？
