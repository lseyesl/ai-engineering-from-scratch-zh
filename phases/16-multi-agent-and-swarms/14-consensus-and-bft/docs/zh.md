# 共识与智能体拜占庭容错

> 经典分布式系统 BFT 遇上随机性 LLM。2025-2026 年出现了三个研究方向：**CP-WBFT**（arXiv:2511.10400）为每个投票附加置信度探针进行加权；**DecentLLMs**（arXiv:2507.14928）无领导者，采用并行工作者提案和几何中位数聚合；**WBFT**（arXiv:2505.05103）结合加权投票与层次结构聚类，将节点分为核心和边缘。"AI Agents 能达成一致吗？"（arXiv:2603.01213）的诚实实证结果是，即使在今天，标量协议也是脆弱的——一个欺骗性智能体就能破坏 Mixture-of-Agents。BFT 是必要但不充分的。本课程构建一个最小 BFT 协议，注入三种智能体特定攻击（拜占庭谎言、谄媚从众、相关错误单一文化），并测量每种共识变体如何应对。

**类型：** Learn + Build
**语言：** Python（标准库）
**前置知识：** Phase 16 · 07（心智社会与辩论），Phase 16 · 13（共享内存）
**时间：** ~75 分钟

## 问题

你有 N 个 LLM 智能体各自产生一个答案。它们意见不一致。多数投票选出了错误的一个，因为两个智能体是相关的（相同的基础模型、相同的训练数据、相同的失败模式）。第三个智能体恰好以一种新颖的方式犯错——所以多数是一个虚假多数。

现在加入一个欺骗性智能体：它故意说谎。或者一个谄媚智能体：它同意最后发言的人。在经典 BFT 中，假设拜占庭节点占比 `f < n/3` 且行为任意。2026 年的现实是，LLM 节点即使在诚实时也是随机性的，跨模型相关，且受彼此输出影响。你不能将它们视为独立的伯努利投票者。

经典 BFT（PBFT，1999）不是错的——它是不完整的。它处理任意的比特翻转。但它不处理"三个诚实的智能体因为共享训练数据而共享幻觉。"本课程从 PBFT 的基础出发，叠加了三种 2025-2026 的适应性方案。

## 概念

### 经典 BFT 给了你什么

实用拜占庭容错（Castro & Liskov, OSDI 1999）容忍 `f < n/3` 的拜占庭节点。协议有三个阶段（预准备、准备、提交）和两个基元（签名消息、仲裁证书）。在 `n >= 3f + 1` 个诚实或恶意节点中就单个值达成一致。

这些保证很强，但假设：

1. **独立故障。** 拜占庭节点不协调。
2. **诚实节点是真的诚实。** 诚实输出的正确性不是问题；协议只对齐分歧。
3. **问题有真实答案。** 对错误事实的共识仍然是共识。

LLM 智能体违反了所有三条。运行相同基础模型的两个智能体共享故障。一个"诚实的"LLM 仍然会幻觉。而在模棱两可的问题上，"真相"就是智能体决定的东西——没有外部预言机。

### 三种 LLM 特定攻击

**拜占庭谎言。** 一个智能体故意输出错误答案。经典 BFT 在 `f < n/3` 时能处理。

**谄媚从众。** 一个智能体在投票前阅读其他人的答案，并与最后发言的人保持一致。不是恶意的，但与最响亮的声音相关。经典 BFT 不能阻止这一点，因为智能体通过了所有签名检查。

**相关错误单一文化。** 三个智能体共享一个基础模型。它们对同一个错误答案产生幻觉。多数是错误的。经典 BFT 无济于事，因为所有三个"诚实地"一致。

### 2025-2026 的回应

**CP-WBFT**（arXiv:2511.10400）——置信度探针加权 BFT。每个投票者在其答案上附加一个置信度探针（自报概率或单独的校准模型预测）。投票权重随置信度缩放。在完全图上报告了 +85.71% 的 BFT 改进。缓解：谄媚从众（从众智能体在其主动立场上往往置信度较低）。

**DecentLLMs**（arXiv:2507.14928）——无领导者。工作者智能体并行提出方案，评估者智能体为方案打分，最终答案是打分位置的几何中位数。在 `f < n/2` 时鲁棒。缓解：拜占庭谎言和相关错误（几何中位数对异常值鲁棒，向密集簇而非模型偏置均值靠拢）。

**WBFT**（arXiv:2505.05103）——带层次结构聚类的加权 BFT。投票权重由响应质量加上从历史中学习的信任分数分配。将智能体聚类为核心和边缘；核心智能体必须首先达成共识，边缘智能体跟随。缓解：可扩展性（核心共识小且快），部分缓解单一文化（核心可选择为多样性）。

### 实证：AI Agents 能达成一致吗？（arXiv:2603.01213）

论文测量了多个前沿模型间的标量协议。发现令人不安：

- 即使没有对手，LLM 智能体在许多基准测试上对标量问题的分歧率超过 30%。
- 一个采用欺骗角色的智能体可以将 Mixture-of-Agents 共识拉离诚实基线 40 多个百分点。
- 分歧率与模型多样性相关——异质集成比同质集成分歧更多（好处：不相关错误），但漂移也更慢（坏处：达成协议的时间更长）。

结论：BFT 给了你对齐输出的机制，但它不告诉你对齐后的输出是否正确。结合验证（Phase 16 · 08 角色专业化）、多样性（Phase 16 · 15 辩论变体）和评估者智能体（Phase 16 · 24 基准测试）。

### 核心协议，简化版

一个针对 LLM 智能体的最小 BFT 轮次：

```
1. 任务到达；每个智能体 i 产生答案 a_i
2. 每个智能体附加置信度探针 c_i in [0, 1]
3. 聚合器从所有 n 个智能体收集 (a_i, c_i)
4. 聚合器按语义簇分组（等价答案）
5. 聚合器计算每个簇 C 的权重：
     w(C) = sum_{i in C} c_i
6. 胜者 = 具有最大权重的簇，如果 max > threshold * sum(c_i)
   否则：重试或升级
7. 少数簇以来源记录方式记录用于事后审计
```

语义聚类步骤是 LLM 特定的变体。两个答案"研究报告了 4.2%"和"4.2% 的改进"是同一个簇。朴素的字符串相等性检查会漏掉这一点。在生产中，使用廉价的嵌入模型或显式规范化。

### 阈值调优

`threshold` 参数决定何时接受和何时重试。太低：你接受弱多数。太高：你永远不接受任何东西。经验范围：`n=5-7` 个智能体时为 0.5-0.67，n 越小阈值越高。低于阈值时，升级到人类或不同的智能体集成。

### 共识无济于事的场景

- **模棱两可的问题。** 如果问题没有真实答案，共识就是一种意见。如实说明。
- **复合问题。** "编写代码并解释"——两个答案。对每个独立投票。
- **对抗性多轮。** 如果智能体可以观察之前的轮次并模仿（Du 2023 辩论），它们不论真相如何都会开始彼此同意。限制轮次（通常 2-3 轮）。

## 构建

`code/main.py` 实现：

- `AgentVoter`——带有（答案，置信度）的脚本策略。
- `MajorityVote`——经典复数投票。
- `CPWBFT`——带有语义聚类的置信度加权投票。
- `DecentLLMs`——对评分提案的几何中位数聚合。
- `Scenario`——在三种攻击模式下运行每个聚合器。

实现的攻击模式：

1. `byzantine`：一个智能体以高置信度说谎。
2. `sycophancy`：一个智能体复制它看到的第一个答案，带有匹配的置信度。
3. `monoculture`：三个智能体以中等置信度共享一个错误答案（相关错误）。

运行：

```
python3 code/main.py
```

预期输出：一个（攻击，聚合器）→ 最终答案的表格，正确答案高亮显示。复数投票在单一文化场景失败。CPWBFT 的置信度加权缓解谄媚从众。DecentLLMs 的几何中位数在单一文化少于一半人口时向诚实簇靠拢。

`outputs/skill-consensus-designer.md` 为多智能体集成设计共识协议：聚类方法、加权、阈值、以及低于阈值轮次的升级策略。

## 交付

在发布任何共识机制之前：

- **至少用以上三种模式进行攻击测试。** 你的协议应该可预测地失败，而不是静默失败。
- **记录每个少数簇及其来源。** 少数簇是你相关错误的早期预警系统。
- **强制执行有界轮次。** 不要"一直辩论直到达成一致"——那会奖励谄媚行为。
- **将协议与正确性分开。** 共识输出传递给验证器；验证器独立于集成。
- **监控协议率。** 急剧上升意味着从众偏差；急剧下降意味着模型漂移。

## 练习

1. 运行 `code/main.py`。确认复数投票在单一文化攻击中失败，而 CPWBFT 在单一文化置信度低于 0.7 时部分缓解。
2. 添加第四种攻击模式：**静默弃权**——一个智能体拒绝回答（"我不知道"）。每个聚合器应如何处理弃权？实现你的选择。
3. 将语义聚类从字符串规范化切换为嵌入相似度（使用任何开源嵌入模型）。谄媚从众攻击会发生什么？
4. 阅读 CP-WBFT（arXiv:2511.10400）。实现置信度探针校准步骤（一个单独的校准模型检查每个智能体的自报置信度）。在单一文化场景下测量准确率提升。
5. 阅读"Can AI Agents Agree?"（arXiv:2603.01213）。复现一个简化的标量协议实验：三个智能体、一个标量问题、欺骗角色提示。CPWBFT 或 DecentLLMs 能抓住它吗？

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|---|---|---|
| BFT | "拜占庭容错" | Castro-Liskov 1999 协议，用于在存在 `f < n/3` 任意故障时达成共识。 |
| Byzantine | "任何不良行为" | 一个可以撒谎、丢消息、静默失败的节点——除了安全崩溃外任何行为。 |
| Confidence probe | "你有多确定？" | 自报或校准器预测的概率，附加到投票上。 |
| Semantic clustering | "相同答案，不同措辞" | 计票前对等价答案进行分组。 |
| Geometric median | "鲁棒中心" | 最小化到样本点的距离和的点。对异常值鲁棒，不同于均值。 |
| Monoculture | "相同模型，相同失败" | 智能体共享训练数据或基础模型时的相关错误。 |
| Sycophantic conformity | "赞同响亮的声音" | 智能体的投票偏向于第一个/最响亮的发言者。 |
| Core/Edge | "层次化 BFT" | WBFT 拆分：小型核心共识先行，边缘节点跟随。限制延迟。 |
