# 智能体经济、Token 激励与声誉

> 长期自主智能体（METR 的 1 小时到 8 小时工作曲线）需要经济主体性。新兴的**五层堆栈**是：**DePIN**（物理计算）→ **身份**（W3C DID + 声誉资本）→ **认知**（RAG + MCP）→ **结算**（账户抽象）→ **治理**（智能体 DAO）。生产级智能体激励网络包括 **Bittensor**（TAO 子网奖励特定任务模型）、**Fetch.ai / ASI Alliance**（ASI-1 Mini LLM + FET token）和 **Gonka**（基于 Transformer 的 PoW，将计算重新分配到生产性 AI 任务）。学术工作：AAMAS 2025 的去中心化 LaMAS 使用 **Shapley 值信用归因**来公平奖励贡献智能体；Google Research 的"大型语言模型的机制设计"提出在单调聚合下使用**第二价格 token 拍卖**。本课程构建一个最小智能体市场，将 Shapley 值信用归因应用于多智能体管道，并运行一个第二价格 token 拍卖，使博弈论机制具体落地。

**类型：** Learn
**语言：** Python（标准库）
**前置知识：** Phase 16 · 16（谈判与讨价还价），Phase 16 · 09（并行群体网络）

## 问题

多智能体系统在智能体共同产生价值但需要单独奖励时会变得复杂。经典机制——平均分配、最后贡献者全拿——不公平或可被利用。通过 Shapley 值进行基于联盟的奖励通过构造是公平的但计算昂贵。2025-2026 年的文献推动了有用的近似：Shapley 采样、单调聚合拍卖和链上声誉（从已确认的贡献中积累）。

除了信用归因，该领域已经转向实际的经济智能体：Bittensor TAO 奖励用于微调子网特定模型的计算，Fetch.ai/ASI 奖励使用 FET token 的 ASI-1 Mini LLM 使用，Gonka 将 Transformer 工作量证明重新分配到生产性 AI 任务。自主交易的智能体今天已经存在；问题是如何对齐激励。

本课程将智能体经济视为一个特定的问题族——信用归因、机制设计和声誉——并用最少的数学构建每一个，使这些想法牢固定位。

## 概念

### 智能体经济五层堆栈

1. **DePIN（物理计算）。** 出租 GPU、存储、带宽的去中心化基础设施。Bittensor 子网、Render Network、Akash。不是智能体特定的；智能体使用它。
2. **身份。** W3C 去中心化标识符（DID）给每个智能体独立于任何平台的持久 ID。声誉累积到 DID。智能体网络协议（ANP）使用 DID 作为发现层。
3. **认知。** 智能体的推理循环：LLM + RAG + MCP。这是其他阶段构建的内容。
4. **结算。** 账户抽象（ERC-4337）让智能体从自己的余额支付 gas 而无需持有 ETH。智能体可以支付服务、彼此或计算。
5. **治理。** 智能体 DAO：人类*和*智能体对协议变更投票的治理结构，投票权与声誉挂钩。

并非每个生产系统都使用所有五层。Bittensor 使用 1、2、部分 3、部分 4、不使用 5。OpenAI 智能体除了 3 外什么都不用。这个栈是一个参考图，不是要求。

### Bittensor、Fetch.ai、Gonka——实际运行的系统

**Bittensor（TAO）。** 子网是专门化的任务（语言建模、图像生成、预测）。矿工提交模型输出。验证者对它们排名；权益加权评分分配 TAO 奖励。每个子网有自己的评估。经济教训：为任务特定的输出质量付费，而不是使用的计算量。

**Fetch.ai / ASI Alliance。** ASI-1 Mini LLM 在 Fetch.ai 网络上运行；用户支付 FET token 进行推理。智能体作为对等体的叙事在这里更强：Fetch 上的一个智能体可以调用另一个完成任务并用 FET 支付。

**Gonka。** Transformer 工作量证明："工作"是 transformer 的前向传播。矿工通过运行具有已知正确输出（来自训练数据）的推理任务来赚取。基于资源的生产性 PoW，而非基于哈希的 PoW。

截至 2026 年 4 月，这三种都是生产级的。收益分配不同。Bittensor 奖励相对于子网验证者的质量；Fetch 奖励由支付用户衡量的效用；Gonka 奖励可验证的推理工作。

### Shapley 值信用归因

三个智能体合作完成一项任务。输出得分为 0.8。谁贡献了多少？

Shapley 值：满足四个公理（效率、对称性、线性、零元素）的唯一信用分配。对于智能体 `i`：

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

其中 `S_i_O` 是在排序 `O` 中 `i` 之前的智能体集合。在实践中：枚举所有排列，记录每个智能体在每个排列中的边际贡献，取平均。

N=3 个智能体有 6 个排列。N=10 时有 360 万——所以在实践中你采样排序而非枚举。

### 聚合的第二价格拍卖

Google Research（"大型语言模型的机制设计"）提出用于聚合 LLM 输出的第二价格 token 拍卖。设置：N 个智能体各提出一个完成方案；每个都有被选择的私人价值。拍卖者选择最高价值的提案并支付*第二高*的价值。在单调聚合下（价值取决于选择了哪个提案，而不是有多少投标），这是真实的——智能体投出真实价值。

为什么这对 LLM 系统很重要：你可以将完成任务外包给具有不同定价的多个智能体；拍卖选择最好的 + 公平支付，且智能体没有动机误报。

### 声誉资本

一个 DID 绑定的声誉分数从已确认的贡献中积累。一个简单的更新规则：

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

衰减因子 `alpha` 接近 1。声誉：

- 读取成本低（用于路由决策："将困难任务发送给高声誉智能体"）。
- 伪造成本高（随时间累积，绑定到 DID）。
- 可以罚没：未通过验证的贡献会扣分。

### AAMAS 2025 去中心化 LaMAS

LaMAS 提案（AAMAS 2025）结合了：DID 身份、Shapley 值信用归因和简单的拍卖机制。关键主张：去中心化信用归因步骤使系统可审计且免疫于单点操纵。

### 经济学在哪些方面失灵

- **价格预言机操纵。** 如果信用函数可以被欺骗，智能体会去欺骗。每个机制都需要对抗性测试。
- **女巫攻击。** 一个操作者启动 N 个假智能体来夸大自己的贡献。DID 减缓但不能阻止这一点；声誉的伪造成本是缓解措施。
- **验证成本。** 信用归因只与验证者一样公平。如果验证便宜（小型 LLM），它可被欺骗；如果昂贵（人类小组），系统无法扩展。
- **监管不确定性。** 智能体经济与金融监管相交。截至 2026 年，Bittensor、Fetch 和 Gonka 在某些司法管辖区都在灰色地带运营。

### 智能体经济何时有意义

- **具有异质操作者的开放网络。** 没有单一团队控制所有智能体。
- **可验证的输出。** 没有验证，信用归因就是猜测。
- **长期工作流。** 一次性任务不会从声誉积累中受益。
- **Token 化支付在你的司法管辖区是合法的。**

在封闭的企业系统中，经济让位于更简单的分配（管理者分配工作，指标是内部的）。经济学文献主要适用于开放网络。

## 交付

在 2026 年运行智能体经济：

- **从声誉开始，而不是 token。** 声誉实现成本低且单独有价值；token 增加了法律和经济复杂性。
- **在奖励之前先验证。** 永远不要在独立验证步骤之前分发信用。自我报告的质量会引发女巫游戏。
- **Shapley 采样，而非精确 Shapley。** 采样 100-1000 个排序；精确枚举无法扩展。
- **为衰减因子和声誉设置上下限。** 无界衰减会清除合法贡献者；太慢的衰减奖励陈旧的高声誉智能体。
- **对抗性地审计机制。** 在开放网络之前运行红队场景。每个机制都有博弈论；你希望找到漏洞，而非攻击者。

## 关键术语

| 术语 | 通俗说法 | 实际含义 |
|---|---|---|
| DePIN | "去中心化物理基础设施" | Token 激励的计算/存储/带宽。Bittensor、Akash、Render。 |
| DID | "去中心化标识符" | W3C 可移植 ID 规范。智能体声誉绑定到 DID，而非平台。 |
| ERC-4337 | "账户抽象" | 可以赞助 gas 的合约账户，使能智能体支付。 |
| Shapley value | "公平信用归因" | 满足效率、对称性、线性、零元素的唯一分配。 |
| Second-price auction | "Vickrey 拍卖" | 真实机制：获胜者支付第二高出价。与单调聚合兼容。 |
| Reputation capital | "累积质量分数" | 已确认贡献的 DID 绑定分数；随时间衰减。 |
| Agentic DAO | "智能体 + 人类共同治理" | 智能体投票者作为一等公民的 DAO，投票权与声誉挂钩。 |
| TAO / FET / GPU credits | "Token 面额" | Bittensor TAO、Fetch.ai FET、各种 DePIN token。 |
