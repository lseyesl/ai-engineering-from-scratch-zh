# InternVL3：原生多模态预训练

> 在 InternVL3 之前，每个开源 VLM 都遵循相同的三步配方：拿一个在数万亿文本 token 上训练的文本 LLM，在其上附加一个视觉编码器，然后微调连接处。这有效，但存在对齐债务——文本 LLM 将其完整的预训练预算花在了纯文本上，并不原生理解视觉 token。当你事后添加视觉时，LLM 必须重新学习如何在不忘记文本的情况下将视觉输入与其文本推理关联起来。InternVL3（Zhu 等人，2025 年 4 月）拒绝了事后添加的方法：一次预训练运行，从第一步开始就文本和多模态交错进行。结果在 78B 开源参数下，在 MMMU-Pro 上匹配了 Gemini 2.5 Pro。本课程阅读支持原生预训练的论据以及做出这一改变后会发生什么。

**类型：** 学习
**语言：** Python（标准库，训练语料混合器）
**前置要求：** Phase 12 · 05，Phase 12 · 07（配方）
**时间：** ~120 分钟

## 学习目标

- 解释为什么事后 VLM 训练会累积对齐债务，引用三个可测量的症状（灾难性遗忘、答案漂移、视觉文本不一致）。
- 描述 InternVL3 的原生预训练语料混合以及为什么文本：交错：描述的比例很重要。
- 比较 V2PE（可变视觉位置编码）与 Qwen2-VL 的 M-RoPE。
- 说出视觉分辨率路由器（ViR）和解耦视觉语言部署（DvD）这些部署优化方法。

## 问题

事后 VLM 训练是默认方式。LLaVA、BLIP-2、Qwen-VL、Idefics——都拿一个已经预训练好的 LLM（Llama、Vicuna、Qwen、Mistral）并添加视觉。训练阶段通常是这样的：

1. 冻结 LLM + 冻结视觉编码器 + 可训练投影器，在描述对上训练以对齐嵌入。
2. 解冻 LLM，在指令数据上训练（LLaVA-Instruct、ShareGPT4V）。
3. 可选的任务特定微调。

对齐债务的三个症状显现出来：

- 灾难性遗忘。事后 VLM 忘记了纯文本技能。GSM8K 分数下降 5-10 个点。Hellaswag 分数下降。纯文本代理出现退化。
- 答案漂移。相同视觉问题的不同措辞得到不同的答案。视觉编码器与 LLM 的连接比 LLM 自身的 token 连接更弱。
- 视觉文本不一致。VLM 可以正确描述图像，然后回答一个与自身描述相矛盾的问题。视觉 token 不像文本那样参与 LLM 的内部一致性检查。

这些症状有充分的文献记录。MM1.5 第 4 节量化了它们。LLaVA-OneVision 的消融实验暗示了它们。原生预训练是答案。

## 概念

### 原生多模态预训练

InternVL3 从第一步开始就在一个原生多模态的语料上从头训练。混合比是：

- 40% 纯文本数据（FineWeb、Proof-Pile-2 等）
- 35% 交错图文数据（OBELICS、MMC4 风格）
- 20% 配对图文描述数据
- 5% 视频文本数据

视觉 token、文本 token 和跨模态交互从第一个梯度步骤开始就参与相同的损失。没有对齐预训练，没有投影器冻结阶段，没有需要恢复的灾难性遗忘。

基础模型的训练是单一阶段。随后进行指令调优，但基础模型已经理解视觉 token 作为一等公民。

### V2PE（可变视觉位置编码）

Qwen2-VL 使用 M-RoPE 和固定的轴分配。InternVL3 引入了 V2PE：位置编码随模态类型（文本、图像、视频）变化，使用可学习的缩放。在实践中：

- 文本 token 获得 1D 位置（文本索引）。
- 图像 patch 获得 2D 位置（行，列）。
- 视频帧获得 3D 位置（时间，行，列）。

三者共享相同的 RoPE 频率基础，但每个频带的隐藏维度分配是一个学习到的参数，而不是固定的分割。在预训练期间可以自由权衡时间 vs 空间频率分辨率。

V2PE 的消融声称：在相同计算量下，在视频基准上比 M-RoPE 高 1-2 个点。不是革命性的，但更干净。

### 视觉分辨率路由器（ViR）

部署优化。并非所有图像都需要全分辨率编码。一张只有一个低细节对象的照片在以 1280px 原生分辨率编码时会浪费 token。ViR 是一个小型分类器，在编码之前预测回答问题所需的最小分辨率。

路由有三个层级：低分辨率（256 token）、中分辨率（576）、高分辨率（2048+）。对于生产流量中 60% 的查询，低或中分辨率就足够了。净效果：相同质量下吞吐量提高 2-3 倍。

### 解耦视觉语言部署（DvD）

当你服务一个大型 VLM 时，视觉编码器每张图像运行一次，但 LLM 为每个输出 token 自回归地运行。这两个组件有不同的瓶颈（视觉 = 卷积 + 注意力的 GPU 内存带宽；LLM = KV 缓存）。DvD 将它们拆分到不同的 GPU 上，之间进行流式传输。

对于一个 8B + 400M 编码器模型，DvD 的每节点吞吐量大致是共置的两倍。

### 单阶段 vs 多阶段质量

InternVL3 的主要基准声明：在 78B 参数下，匹配 Gemini 2.5 Pro 的 MMMU-Pro。在 38B 下，匹配 GPT-4o。在 8B 下，领先开源 8B 排行榜。全部基于单阶段预训练 + 指令调优配方。

对齐债务假说是可测量的：InternVL3-8B 每单位视觉基准增益丢失的文本基准点数（MMLU、GSM8K）比 Qwen2.5-VL-7B 更少。模型更通用，因为训练是一体的，而不是两部分。

### InternVL3.5 和 InternVL-U

InternVL3.5（2025 年 8 月）扩展了配方。相同的原生预训练方法，更多的数据，更多的参数。MMMU 改进是增量式的。

InternVL-U（2026 年）增加了统一生成——基于相同骨干网的 MMDiT 头输出图像。"U"代表"理解 + 生成"，追赶 Transfusion 风格统一模型（课程 12.13）。相同的原生预训练骨干网支持理解和生成头。

### 原生预训练的权衡

原生预训练并非免费：

- 计算。从头训练一个新的 VLM 的成本与训练文本 LLM 相同——数百万 GPU 小时。事后适配重用已有的 LLM 权重，节省了大部分成本。
- 数据。大规模的交错图文语料很稀缺。OBELICS 是 1.41 亿文档；MMC4 是 5.71 亿。纯文本本身以 15T token 计。多模态预训练数据的稀缺是一个硬约束。
- 基础 LLM 复用。原生预训练放弃了稍后换入新 LLM 的选项。事后允许你通过仅重新训练适配器来将 Llama-3.1 换成 Llama-4。

InternVL3 下的赌注是：对齐债务比复用损失更严重。基准测试支持这一说法。生产成本阻止了未来的实验室廉价复制。事后 VLM 将继续存在，因为它们对大多数项目来说仍然更便宜。

## 使用它

`code/main.py` 是一个训练语料混合器和 ViR 路由器模拟器。它：

- 接受目标语料混合比例（%文本、%交错、%描述、%视频）并计算每个模态的预期步数。
- 模拟在批查询上的 ViR 路由（分布：50% 低细节、30% 中等、20% 高细节）并报告平均 token 数量。
- 报告给定编码器 vs LLM FLOPs 的 DvD 吞吐量估算。
- 并排打印事后 vs 原生预训练在参数、计算、数据和预期对齐债务症状方面的比较。

## 交付物

本课程产生 `outputs/skill-native-vs-posthoc-auditor.md`。给定一个提议的 VLM 训练计划，它审计是采用原生还是事后方式，标记对齐债务风险，并推荐语料混合比例。当你确定新的开源 VLM 项目规模并需要选择训练策略时使用。

## 练习

1. 估计 InternVL3-8B（原生预训练）和 LLaVA-OneVision-7B（事后）之间的计算差异。GPU 小时的比例大约是多少？什么解释了这种差距？

2. InternVL3 报告 40% 文本 / 35% 交错 / 20% 描述 / 5% 视频。如果你的目标任务侧重于视频，提出一个新的比例并论证为什么基础模型仍然需要大量的文本和描述数据。

3. 阅读 MM1.5 第 4 节关于遗忘的内容。说出事后训练显示出最大退化的确切基准名称。退化损失了多少？

4. ViR 将 60% 的流量路由到低分辨率编码。什么样的查询会被误路由（需要高分辨率时被发送到低分辨率）？提出三种路由器故障模式。

5. DvD 将视觉和 LLM 拆分到不同的 GPU 上。在什么流量模式下 DvD 会损害吞吐量而不是帮助？

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 原生多模态预训练 | "一起从头开始" | 文本 + 图像 + 视频 token 从第 1 步就参与损失，而不是事后添加 |
| 对齐债务 | "事后惩罚" | 文本技能和答案一致性方面可测量的退化，源于将视觉附加到冻结 LLM 上 |
| V2PE | "可变视觉位置编码" | 每模态可学习的位置编码分配；InternVL3 对 M-RoPE 的继承 |
| ViR | "分辨率路由器" | 编码前为每个查询选择所需最小分辨率的小型分类器，节省推理 token |
| DvD | "解耦部署" | 视觉编码器在一个 GPU 上，LLM 在另一个 GPU 上，流式传输交接；使大型 VLM 吞吐量翻倍 |
| InternVL-U | "统一理解 + 生成" | 2026 年的后续，为原生预训练骨干网添加图像生成头 |
| 交错语料库 | "OBELICS / MMC4" | 文本和图像按自然阅读顺序排列的文档；原生预训练的原材料 |
