# 具身 VLA：OpenVLA、π₀ 和 GR00T

> 视觉-语言-动作模型（VLA）将 VLM 与动作预测相结合。在 VLA 中，LLM 生成的不再是文本 token，而是物理动作 token（机器人关节位置、末端执行器位置、导航航路点）。OpenVLA（2024 年 6 月）是第一个在 Open X-Embodiment 数据上训练、遵循机器人语言指令的开源 VLA。π₀（Physical Intelligence，2024 年 10 月）将动作预测帧率为流匹配（扩散）而非离散 token。GR00T 是 NVIDIA 的 VLA 平台。对于学习来说，机器人领域更像是这些模型的直接应用——而不是全新的架构创新——因为多模态基础模型的内核（视觉编码器 → 连接器 → LLM → 输出投影）保持不变，只是输出从文本 token 映射为机器人动作。本课程构建一个玩具 VLA：给定一个"视觉场景"（2D 网格）和一个"语言指令"（文本），预测"动作"（目标位置）。

**类型：** 构建
**语言：** Python（标准库，玩具网格世界中的 VLA）
**前置要求：** Phase 12 · 05（LLaVA）
**时间：** ~180 分钟

## 学习目标

- 解释 VLA 模型如何将相同的视觉编码器 + LLM 公式从文本输出映射到动作输出。
- 描述 OpenVLA 中用于动作预测的联合训练损失（文本 NTP + 动作 MSE）。
- 比较 π₀ 的连续动作空间（去噪 MSE）与 OpenVLA 的离散动作 token 化。
- 根据 Robotics Benchmarking 中的标准，设计并评估在模拟网格世界中的动作预测准确性。

## 问题

VLM 擅长识别、描述和回答有关图像的问题。但它如何将视觉理解转化为机器人执行器动作？这是具身 AI（Embodied AI）的目标：创建一个能"看、听、说并对世界采取行动"的模型。

人类可以通过视觉定位来执行这一过程——当你说"拿起杯子"时，你的眼睛定位杯子，估算距离，然后发送信号给手臂。VLA 将这一过程外部化：VLM 对场景进行推理，LLM 决定动作，动作投影器输出动作 token。

但文本 token ≠ 动作 token。文本是离散的（单词以分隔符结尾）高维的（50000 个 token）。动作是连续的（关节位置是浮点数，通常是 6-14 维）且要求精确（偏差 0.1 弧度 = 末端执行器偏差 5 厘米）。一个用于文本的 LLM head 是不适用的。

## 概念

### 动作 token 化

OpenVLA 使用简单的 bin 化：将每个动作维度（关节位置）划分为 256 个离散 bin。然后 LLM 从 256 个动作 token 的词汇表中预测下一个动作 token。

举例：一个 7 自由度的机器人手臂有 7 个关节。OpenVLA 将 7 个关节中的每一个映射到 256 个 bin 中。因此，空间大小为 7 * 256 = 1792 个动作 token，以及一个特殊的"停止动作"token。

它的推理过程如下：给定图像和"左移 10 厘米"的提示，LLM 生成一个 token，该 token 对应于关节修正 bin，从而实现该动作。

### π₀（Physical Intelligence，2024 年 10 月）

π₀ 处理了行动问题。与其将动作离散为 bin，π₀ 将动作视为应通过扩散产生的连续噪声：

- 动作是 7 维浮点数。
- 训练：模型从图像 + 语言输入中预测噪声（MSE 损失）。
- 推理：网络从纯噪声开始，迭代去噪，输出 7 维浮点数。
- 这最接近 Transfusion（课程 12.13）中用于图像的连续扩散思想。

π₀ 是 Physical Intelligence 的人形机器人基础模型，在多样化的机器人数据上进行训练。它的核心口号是："简单的去噪比复杂的 256-bin 划分更好。"

### 联合训练损失

VLA 的训练需要同时优化两个损失：
+ 文本 NTP（标准 LLM 损失）：用于文本理解（描述、QA）。模型在其文本数据集上训练时仍应学习语言。
+ 动作 MSE（robot 损失）或动作 CE（离散）：用于动作。取决于动作是连续的还是离散的。

一个常见的技巧是对两个损失使用相似的大小：机器人动作损失的尺度大致与文本 NTP 损失的自然大小相匹配。在连续动作空间中，动作 MSE 通常比 NTP 大 10 倍，因此需要额外的加权项。

### GR00T（NVIDIA，2025 年 3 月）

GR00T 是 NVIDIA 的机器人基础模型。它不像 OpenVLA 或 π₀ 那样输出低级动作，而是输出高级任务规划（"走到桌子旁，拿起杯子，把它带到水槽边"）。GR00T 基本上是 VLM，但与规划器相连。


执行层面的实际架构：GR00T 使用 VLM 作为"大脑"，而一个低级控制器（通常是一个更小的扩散策略）执行每个步骤的动作。

### 评估

在物理机器人上进行真实评估是不可行的，因此 VLA 研究大多依靠模拟基准：

- **RLBench**：模拟桌面操作。奖励是完成任务的成功率。
- **CALVIN**：长视界桌面操作。通过在 5 个连续任务上的成功率来评估。
- **BridgeData v2**：大规模机器人数据集。在保留的任务上进行评估。
- **RT-1 / RT-2**：Google 的机器人基准测试。在一台真正的机器人上评估（这很昂贵）。

## 使用它

`code/main.py` 构建了一个在玩具网格世界（2D 网格上的 5x5 箭头）上的 VLA：

- 视觉编码器：将 5x5 网格编码为 25 个 token。
- 文本指令：短语，如"up 2"、"down 1 left 2"。
- 动作输出：模型输出目标网格位置（x，y）。
- 损失：文本 NTP + 动作 MSE（像 π₀ 一样）。

模型在 100 个合成指令-动作对上过拟合，并可视化输出。

## 交付物

本课程产出 `outputs/skill-vla-embodied-designer.md`。给定一个机器人操作问题（自由度数量、任务类型、精度要求），推荐动作表示（离散 bin 与连续扩散）、训练损失（NTP + MSE 与 NTP + CE）以及模型架构（VLM + 动作头与 VLM + 扩散）。

## 练习

1. π₀ 在 7-DOF 动作空间上使用去噪 MSE。针对 16-DOF 灵巧手，连续动作空间是否仍然最优？何时离散动作空间收敛更快？

2. 给定 RLBench 的 "拿起杯子" 子任务。设计一个奖励函数，其输出为成功/失败。你的 VLA 如何通过这种奖励进行改进？

3. 在 OpenVLA 中，动作被离散化为 256 个 bin。如果你的机器人关节具有 ±2 弧度的精度且需要 0.01 弧度的分辨率，256 个 bin 够用吗？证明你的理由。

4. 比较 GR00T 的高级规划方法与 π₀ 的低级动作方法。提出一个需要两者的场景。

5. 在仿真到现实的迁移中，仿真训练的动作 MSE 较低，但在真实硬件上却失败了。输入差异有哪些可能的来源？（提示：光照、物理特性、标定。）

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 视觉-语言-动作模型 | "VLA" | 预测机器人动作的 VLM，该动作以文本 token 或扩散噪声的形式输出 |
| 动作离散化 | "256 个 bin" | 将连续关节位置划分为 256 个区间，使其成为 LLM 可预测的分类问题 |
| 动作去噪 | "π₀ 风格" | 连续动作空间（扩散）直接从噪声恢复动作 |
| 联合训练损失 | "NTP + MSE" | 同时优化文本理解（LLM）和动作（机器人）的复合损失 |
| 低级动作 | "原始控制" | 直接关节位置命令（例如 7-DOF 臂位置） |
| 高级规划 | "任务分解" | 分解为子步骤（"走到桌子边"、"拿杯子"），交由低级策略执行 |
