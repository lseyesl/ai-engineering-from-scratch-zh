# 从 CLIP 到 BLIP-2——Q-Former 作为模态桥梁

> CLIP 可以对齐图像和文本，但不能生成描述、回答问题或进行对话。BLIP-2（Salesforce，2023 年）用一个小型可训练桥梁解决了这个问题：32 个可学习的查询向量通过交叉注意力关注冻结 ViT 的特征，然后直接插入冻结 LLM 的输入流。188M 参数的桥梁连接了一个 11B 的 LLM 和一个 ViT-g/14。到 2026 年为止，每个基于适配器的 VLM——MiniGPT-4、InstructBLIP、LLaVA 的同类模型——都是其后代。本课程阅读 Q-Former 的架构，解释其两阶段训练，并构建一个将视觉 token 馈入冻结文本解码器的玩具版本。

**类型：** 构建
**语言：** Python（标准库，交叉注意力 + 可学习查询演示）
**前置要求：** Phase 12 · 02（CLIP），Phase 7（Transformer）
**时间：** ~180 分钟

## 学习目标

- 解释为什么在冻结的视觉编码器和冻结的 LLM 之间使用可训练的瓶颈在成本和稳定性上优于端到端微调。
- 实现一个交叉注意力块，其中固定数量的可学习查询关注外部图像特征。
- 讲解 BLIP-2 的两阶段预训练：表示学习（ITC + ITM + ITG）然后是生成学习（带冻结解码器的 LM 损失）。
- 将 Q-Former 与 LLaVA 中使用的更简单的 MLP 投影器进行比较，并说明每种选择何时胜出。

## 问题

你有一个冻结的 ViT，每张图像产生 256 个维度为 1408 的 patch token。你有一个冻结的 7B LLM，期望的 token 嵌入维度为 4096。明显的桥梁——一个从 1408 到 4096 的线性层——有效，但将所有 256 个 patch token 送入 LLM 的上下文需要每张图像多消耗 256 个 token。在 32 张图像的 batch 中，仅视觉模态就消耗了 8192 个 token。

BLIP-2 的问题是：你能将 256 个 token 的图像表示压缩到更少的 token（比如 32 个），同时保留足够的信息供 LLM 进行描述、问答和图像推理吗？而且你能在不触及冻结骨干网的情况下训练这个桥梁，使训练成本仅限桥梁的参数吗？

答案是：一个 Q-Former。32 个可学习的"查询"向量交叉关注 ViT 的 patch token，产生一个 32 个 token 的视觉摘要供 LLM 消费。总共 188M 参数。在触及 LLM 之前，使用对比、匹配和生成目标进行训练。

## 概念

### 可学习查询

Q-Former 的核心技巧：不让 LLM 的文本 token 关注图像 patch，而是引入一组新的 32 个可学习查询向量 `Q`，让*它们*关注图像 patch。这些查询是模型的参数——它们在训练期间学习，并且相同的 32 个查询用于每张图像。

经过交叉注意力后，每个查询持有一个图像的压缩摘要——"描述主要对象"、"描述背景"、"计数对象"等。这些查询并不是字面上专门化于语义标签；它们学习任何能使下游损失下降的编码方式。

### 架构

Q-Former 是一个小型 Transformer（12 层，约 1 亿参数），有两条路径：

1. 查询路径：32 个查询向量流经自注意力（在它们自己之间），然后是对冻结 ViT 的 patch token 的交叉注意力，然后是 FFN。
2. 文本路径：一个类似 BERT 的文本编码器，与查询路径共享自注意力和 FFN 权重。文本路径禁用交叉注意力。

训练时两条路径都运行。查询和文本通过共享的自注意力进行交互，这意味着查询可以在需要时（ITM、ITG）根据文本条件调整。在推理时用于 VLM 移交时，只有查询流经，产生 32 个视觉 token。

### 两阶段训练

BLIP-2 分为两个阶段预训练：

阶段 1：表示学习（无 LLM）。三个损失：
- ITC（图文对比）：CLIP 风格的对比损失，在池化的查询 token 和文本 CLS token 之间。
- ITM（图文匹配）：二元分类器——这个图文对匹配吗？使用难负样本挖掘。
- ITG（基于图像的文本生成）：基于查询条件的文本上的因果 LM 头。迫使查询编码可生成文本的内容。

只有 Q-Former 训练。ViT 冻结。不涉及 LLM。

阶段 2：生成学习。附加一个冻结的 LLM（OPT-2.7B 或 Flan-T5-XL 等）。通过一个小型线性层将 32 个查询输出投影到 LLM 的嵌入维度。将它们前置到文本提示之前。仅在线性投影和 Q-Former 上训练 LM 损失，处理拼接后的提示 + 图像 + 描述序列。

阶段 2 之后，Q-Former + 投影就是完整的视觉适配器。推理时：图像 → ViT → Q-Former → 线性投影 → 前置到文本 → 冻结的 LLM 输出。

### 参数经济学

BLIP-2 使用 ViT-g/14（1.1B，冻结）+ OPT-6.7B（6.7B，冻结）+ Q-Former（188M，训练）= 总计 8B，训练 188M。仅 Q-Former 就占完整堆栈参数的约 2.4%。训练成本反映了这一点：在少量 A100 上只需几天，而端到端需要数周。

质量：BLIP-2 在零样本 VQA 上匹配或超越 Flamingo-80B，同时体积小 50 倍。桥梁有效。

### InstructBLIP 和指令感知的 Q-Former

InstructBLIP（2023 年）扩展了 Q-Former，增加了一个额外输入：指令文本本身。在交叉注意力时，查询现在可以同时访问图像 patch 和指令。查询可以按指令特化（"数汽车数量"、"描述情绪"），而不是学习一个单一的固定摘要。在保留任务上的基准测试获得了提升。

### MiniGPT-4 和纯投影器方法

MiniGPT-4 保留了 Q-Former，但只训练输出线性投影，同时冻结其他所有内容。成本低，但代价是质量——查询是 BLIP-2 的，不是你自己的。适合快速迭代，但不是最佳架构。

### 为什么 LLaVA 选择了更简单的方案

LLaVA（2023 年，课程 12.05）用简单的 2 层 MLP 替换了 Q-Former，该 MLP 将每个 ViT patch token 投影到 LLM 空间——每张图像 576 个 token（24x24 网格），全部馈入 LLM。压缩更差，但让 LLM 可以关注原始 patch。在当时这有争议；到 2023 年底，它变得占主导地位，因为视觉指令数据（LLaVA-Instruct-150k）证明 MLP 可以被训练到保留足够的信号。权衡：LLaVA 的上下文填充更快，但它自然地扩展到多图像和视频。

到 2026 年，领域出现了分化：Q-Former 在 token 预算受限时（长视频、多张图像）存活下来；MLP 投影器在每 token 原始质量优先时占主导地位。

### 门控交叉注意力：Flamingo，祖先

Flamingo（课程 12.04）先于 BLIP-2，使用了相同的交叉注意力思想，但在每个冻结的 LLM 层都使用，而不是作为一个单一的桥梁。BLIP-2 表明你可以仅压缩到输入层，仍然有效。Gemini 和 Idefics 结合了两种方式：交错输入 token 加上可选的门控交叉注意力以支持上下文中的少样本学习。

### 2026 年的后代

- Q-Former：BLIP-2、InstructBLIP、MiniGPT-4 以及大多数视频语言模型（出于 token 预算原因）。
- Perceiver resampler：Flamingo 的变体（课程 12.04）；Idefics 家族、Eagle、OmniMAE。
- MLP 投影器：LLaVA、LLaVA-NeXT、LLaVA-OneVision、Cambrian-1。
- 注意力池化：VILA、PaliGemma。

所有四种方案都有效。决定性的问题是：你受限于 token 预算还是每 token 的质量。

## 使用它

`code/main.py` 构建了一个标准库风格的 Q-Former 交叉注意力：

1. 模拟 256 个图像 patch token（维度 128）。
2. 实例化 32 个可学习查询（维度 128）。
3. 运行缩放点积交叉注意力（Q 来自查询，K/V 来自 patch）。
4. 通过线性层投影到 LLM 维度（512）。
5. 输出 32 个 LLM 就绪的视觉 token。

所有数学运算在纯 Python 中完成（向量的嵌套循环）。玩具级别但形状正确。打印注意力权重矩阵，以便查看每个查询从哪些 patch 中提取了信息。

## 练习

1. 在 PyTorch 中实现交叉注意力块。验证使用 32 个查询和 256 个 key/value 时，注意力权重矩阵是 32 x 256，并且每行在 softmax 后和为 1。

2. 在 BLIP-2 阶段 1 中，Q-Former 同时运行三个损失：ITC、ITM、ITG。用伪代码写出每个损失的前向签名。哪个需要文本编码器路径处于激活状态？

3. 比较参数量：Q-Former（12 层，768 隐藏维度）与 2 层 MLP 投影器（1408 → 4096，两层）。在什么 LLM 规模下，188M 的 Q-Former 成本在训练效率上得到回报？

4. 阅读 BLIP-2 论文（arXiv:2301.12597）第 3.2 节，关于 Q-Former 如何初始化的。解释为什么从 BERT-base（而非随机）初始化可以加速收敛。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| Q-Former | "查询 Transformer" | 小型 Transformer，带有 32 个可学习查询向量，交叉关注冻结的 ViT 特征 |
| 可学习查询 | "视觉的软提示" | 固定数量的参数，作为交叉注意力的查询侧；每个模型学习，在所有输入间共享 |
| 交叉注意力 | "Q 来自这里，K/V 来自那里" | 查询、键和值来自不同源的注意力；查询从 ViT patch 中提取信息的方式 |
| ITC | "图文对比" | CLIP 风格的损失，应用于 Q-Former 池化查询与文本 CLS 之间 |
| ITM | "图文匹配" | 对难负样本挖掘的 pair 进行二分类；迫使查询区分细粒度的不匹配 |
| ITG | "基于图像的文本生成" | 因果 LM 损失，其中文本基于查询条件生成；迫使查询编码可文本解码的内容 |
| 两阶段预训练 | "表示学习然后生成学习" | 阶段 1 单独训练 Q-Former（ITC/ITM/ITG）；阶段 2 附加冻结的 LLM，仅训练投影 + Q-Former |
| 冻结骨干网 | "不微调" | 视觉编码器和 LLM 权重固定；仅桥梁训练 |
| 投影头 | "线性映射到 LLM 维度" | 最终的线性层，将 Q-Former 输出映射到 LLM 的嵌入维度 |
| Perceiver resampler | "Flamingo 的版本" | 类似的可学习查询交叉注意力，Flamingo 在每一层使用而非作为单一桥梁 |
