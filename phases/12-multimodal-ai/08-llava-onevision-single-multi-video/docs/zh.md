# LLaVA-OneVision：单图像、多图像、视频一体模型

> 在 LLaVA-OneVision（Li 等人，2024 年 8 月）之前，开源 VLM 世界有不同的分支：LLaVA-1.5 用于单图像，Mantis 和 VILA 等多图像模型，Video-LLaVA 和 Video-LLaMA 等视频模型。每个在自己的基准上胜出，但在其他方面失败。LLaVA-OneVision 认为单一的课程可以训练一个模型主导所有三种场景，并且涌现出的任务迁移效应（单图像技能导出到视频，多图像推理导出到单图像）超过了专家的总和。这个配方看似简单：一个跨场景保持恒定的视觉 token 预算，加上一个从单图像到 OneVision（多图像）再到视频的显式课程。本课程阅读预算、课程和涌现行为。

**类型：** 构建
**语言：** Python（标准库，token 预算求解器 + 课程规划器）
**前置要求：** Phase 12 · 05（LLaVA），Phase 12 · 06（任意分辨率）
**时间：** ~180 分钟

## 学习目标

- 设计一个在单图像、多图像和视频输入之间保持恒定的视觉 token 预算。
- 安排一个训练课程，将技能从单图像迁移到视频，而不会出现灾难性遗忘。
- 解释为什么在正确的课程下，单个模型在相同参数量下击败专家模型。
- 说出 LLaVA-OneVision 报告的三种涌现能力：多摄像头推理、标记提示、iPhone 截图代理。

## 问题

图像、多图像和视频对模型施加了不同的压力。

单图像需要高分辨率 token（AnyRes，约 2880 个视觉 token）以捕捉 OCR 和精细细节。每个样本预算：一张图像，2880 个 token。

多图像需要几张中等分辨率的图像（每张约 576 个 token），以便跨图像的推理能容纳在上下文中。每个样本预算：4-8 张图像，每张 576 个 token，2300-4600 个 token。

视频需要许多低分辨率的帧（池化后每帧约 196 个 token）以捕捉时间动态。每个样本预算：8-32 帧，每帧 196 个 token，1600-6200 个 token。

如果你训练单独的模型，你选择一个预算。如果你训练一个模型，你需要预算在场景之间合理扩展，而不会炸掉上下文。

OneVision 之前的默认答案是"训练一个场景，忽略其他场景。"Video-LLaVA 用额外的训练阶段将视频改造到图像模型上。LLaVA-NeXT 用平铺增加了多图像支持。没有哪个能干净地处理所有三种场景。

## 概念

### OneVision token 预算

LLaVA-OneVision 选择了一个统一的视觉 token 预算，每个样本约 3000-4000 个 token，根据不同场景分配：

- 单图像：AnyRes-9（3x3 平铺 + 缩略图），每个平铺在 384 下为 729 个 patch，激进 2x2 双线性池化 → 每个平铺 182 个 token。总数：9 * 182 + 182 = 1820 个 token。或 AnyRes-4 在 729 个 token 每个平铺下 = 2916 + 729。
- 多图像：每张图像中等分辨率（384，不平铺），729 个 token，无池化。预算 6 张图像 → 4374 个 token。
- 视频：32 帧在 384 分辨率下，激进 3x3 双线性池化 → 每帧 81 个 token。总数：32 * 81 = 2592 个 token。

分配保持了大致恒定的总 token 数量。LLM 永远不会看到一个会炸掉其上下文的 batch。编码器针对不同场景产生不同的几何形状，但 LLM 消费相同的预算。

### 三阶段课程

LLaVA-OneVision 在三个阶段训练：

1. 单图像 SFT（SI 阶段）。所有数据是单图像加文本。在高分辨率 AnyRes 输入上训练。这教会了感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加上 OneVision 特定的单图像数据。
2. OneVision SFT（OV 阶段）。混合单图像 + 多图像 + 视频（均匀采样帧）。在统一的 token 预算上训练。这教会模型处理异构 batch 形状。不重置权重——从 SI 阶段继续。
3. 任务迁移（TT 阶段）。继续使用目标任务混合，通常根据产品需求侧重多图像或视频。可选的部署微调。

关键点：课程顺序很重要。先视频或多图像训练比先单图像训练产生更差的图像性能，即使使用相同的数据。论文明确消融了这一点。

### 为什么课程有效

单图像训练建立感知基础。Patch token 携带细粒度的视觉特征；LLM 学习将它们与文本集成。多图像和视频引入了结构挑战（哪张图像是哪张、哪个先发生），如果没有强大的感知基础，这些是很难学习的。

如果你从零开始一起训练所有场景，模型会欠拟合感知（每个 batch 中有限的单图像数据）并过拟合结构（大量的多图像/视频数据）。结果是一个模型能跟随跨图像推理模式，但视觉上很浅。

课程排序从 SI 阶段给你感知强度，然后从 OV 阶段给你组合/时间推理能力，而不会丢失任何一个。

### 涌现的跨场景技能

LLaVA-OneVision 论文报告了三种涌现能力：

1. 多摄像头推理。在多图像和视频上分别训练；推理时，被要求对多摄像头驾驶场景进行推理。模型正确整合了多个视角，尽管从未在训练中见过这种确切格式。
2. 标记提示。用户用编号标记在图像中标注对象；模型推理"标记 3 相对于标记 7 在做什么"。在标记或标注上都没有训练过；从空间定位 + 多图像引用的组合中学习。
3. iPhone 截图代理。用户提供 iPhone 屏幕截图并要求计划下一次点击。在 UI 截图、用户工作流程视频和多图像前后对比对上训练。泛化到代理用例。

这些不是训练好的任务；它们从课程的组合结构中涌现出来。

### 视觉 token 池化

Token 预算需要池化。OneVision 在 2D patch 网格上使用双线性插值：24x24 = 576 个 patch 变成 12x12 = 144（2 倍因子）或 8x8 = 64（3 倍因子）。池化是在 patch 网格空间中完成的，而不是在 token 空间中，以保持局部性。

每个场景的池化因子选择本身就是一个超参数。较少的池化 = 更多的 token = 更丰富的表示。更多的池化 = 更少的 token = 可以容纳更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续（LLaVA-OneVision-1.5，arXiv 2509.23661）在训练数据、模型权重和代码上"完全开放"。在某些基准上缩小了与专有模型的差距，并普及了配方。相同的课程，更多的数据，更好的基础 LLM。架构没有变化。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（课程 12.09）做出了不同的选择。它使用 M-RoPE 和动态 FPS 而不是固定池化。它的预算随输入缩放——1 分钟的视频比 5 秒的视频使用更多 token。LLaVA-OneVision 固定预算并缩放池化。两者都有效；它们在可配置性和可预测性之间进行权衡。

## 使用它

`code/main.py` 是一个 OneVision 风格 VLM 的课程和预算规划器。给定每个样本的 token 预算和目标场景混合（比如 40% 单图像、30% 多图像、30% 视频），它：

- 按场景分配分辨率、池化因子和帧数。
- 检查每个场景是否适合共享预算。
- 报告预期的 token 数量、LLM FLOPs 以及哪些场景 token 不足。
- 打印逐阶段的训练计划。

使用它来规划 OneVision 微调，或检查 VLM 部署的每请求成本。

## 交付物

本课程产生 `outputs/skill-onevision-budget-planner.md`。给定目标任务分布和每个样本预算，它输出 AnyRes 因子、每帧池化、视频帧数和课程阶段权重。每当你训练或微调统一场景的 VLM 时使用此工具。

## 练习

1. 你的产品支持 80% 单图像、10% 多图像（2-4 张图像）、10% 视频（8-16 帧）。设计 token 预算。你会在哪里投入从不需要大量多图像处理中节省下来的额外预算？

2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出一个课程可能解锁但论文未报告的第四种涌现技能。

3. 交换课程顺序——先训练多图像，然后单图像，然后视频。预测哪些基准会下降以及为什么。

4. 论文报告在仅 8 帧每样本上训练的视频基准。这能泛化到推理时的 30 秒视频吗？什么先出问题——token 预算还是时间推理？

5. 将 24x24 patch 双线性池化到 12x12 是每个维度 4 倍缩减。在标准库 Python 中实现池化，并验证每个 2x2 块上的均值与双线性输出匹配。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| OneVision 场景 | "单图像、多图像或视频" | 统一 VLM 处理的三种输入形状之一；预算保持恒定 |
| Token 预算 | "每个样本多少 token" | LLM 在每个训练/推理样本中看到的视觉 token 总数，通常为 3000-4000 |
| 课程 | "训练顺序" | 为涌现迁移选择的阶段排序（单图像 → 多图像 → 视频） |
| 双线性池化 | "token 缩减" | 对 patch 网格（2D）应用双线性插值以减少 token 数量同时保持局部性 |
| 涌现技能 | "未经训练，仍能工作" | 在推理时未经过匹配训练数据却出现的能力，源于课程组合 |
| AnyRes-k | "k 平铺设置" | k 个固定分辨率的子平铺加一个缩略图，典型 k ∈ {4, 9} |
| 任务迁移 | "跨场景泛化" | 在单图像上学到的技能通过共享骨干网应用到视频（反之亦然） |
