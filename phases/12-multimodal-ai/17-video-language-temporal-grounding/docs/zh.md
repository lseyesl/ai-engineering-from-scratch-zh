# 视频语言理解：时间定位与密集描述

> VLM（视觉语言模型）在单张图像上表现良好。视频是其测试极限。视频理解不仅仅是"每秒 24 帧的图像理解"——还有时间结构、因果关系和稀疏的依赖关系。两个任务以不同方式对其施加压力：时间定位（"找出说 X 的部分"）和密集视频描述（"详细描述这个视频"）。本课程阅读时间定位——3D 因果卷积、时间注意力、片段级对比损失——并构建一个在视频中找到文本查询的模型。

**类型：** 构建
**语言：** Python（标准库，时间定位和视频描述的玩具 Transformer）
**前置要求：** Phase 12 · 05（LLaVA），Phase 7（视觉）
**时间：** ~180 分钟

## 学习目标

- 在无监督视频分解（镜头分割、时间原子）和监督时间定位（边界 Transformer、SegmentFormer）之间做出选择。
- 实现时间对比损失——鉴别一个视频中正确时间段与错误时间段的损失——并解释它如何比 L1/L2 回归更好地捕获时间边界。
- 构建一个在三种相关基准上压缩视频表示的时间编码器：ActivityNet Captions、YouCook2 和 DiDeMo。
- 设计密集视频描述的词汇表——在语言模型中何时使用视频特征 vs 语言特征。

## 问题

图像模型在基础数据集（VQAv2、COCO Captions）上达到了 95%+ 的准确率。视频 VLM 更困难。原因：

- 时间结构。视频不仅仅是一个静态帧集合。有时一帧依赖于前一帧。有时前一帧没有有帮助的信息。
- 稀疏性。一段 5 分钟的视频可能只有 3 秒相关。没有标记的视频位浪费了 token。
- 长上下文。例如，ActivityNet Captions（最流行的验证基准）每个视频平均 180 秒。在 1 FPS 下，那需要 180 个视觉 token + 相关的文本描述 token。在 4 FPS 下，720 个 token。在 30 FPS 下，5400 个。在有限的上下文中保持计算量可控需要一个策略。
- 时间定位。模型必须知道"何时"，而不仅仅是"什么"。一个损失需要设计来在视频帧的子跨度中找到文本描述。

## 概念

### 无监督视频分解

在训练时间编码器之前，你应该将视频分割成镜头——演员和背景不变的时间跨度。帧被分组为语义"原子"。FFmpeg `scene detect` 或 OpenCV 直方图差将视频分成镜头。

为什么：原始帧并没有捕捉到"场景 A，场景 B，场景 C"的粒度。镜头分割为模型提供了自然的分解。每个镜头平均持续 2-8 帧。

### 时间编码器

时间编码器将视频帧映射到时间标记。你可以使用：

- 空间编码器（CLIP ViT）→ 1D 位置嵌入 → 序列（基本）。每帧独立编码，丢失时间上下文。
- 时态 Transformer（TimeSformer / VideoMAE / 3D CNN）→ 在帧和空间 patch 上使用分解的时空注意力。每帧编码关注前 n 帧。
- 因果卷积。3D 卷积时间上推理。中间表示统一时间和空间。

对于我们的目的，我们使用轻量级的 TimeSformer 变体：一个 ViT，其注意力分解为空间（patch 到 patch）和时间（帧到帧）。视频变为一个 (num_frames * num_patches) 的序列。

### 时间边界 Transformer

边界 Transformer（BdnTR）预测边界 logits。给定一个文本查询和一个视频：

1. 视频编码器产生逐帧嵌入。
2. 文本编码器产生一个查询嵌入。
3. 跨模态编码器使用交叉注意力将查询嵌入与视频嵌入融合。
4. 边界解码器（一个小型 MLP）预测两个标量：start_prob 和 end_prob。预测可能的时间边界是 softmax 概率。

边界回归（L2 损失）：惩罚 start/end 预测与 ground truth 之间的 MSE。简单但迟钝——它将所有时间跨度同等对待。

时间对比损失：对比学习区分两个候选边界之间是否匹配。正样本：重叠 >0.5 的预测。负样本：重叠 <0.1。损失：InfoNCE（温度缩放交叉熵）。比 L2 边界回归更平滑。

### 密集视频描述的词汇表

一旦你找到了跨度，密集视频描述将其转换为语言。视频不是作为帧馈入，而是作为时间压缩的 token 序列——每个镜头 1 个 token。结合文本 token 形成单个序列。模型在无需处理所有帧的情况下生成字幕描述。

实际考虑：

- 语言模型词汇表是最小公分母。视频 token 需要映射到 LLM 的词汇表（一段投影+softmax）。
- 词汇表应涵盖视频实体（"汽车"、"人"、"说话"）、时间关系（"之前"、"之后"、"在……期间"）和空间前缀词（"角落里"、"后面"）。
- 带分割标记的 BPE 词汇表对于密集视频词来说还不够。添加一个小的视频特定词汇表（256-512 个 token）。用 LoRA 训练它们。

### 基准测试

| 基准 | 任务 | 指标 | 挑战 |
|------|------|------|------|
| ActivityNet Captions | 密集描述 + 时间定位 | 在 IoU=0.5 时的 mAP | 长视频（平均 180 秒），冗长描述 |
| YouCook2 | 长视频（平均 320 秒），77 道食谱 | mAP | 伪标签传播很重要 |
| DiDeMo | 时间定位（段落到段落的对应） | R@1,N=5 | 多个注释的可变性 |
| MAD（2022 年） | 定位（平均 5.3M 帧） | mAP | 极端长视频（完整电影） |

前三者都是评估视频时间定位的合理选择。选择取决于你的视频长度（ActivityNet = 日常事件，YouCook2 = 视觉相似动作，DiDeMo = 众包环境）。MAD 是更近期的，但你的模型可能不擅长完整电影。

代码展示了固定帧计数 + 可学习时间编码，以及视频如何对抗帧计数变化。演示了边界 Transformer + 时间对比损失。

## 练习

1. 下载一个 5 秒的短视频。将其分解为镜头（平均每镜头 3 帧）。每个镜头产生多少个 patch？在 4 FPS 下一分钟的视频会产生多少个 token？

2. 实现边界 Transformer：给定 128 维查询嵌入和 64 帧的 1024 维视频帧编码，设计一个解码器来预测 start/end logits。

3. 直觉：为什么 L2 边界回归在时间定位中效果不佳？描述一个场景，其中 ground truth 边界为 [10, 20]，预测 [30, 40] 得到的 L2 损失很小，但重叠为 0。证明时间对比损失为何解决了这个问题。

4. ActivityNet Captions 视频平均 180 秒。在 4 FPS 下，以 14x14 的 patch 大小，每个视频每帧会产生多少个 token？你可以使用多少帧的上下文窗口，而这对于基于视频的描述是否足够？

5. 使用 TimeSFormer（n=4 帧，每帧 m=16 个 patch）的比较：空间注意力和时间注意力的复杂度是多少？证明组合注意力的复杂度从 O(n^2 × m^2) 降低到 O(n^2 + m^2)。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 时间定位 | "找到视频中的跨度" | 给定一个文本查询，预测一个开始/结束时间跨度 |
| 时间对比损失 | "边界对比" | 对比学习损失，对正确时间跨度嵌入的信息更丰富，优于 L2 边界回归 |
| 镜头级分解 | "切视频" | 将视频分解为动作发生的语义时间跨度 |
| 边界 Transformer | "BdnTR" | 预测 start/end 边界 logits 的解码器 |
| 密集视频描述 | "视频到语言" | 为时间跨度生成字幕文本，在一个视频中生成多段描述 |
| 时间编码器 | "TimeSFormer" | 在 patch 和帧上使用分解注意力的视频编码器 |
