# 开源权重 VLM 配方：什么才真正重要

> 2024-2026 年的开源权重 VLM 文献是一片消融实验的森林。Apple 的 MM1 测试了图像编码器、连接器和数据混合的 13 种组合。Allen AI 的 Molmo 证明了详细的人工描述胜过 GPT-4V 蒸馏。Cambrian-1 进行了 20 多个编码器比较。Idefics2 形式化了五轴设计空间。Prismatic VLMs 在受控基准上比较了 27 个训练配方。在所有这些噪音中，一组小规模的结果在各种论文中保持一致：图像编码器比连接器架构更重要，数据混合比前两者都更重要，详细的人工描述胜过蒸馏合成数据。本课程阅读这些表格，这样你就不必自己做了。

**类型：** 学习 + 实验
**语言：** Python（标准库，消融表格解析器 + 配方选择器）
**前置要求：** Phase 12 · 05（LLaVA 基线）
**时间：** ~180 分钟

## 学习目标

- 说出 VLM 的五轴设计空间：图像编码器、连接器、LLM、数据混合、分辨率计划。
- 阅读 MM1 / Idefics2 / Cambrian-1 的消融表格，预测哪个旋钮会移动给定的基准分数。
- 为给定计算预算和任务混合的新 VLM 选择一个配方（编码器、连接器、数据、分辨率）。
- 解释为什么在相同的 token 数量下，详细的人工描述优于 GPT-4V 蒸馏。

## 问题

存在数百种开源权重 VLM。"好"和"最先进"之间的大部分差距不是架构。而是数据、分辨率计划和编码器选择。知道当你的模型表现不佳时首先转动哪个旋钮，可以帮你避免 500 万 GPU 小时的错误。

2023 年的浪潮（LLaVA-1.5、InstructBLIP、MiniGPT-4）基于描述对预训练 + LLaVA-Instruct-150k。不错的基线。MMMU 上限约为 35%。

2024 年的浪潮（MM1、Idefics2、Molmo、Cambrian-1、Prismatic VLMs）进行了详尽的消融实验。结果令人惊讶且实用。

## 概念

### 五轴设计空间

Idefics2（Laurençon 等人，2024 年）命名了这些轴：

1. 图像编码器。CLIP ViT-L/14、SigLIP SO400m/14、DINOv2 ViT-g/14、InternViT-6B。编码器在 patch 大小、分辨率和预训练目标上有所不同。
2. 连接器。MLP（2-4 层）、Q-Former（32 个查询 + 交叉注意力）、Perceiver Resampler（64 个查询）、C-Abstractor（卷积 + 双线性池化）。
3. 语言模型。Llama-3 8B / 70B、Mistral 7B、Phi-3、Gemma-2、Qwen2.5。LLM 大小是主要的参数成本。
4. 训练数据。描述对（CC3M、LAION）、交错数据（OBELICS、MMC4）、指令数据（LLaVA-Instruct、ShareGPT4V、PixMo、Cauldron）。
5. 分辨率计划。固定 224/336/448、AnyRes、原生动态。在训练期间递增或恒定。

每个生产型 VLM 都在每个轴上做出选择。MMMU 分数的大部分差异由轴 1、4 和 5 解释——而不是你选择了哪个连接器。

### 轴 1：编码器 > 连接器

MM1 第 3.2 节显示：从 CLIP ViT-L/14 切换到 SigLIP SO400m/14 增加了 3 个以上的 MMMU 点数。将连接器从 MLP 切换到 Perceiver Resampler 增加了不到 1 个点。Idefics2 复现了：SigLIP > CLIP，Q-Former ≈ MLP ≈ Perceiver 在相同的 token 数量下。

Cambrian-1 的 "Cambrian 视觉编码器对决"（Tong 等人，2024 年）在视觉中心基准（CV-Bench）上测试了 20 多个编码器。排行榜前列是 DINOv2 和 SigLIP 的混合；CLIP 处于中间位置；ImageBind 和 ViT-MAE 较低。从 CLIP ViT-L 到 DINOv2 ViT-g/14 的差距在 CV-Bench 上约为 5-7 个点。

2026 年开源 VLM 的默认编码器是 SigLIP 2 SO400m/14（用于语义 + 密集特征），有时与 DINOv2 ViT-g/14 特征拼接（Cambrian 的"空间视觉聚合器"就是这样做的）。

### 轴 2：连接器设计几乎无关紧要

MM1、Idefics2、Prismatic 和 MM-Interleaved 都得出了相同的结论：在固定的视觉 token 数量下，连接器架构几乎没有影响。在均值池化的 patch 上使用 2 层 MLP 的性能与在相同 token 预算下的 32 查询 Q-Former 相差在 1 个点以内。

真正重要的是 token 数量。更多的视觉 token = 更多的 LLM 计算 = 更好的性能，直到某个点，然后收益递减。每张图像 64 个 token 对 OCR 来说太少了。576-1024 个 token 是大多数开源 VLM 的最佳点。2048+ 仅在文档和图表上有帮助。

Q-Former vs MLP 是一个成本问题，而不是质量问题：Q-Former 将 token 限制在 32-64 个，无论图像分辨率如何；MLP 发出所有 patch token。对于高分辨率输入，Q-Former 节省 LLM 上下文；对于低分辨率，差异只是噪音。

### 轴 3：LLM 大小决定天花板

将 LLM 从 7B 翻倍到 13B，每篇 VLM 论文中 MMMU 都可靠地增加 2-4 个点。在 70B 时，你饱和了大多数基准测试。VLM 的多模态推理天花板就是 LLM 的文本推理天花板——视觉编码器只能馈入信息，不能替它推理。

这就是为什么 Qwen2.5-VL-72B 和 Claude Opus 4.7 在 MMMU-Pro 和 ScreenSpot-Pro 上碾压对手：语言大脑很大。一个 7B 的 VLM 不能通过巧妙的连接器设计替代 70B 的 VLM。

### 轴 4：数据——详细的人工描述胜过蒸馏

Molmo + PixMo（Deitke 等人，2024 年）是每个人都应该阅读的 2024 年结果。Allen AI 让人工标注者用 1-3 分钟的密集语音到文本过程描述图像，产生了 71.2 万张密集描述的图像。训练数据中没有任何 GPT-4V 蒸馏。

Molmo-72B 在 11 个基准中的 11 个上击败了 Llama-3.2-90B-Vision。差异不是架构——而是描述质量。详细的包含描述的图像比简短的网络描述多 5-10 倍的信息，并且在 GPT-4V 蒸馏产生幻觉的地方保持事实准确性。

ShareGPT4V（Chen 等人，2023 年）和 Cauldron（Idefics2）遵循了同样的策略，混合了人工和 GPT-4V 描述。趋势很明确：对于 2026 年的前沿，描述密度 > 描述数量 > 蒸馏便利性。

### 轴 5：分辨率及其计划

Idefics2 的消融实验：384 -> 448 增加 1-2 个点。448 -> 980 加图像分割（AnyRes）在 OCR 基准上再增加 3-5 个点。固定分辨率训练在中低准确率处达到平台期；分辨率递增（从 224 开始，以 448 或原生分辨率结束）训练更快，终点更高。

Cambrian-1 进行了分辨率 vs token 的权衡：在固定计算下，你可以有较低分辨率下的更多 token，或较高分辨率下的更少 token。较高分辨率在 OCR 上胜出；较低分辨率更多 token 在一般场景理解上胜出。

2026 年的生产配方：阶段 1 在 384 固定下训练，阶段 2 使用动态分辨率，对于 OCR 重任务可达 1280。

### Prismatic 受控比较

Prismatic VLMs（Karamcheti 等人，2024 年）是一篇控制了所有轴的论文。相同的 13B LLM、相同的指令数据、相同的评估——一次只有一个轴变化。结果：

- 每张图像的视觉 token 数量解释了约 60% 的方差。
- 编码器选择解释了约 20%。
- 连接器架构解释了约 5%。
- 其他一切（数据混合、调度器、学习率）解释了剩余的约 15%。

这是一个粗略的分解，但它是文献中"我应该首先消融什么"的最清晰答案。

### 2026 年的选择器

鉴于证据，2026 年新项目的默认开源 VLM 配方：

- 编码器：SigLIP 2 SO400m/14，使用 NaFlex 的原生分辨率，如果你需要分割/接地，可拼接 DINOv2 ViT-g/14 用于密集特征。
- 连接器：patch token 上的 2 层 MLP。除非你受 token 约束，否则跳过 Q-Former。
- LLM：Qwen2.5 / Llama-3.1 / Gemma 2，7B 用于成本，70B 用于质量，根据目标延迟选择。
- 数据：PixMo + ShareGPT4V + Cauldron，加上特定任务的指令数据。
- 分辨率：动态（每长边最小 256，最大 1280 像素）。
- 计划：阶段 1 对齐（仅投影器）、阶段 2 完全微调、阶段 3 任务特定微调。

每个默认值都可以追溯到本课程末尾引用的论文中的测量消融实验。

## 使用它

`code/main.py` 是一个消融表格解析器和配方选择器。它编码了 MM1 和 Idefics2 的消融表格（压缩形式），让你查询：

- "给定预算 X 和任务 Y，哪个配方胜出？"
- "如果我在 7B Llama 上将 SigLIP 换成 CLIP，预期的 MMMU 差异是多少？"
- "对于 80% 置信度的答案，我应该首先消融哪个轴？"

输出是一个排序的配方列表，带有预期的基准差异和"首先消融"的建议。

## 交付物

本课程产生 `outputs/skill-vlm-recipe-picker.md`。给定目标任务混合、计算预算和延迟目标，它输出完整的配方（编码器、连接器、LLM、数据混合、分辨率计划），并附有证明每个选择合理的消融实验引用。防止工程师每次启动新的 VLM 项目时都重新发明 Idefics2 的消融表格。

## 练习

1. 阅读 MM1 第 3.2 节。对于一个固定的 2B LLM，预算 5000 万张图像，哪个编码器胜出？如果换成 13B LLM，答案会翻转吗？为什么？

2. Cambrian-1 发现拼接 DINOv2 + SigLIP 在视觉中心基准上优于任何一个单独使用，但在 MMMU 上没有增加信号。预测哪些基准会提升，哪些会保持平稳。

3. 你的目标是在 2B LLM 上做一个移动 UI 代理。选择编码器、连接器、分辨率和数据混合。用具体的消融表格证明每个选择。

4. Molmo 提供了 4B 和 72B 模型。4B 与闭源 7B VLM 竞争；72B 在 11/11 基准上击败了 Llama-3.2-90B-Vision。这告诉你关于 LLM 大小平台期假设的什么信息？

5. 设计一个消融表格，在 7B VLM 上隔离数据混合质量与编码器质量的影响。最少需要多少训练运行？提出四个轴设置。

## 关键术语

| 术语 | 人们说的 | 实际含义 |
|------|----------|----------|
| 消融 | "转动一个旋钮" | 在恰好一个设计空间轴上不同的多次训练运行，保持其他一切不变 |
| 连接器 | "桥梁" / "投影器" | 将视觉编码器输出映射到 LLM token 空间的可训练模块（MLP、Q-Former、Perceiver） |
| 详细人工描述 | "密集描述" | 多句人工撰写的描述（通常 80-300 token），比网络替代文本更丰富 |
| 蒸馏 | "GPT-4V 描述" | 由更强的专有 VLM 生成的训练数据；方便但容易出现继承的幻觉 |
| AnyRes / 动态分辨率 | "高分辨率路径" | 通过平铺或 M-RoPE 将大于编码器原生分辨率的图像馈入模型的策略 |
| 分辨率递增 | "课程" | 从低分辨率开始并逐渐增加的训练计划，加速对齐学习 |
| 视觉中心基准 | "CV-Bench / BLINK" | 强调细粒度视觉感知而非语言重推理的评估 |
| PixMo | "Molmo 的数据" | Allen AI 的 71.2 万张密集描述图像数据集；人工语音转录为密集描述 |
