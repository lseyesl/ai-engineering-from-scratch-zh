# 多 Token 预测 (MTP)

> 从 GPT-2 到 Llama 3，每个自回归 LLM 在每个位置上只训练一个损失：预测下一个 token。DeepSeek-V3 在每个位置上增加了第二个损失：预测再下一个 token。额外的 14B 参数（在 671B 模型上）通过梯度流被蒸馏回主模型，训练好的 MTP 头在推理时被重新用作推测解码的草稿器，接受率 80%+。1.8 倍生成吞吐量免费获得。本课程从 DeepSeek 技术报告构建顺序 MTP 模块，计算损失和共享头参数布局，并解释为什么 MTP 保持了因果链，而 Gloeckle 等人的原始并行 MTP 打破了它。

**类型：** 构建
**语言：** Python (stdlib)
**前置课程：** Phase 10 · 04 (预训练 mini GPT), Phase 10 · 15 (推测解码)
**时长：** ~60 分钟

## 学习目标

- 说出 MTP 训练目标并推导跨预测深度的联合损失。
- 解释 Gloeckle 等人的并行 MTP 头（2024）与 DeepSeek-V3 的顺序 MTP 模块之间的区别，以及为什么顺序设计保持了因果链。
- 计算在预训练运行中添加 MTP 模块的参数和内存开销。
- 从零实现一个 MTP 模块：共享嵌入、逐深度 transformer 块、投影和共享输出头。

## 问题所在

下一 token 预测是标准 LLM 训练目标。每个隐藏状态被监督预测恰好一件事：紧随其后的 token。这是一个令人惊讶地弱的信号。序列中的大部分信息延伸超过一个 token——结构、连贯性、事实性、算术流。模型必须通过在万亿 token 上积累许多单 token 信号来学习这些。

MTP 问：如果每个隐藏状态被同时监督预测多个未来 token 会怎样？Gloeckle 等（Meta，2024）表明这有帮助。他们的实现在骨干网络顶部放置了几个独立的输出头，每个预测不同的偏移量。并行、简单，但这些头看到相同的隐藏状态而没有任何层次化精炼——而且预测之间没有因果链，所以它们不能用于推测解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为保持每个预测深度因果链的顺序模块。模型从 `h_i^(0)` 预测 `t+1`，然后从结合了 `h_i^(0)` 与 `E(t+1)` 嵌入的新隐藏状态 `h_i^(1)` 预测 `t+2`，以此类推。每个深度是自己的小型 transformer 块。共享嵌入和共享输出头保持参数开销适中。在 DeepSeek-V3 的规模上，MTP 模块在 671B 主模型权重之上增加了 14B 额外参数。2% 的开销买到了更密集的训练信号和推理时现成的推测解码草稿。

本课程从零构建单个 MTP 模块和 D 深度损失。数学很整洁。实现是 150 行。

## 核心概念

### 顺序 MTP 方案

DeepSeek-V3 在主模型之上添加 `D` 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度 `k` 处的 token——即给定通过位置 `i` 的前缀，预测 `t_{i+k}`。

模块 `k` 由以下组成：

- 一个具有自己注意力和 MLP 的 transformer 块 `T_k`。
- 一个投影矩阵 `M_k`，将前一深度隐藏状态与下一深度真实 token 的嵌入组合。
- 共享嵌入 `E`（与主模型相同）。
- 共享输出头 `Out`（与主模型相同）。

在训练时，对于通过位置 `i` 的前缀，逐深度隐藏状态为：

```
h_i^(0) = 主模型骨干在位置 i 的输出
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) ) 对于 k >= 1
```

逐深度预测为：

```
logits_{i+k} = Out(h_i^(k-1)) 对于 k = 1..D
```

逐深度损失是对真实 `t_{i+k}` 的交叉熵：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个小的加权因子——DeepSeek-V3 在训练前 10% 使用 0.3，之后使用 0.1。总训练损失为 `L_main + L_MTP`。

### 为什么是顺序而非并行

Gloeckle 的原始并行 MTP 有 D 个输出头，每个直接应用于 `h_i^(0)`。每个头从相同的骨干隐藏状态预测 `t_{i+k}`。训练效果不错，但预测之间没有条件依赖。你不能用 `head_1` 的输出来帮助 `head_2`——这些头并行触发。

DeepSeek-V3 的顺序设计从 `h_i^(k-1)` 加上实际下一 token 嵌入 `E(t_{i+k})` 构建 `h_i^(k)`。这保持了因果链：要预测 `t_{i+k+1}`，深度 `k+1` 的模块看到了 `t_{i+k}` 处的内容。这在结构上与自回归解码器消费自身输出的方式相同——使得 MTP 模块可直接用作推测解码的草稿器。

在推理时：将 `h_i^(k-1)` 和草拟的 `t_{i+k}` 输入模块 `k+1`，得到 `t_{i+k+1}` 的预测。重复。这正是 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告第一个 MTP 模块的接受率 80%+，约 1.8 倍加速。

### 参数核算

对于隐藏维度为 `h`、词汇量为 `V` 的模型：

- 主模型：数十亿参数，加上一个大小为 `V * h` 的输出头。
- 共享输出头：复用主模型的头。无额外参数。
- 共享嵌入：复用主模型的嵌入。无额外参数。
- 每个 MTP 模块：
  - 投影 `M_k`：`(2h) * h = 2h^2`。
  - Transformer 块 `T_k`：注意力（MHA 为 `4h^2`）加 MLP（SwiGLU 比例 8/3 时通常为 `8h^2`）。约 `12h^2` 每块。

每个模块额外总计：`~14h^2`。对于 DeepSeek-V3 的 `h = 7168`，D = 1 个模块：`~14 * 7168^2 = ~720M` 参数（理论值）。DeepSeek-V3 报告 14B——差异主要来自 MTP 模块中的专家层也是 MoE。

### 推测解码的回报

在预训练期间，MTP 模块使训练减慢约 10%（更多前向计算，额外损失）。回报是双重的：

1. 更密集的训练信号。每个隐藏状态看到 D+1 个监督目标。在 MMLU、GSM8K、MATH、HumanEval 上的测量效果：DeepSeek-V3 的消融实验中一致的几个百分点提升。

2. 推理时免费的推测解码草稿。MTP 模块已经训练好预测接下来几个 token。重新用作草稿网络，它提供 80%+ 的接受率。在该水平下，N=3 或 N=5 的推测解码给出 1.8 倍吞吐量。10% 的训练时间成本在你第一次运行推理时就回本了。

### 与 EAGLE 的关系

EAGLE 在预训练之后单独训练一个小型草稿模型。MTP 将草稿烘焙到预训练中。两种方法在类似的接受率上趋同，但通过不同的流水线：

| 维度 | EAGLE-3 | MTP (DeepSeek-V3) |
|------|---------|------------------|
| 训练时机 | 预训练之后 | 预训练期间 |
| 与现有权重向后兼容 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2 个 transformer 层 | 1 个 transformer 块 + 投影 |
| 接受率 | 0.88-0.92 | 深度 1 处 0.80+ |
| 加速之外的收益 | 仅推测解码 | 更密集训练信号 + 加速 |

## 构建它

`code/main.py` 端到端构建单个 MTP 模块：共享嵌入、投影、transformer 块、共享输出头。然后在短合成序列上计算逐深度交叉熵损失，并按组件打印参数计数。32 个 token 的玩具词汇量使数字可读。

### 步骤 1：共享嵌入表

一个 `vocab_size x hidden` 的表被主模型和每个深度的每个 MTP 模块使用。不是第二个副本——字面上是同一个张量。

### 步骤 2：逐深度组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维度拼接，然后投影回隐藏维度
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed) # 向量加法替代
    projected = matvec(M_k, concat)
    return projected
```

真正的 DeepSeek-V3 将两个 RMSNorm 后的向量拼接为 `[2h]` 并用 `h x 2h` 矩阵投影。玩具实现使用向量加法以保持 stdlib 简洁。

### 步骤 3：深度 k 的 transformer 块

自注意力加 MLP。在玩具实现中，一层线性注意力块和 SwiGLU MLP 保持结构可见而无需 numpy。

### 步骤 4：共享输出头

复用主模型的输出投影。词汇上的 logits。

### 步骤 5：逐深度损失

softmax(logits) 对偏移 `k` 处真实 token 的交叉熵。用 `lambda / D` 缩放因子跨深度聚合。

### 步骤 6：参数核算

打印总参数计数、共享（嵌入、头）计数和每模块额外计数。展示 MTP 额外参数与主模型大小的比率。

## 使用它

MTP 已集成到 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列中。在推理时：

- DeepSeek 自身的推理栈开箱即用地将 MTP 模块用作推测解码器。
- 截至 2026 年 4 月，vLLM 和 SGLang 有 DeepSeek-V3 MTP 的集成路径。
- AMD 的 ROCm SGLang 教程展示了特定的 MTP 推测解码配置，在 V3 检查点上测量到 1.8 倍加速。

何时在新预训练运行中使用 MTP：

- 你控制完整的预训练流程并希望获得更密集的训练信号。
- 你知道你将大规模服务模型并希望免费获得推测解码。
- 你的隐藏维度至少为 4096。在 1B 规模上，开销的伤害大于收益的帮助。

何时不选：

- 微调现有预训练密集模型。MTP 模块未训练。
- 你想要干净基线进行比较的研究模型。MTP 改变了架构。

## 练习

1. 运行 `code/main.py`。展示逐深度损失随合成信号增强而单调递减。修改合成数据使用固定模式，验证深度 1 和深度 2 的损失都收敛。

2. 计算密集 70B 模型（隐藏 8192，80 层）加 D=1 MTP 模块的参数开销。与 DeepSeek-V3 报告的 14B 开销比较。解释为什么 DeepSeek 的数字更高：MTP transformer 块继承了相同的 MoE 结构，膨胀了每模块参数计数。

3. 在玩具实现中实现 D=2：添加第二个 MTP 模块，取 h^(1) 并预测 `t_{i+2}`。验证联合损失和参数核算与 DeepSeek 论文公式 19-21 匹配。

4. 将玩具实现切换为并行 MTP（Gloeckle 风格）：在主隐藏状态顶部添加 D 个输出头，每个预测不同偏移。测量在相同合成信号上每个深度的损失与顺序版本的比较。顺序版本应在 k > 1 时产生更低的深度 k 损失，因为它以中间预测为条件。

5. 使用训练好的 MTP 模块作为 EAGLE 风格的草稿：在推理时调用模块 k 来提议 `t_{i+k}`。测量这些草稿 token 对主模型预测在留出序列上的接受率。如果在玩具实现上达到 50%+，你就复现了 MTP 作为草稿的经验特性。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|-----------|---------|
| MTP module | "额外损失块" | 一个小型 transformer 块加投影，预测主模型前方 `k` 个位置的 token |
| Prediction depth | "哪个偏移" | 整数 `k`，使得模块 `k` 从通过位置 `i` 的前缀预测 `t_{i+k}` |
| Parallel MTP | "Gloeckle 风格" | 同一骨干隐藏状态上的 D 个独立头，无条件链 |
| Sequential MTP | "DeepSeek-V3 风格" | 每个模块以前一深度隐藏状态加下一 token 嵌入为条件；保持因果链 |
| Shared output head | "复用主头" | MTP 模块调用主模型的 LM 头，而非单独的输出投影 |
| Shared embedding | "复用主表" | 同一词汇嵌入表到处使用；无重复参数 |
| Projection matrix M_k | "组合隐藏 + 下一 token" | 将前一隐藏状态和目标 token 嵌入折叠为下一深度输入的 `h x 2h` 线性层 |
| Joint loss L_MTP | "平均额外损失" | 逐深度交叉熵损失的算术平均，以 `lambda` 缩放 |
| Acceptance rate at depth 1 | "MTP 草稿正确的频率" | D=1 MTP 模块的 top-1 预测等于主模型 top-1 预测的比率；DeepSeek-V3 上 80%+ |
| Lambda weighting | "额外损失重要性" | 逐深度缩放因子；DeepSeek-V3 训练开始时 0.3，之后 0.1 |
