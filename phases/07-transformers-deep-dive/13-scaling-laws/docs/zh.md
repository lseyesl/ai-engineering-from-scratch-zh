# 扩展定律

> 2020 年的 Kaplan 论文说：更大的模型，更低的损失。2022 年的 Hoffmann 论文说：你训练不足。计算分为两个桶——参数和 token——而分配并不明显。

**类型：** 学习
**语言：** Python
**前置知识：** 阶段 7 · 05（完整 Transformer），阶段 7 · 07（GPT）
**时间：** ~45 分钟

## 问题

当你拥有 C FLOPs 的训练计算资源并想要最佳模型时，你面临两个旋钮：

1. **多少参数（N）？** 更大的模型，更高的容量。
2. **多少训练 token（D）？** 更多的数据，更好地利用容量。

FLOPs 大约以 `6 × N × D` 的比例扩展。你可以把 N 推高而 D 降低，或者 D 推高而 N 降低。哪个更好？

在 2022 年之前，答案是"大力推 N"。GPT-3（2020）是 175B 参数，在约 300B token 上训练。每个参数约 1.7 个 token。Kaplan 扩展定律支持这一点。

Hoffmann 等人（2022）训练了一个名为 Chinchilla 的小型模型家族，发现了不同之处：最优比率接近**每参数 20 个 token**。GPT-3 训练不足 10 倍。Chinchilla（70B 参数，1.4T token）在每个基准测试上以 2.5 倍更低的推理成本击败了 GPT-3（175B，300B token）。

2026 年是 Chinchilla 的世界——但有一个重要的转折。Llama 3 8B 在 15 万亿 token 上训练，每参数 1,875 个 token 的比率。这是 Chinchilla 最优的 94 倍。对于将被大规模使用的模型，推理成本比训练成本更重要，因此在 2026 年，过度训练（超过 Chinchilla）以获得更小的可部署体积是默认选择。

## 概念

![Chinchilla 曲线：各种 N/D 比率下损失 vs 计算量](../assets/scaling-laws.svg)

### Hoffmann 定律

从 Chinchilla 论文来看，损失遵循：

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 参数（非嵌入）。
- `D` = 训练 token。
- `α ≈ 0.34`，`β ≈ 0.28`（大致对称）。
- `E ≈ 1.69`，不可约损失上限。
- `A ≈ 406`，`B ≈ 411`。

随着你进行扩展，两个项相互权衡。在固定计算量（C = 6ND）下对 N 求导并求解：

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

计算最优：每参数 20 个 token。

### 为什么要过度训练

Chinchilla 最优最小化每训练 FLOP 的训练损失。但训练成本你只付一次；推理成本永远持续。

对于一个每月服务万亿个 token 的聊天机器人，推理主导总成本。Llama 的方法：训练更久、更小。8B 在 15T token 上训练是深度推理优化的：

- 适合消费级 GPU。
- 延迟只是 70B Chinchilla 最优模型的一小部分。
- 质量在大多数任务上足够接近。

DeepMind 2024 年的论文（"过度训练是新的最优"）形式化了这一点。对于推理主导的工作负载，根据服务量，正确的比率接近每参数 100-500 个 token。

### 涌现 vs 平滑性

有人声称：某些能力（算术、多步推理、思维链遵循）在某个规模上"突然涌现"。

Schaeffer 等人（2023）认为这是一个测量伪影：涌现度量使用不连续的评分（精确匹配、阈值准确率），隐藏了底层 logits 中的平滑改进。连续度量（交叉熵）显示平滑曲线。

2026 年的共识是：通过连续损失进行预测是可靠的。基准测试的跳跃通常是评分器伪影。根据连续指标规划预算。

### 2026 年图景

扩展定律仍然有效，但：

| 因素 | 变化方式 |
|--------|-------------|
| 数据质量 | 策划"好的"token（Phi 风格）将曲线移动 >2 倍有效计算 |
| MoE | 总参数与活跃 FLOPs 解耦；每活跃 FLOP 的扩展定律 |
| 后训练 | 某些能力（指令遵循、代码）随 SFT+RLHF 的变化比预训练更大 |
| 多模态 | 图像 + 文本 token 一起扩展；每模态有单独的曲线 |
| 合成数据 | 模型生成训练数据；有效计算可以复合 |

Muon 优化器（Kimi Moonlight，2024）在相同数据下显示出约 2 倍于 AdamW 的有效计算增益。一些 2026 年训练运行默认使用 Muon。改变扩展定律中的绝对常数，而不是其形状。

## 动手实现

参见 `code/main.py`。我们实现 Chinchilla 损失方程，并在几个计算预算下求解计算最优 `(N, D)`。

### 步骤 1：Chinchilla 损失

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

在固定 `C = 6ND` 下绘制 `(N, D)` 上的 `L` 等高线图。找到最小值。

### 步骤 2：计算最优前沿

对于从 `1e17` 到 `1e25` FLOPs 的计算预算，找到使损失最小化的 `(N, D)`，约束条件为 `6ND = C`。验证比率 `D/N ≈ 20`。

### 步骤 3：过度训练成本

计算训练一个 10 倍更小的模型（1/10 的最优 N，10 倍的最优 D）需要支付的额外损失。报告交换而来的推理 FLOPs 节省（与 N 成比例）。

### 步骤 4：与真实模型比较

输入 GPT-3、Chinchilla、Llama 3 8B、DeepSeek-V3（活跃参数）的已知 `(N, D)` 对，并比较预测损失与实际报告的损失。

## 使用

你不太可能自己训练前沿模型。但扩展定律告诉你：

1. **你的微调是否有足够的数据。** 如果你的任务特定数据低于基础模型每参数 20 个 token，预计会在某个损失底线处饱和。
2. **是否选择更大的基础模型。** 如果你把所有预算都花在推理上，宁愿选择更小、训练更久的模型。
3. **收益递减点在何处。** 超过 Chinchilla 最优 1,000 倍后，对数损失变化变成噪声。

**2026 年的研究轨迹：**

- **数据受限时代。** 网络上的高质量 token 数量有限（过滤后约 5-10 万亿英语 token）。前沿预训练正在接近这个上限。合成数据、多语言、多模态和 RLHF 规模的微调是下一个杠杆。
- **计算倍增技巧。** Muon 优化器、MoE、更好的数据策划——每个都改变绝对常数，而不是渐近线。
- **RL 的扩展定律。** 开放问题。早期证据表明 RL 样本中的幂律但指数与预训练非常不同。

## 关键术语

| 术语 | 人们说它是什么 | 它实际意味着什么 |
|------|-----------------|-----------------------|
| 参数（N） | "模型大小" | 非嵌入权重数量；决定容量。 |
| Token（D） | "训练数据" | 见过的训练 token 数量；决定参数被利用的程度。 |
| 计算量（C） | "消耗的 FLOPs" | 对于标准 transformer 约为 `6 × N × D`。 |
| Chinchilla 最优 | "D/N ≈ 20" | 最小化每 FLOP 预训练损失的比率。 |
| 过度训练 | "超过 Chinchilla" | 花费额外的训练 FLOPs 以节省推理 FLOPs；D/N >> 20。 |
| 不可约损失 | "底线" | 扩展定律中的 `E` 项；数据本身的熵。 |
| 涌现能力 | "规模上的突然跳跃" | 通常是评分器伪影；连续损失是平滑的。 |
| 有效计算 | "训练效率倍增器" | 更好的数据/优化器/架构将每个 FLOP 的效力放大。 |
