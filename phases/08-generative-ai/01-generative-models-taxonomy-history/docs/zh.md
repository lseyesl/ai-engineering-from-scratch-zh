# 生成模型 — 分类与历史

> 每个图像模型、文本模型、视频模型和 3D 模型都属于五个桶之一。选错桶，你将与数学搏斗数周。选对桶，该领域过去十二年的进展就会清晰地堆叠在你的脑海中。

**类型：** 学习
**语言：** Python
**前置知识：** 阶段 2（ML 基础），阶段 3（深度学习核心），阶段 7 · 14（Transformer）
**时间：** ~45 分钟

## 问题

生成模型做一件事：给定从某个未知分布 `p_data(x)` 中抽取的训练样本，输出看起来来自同一分布的新样本。人脸、句子、MIDI 文件、蛋白质结构——如果你眯起眼睛，都是同样的问题。

问题在于 `p_data` 存在于一个拥有数百万维度的空间中（512x512 RGB 图像约 786k 维），样本位于该空间内的一个薄流形上，而你只有大约 1000 万个样本。暴力求解密度是不可行的。每个生成模型都是一种妥协，将一个难题换成一个稍微不那么难的。

五个家族在过去十二年中幸存下来。了解每个家族做出的妥协，就能告诉你为什么它在某些任务上获胜而在其他任务上崩溃。

## 概念

![五个生成模型家族——按照它们建模的对象分类](../assets/taxonomy.svg)

**1. 显式密度，可处理。** 将 `log p(x)` 写成一个你可以实际计算的和。自回归模型（PixelCNN、WaveNet、GPT）将 `p(x)` 分解为 `∏ p(x_i | x_<i)`。归一化流（RealNVP、Glow）将 `p(x)` 构建为简单基的可逆变换。优点：精确似然，干净的训练损失。缺点：自回归推理是串行的（对长序列慢），流需要可逆架构（架构上受限）。

**2. 显式密度，近似。** 从下界（ELBO）约束 `log p(x)` 并优化该界。VAE（Kingma 2013）使用带变分后验的编码器-解码器。扩散模型（DDPM，Ho 2020）训练一个去噪器，隐式地优化加权 ELBO。扩散是 2026 年图像、视频和 3D 的主导骨干。

**3. 隐式密度。** 完全跳过密度；学习一个生成器 `G(z)` 产生样本和一个判别器 `D(x)` 区分真假。GAN（Goodfellow 2014）。推理时快速（一次前向传播），但训练时臭名昭著地不稳定。StyleGAN 1/2/3 即使在 2026 年仍然是固定域照片级真实感（人脸、卧室）的最先进技术。

**4. 基于分数 / 连续时间。** 直接学习对数密度的梯度 `∇_x log p(x)`（分数）。Song 和 Ermon（2019）表明分数匹配将扩散推广到 SDE。流匹配（Lipman 2023）是 2024-2026 年的热门：无需模拟的训练、更直的路径、比 DDPM 快 4-10 倍的采样。Stable Diffusion 3、Flux、AudioCraft 2 都使用流匹配。

**5. 基于离散编码的 token 自回归。** 使用 VQ-VAE 或残差量化器将高维数据压缩为短离散 token 序列，然后使用 Transformer 对 token 序列建模。Parti、MuseNet、AudioLM、VALL-E、Sora 的补丁分词器都使用这种方法。这是桶 1 加上一个学习到的分词器。

## 五个问题分类法

当一篇新的生成模型论文发表时，在阅读方法部分之前先回答这五个问题。

1. **正在建模什么？** 像素、潜在变量、离散 token、3D 高斯、网格、波形？
2. **密度是显式的还是隐式的？** 他们是否写下了 `log p(x)`？
3. **采样：一次性还是迭代式？** 迭代意味着较慢的推理；一次性通常意味着对抗性或蒸馏。
4. **条件：无条件、类别、文本、图像、姿态？** 这决定了损失和架构骨架。
5. **评估：FID、CLIP 分数、IS、人类偏好、任务准确率？** 每个都有已知的失败模式（见第 14 课）。

你将在本阶段的每节课中重新回答这五个问题。到最后，它们将成为你的本能反应。

## 动手实现

本课的代码是一个轻量级可视化：使用三种玩具方法（核密度、离散直方图和最近样本"GAN 式"生成器）从样本中拟合一维高斯混合模型，这样你就可以在一个屏幕上打印的问题上看到显式 vs 隐式密度的区别。

运行 `code/main.py`。它从双峰高斯混合中抽取 2000 个样本，然后打印：

```
显式密度（直方图）：p(x in [-0.5, 0.5]) ≈ 0.38
近似密度（KDE）：    p(x in [-0.5, 0.5]) ≈ 0.41
隐式（最近邻生成器）：打印了 20 个新样本，没有 p(x)
```

注意：前两个让你可以问"这个点有多可能？"第三个不行。这就是*显式 vs 隐式*的区别，它将影响未来的每节课。

## 使用

2026 年，哪个家族，用于哪个任务？

| 任务 | 最佳家族 | 为什么 |
|------|-------------|-----|
| 照片级真实感人脸，窄域 | StyleGAN 2/3 | 仍然最清晰，推理最快。 |
| 通用文生图 | 潜在扩散 + 流匹配 | SD3、Flux.1、DALL-E 3。 |
| 快速文生图 | 修正流 + 蒸馏 | SDXL-Turbo、SD3-Turbo、LCM。 |
| 文生视频 | 扩散 Transformer + 流匹配 | Sora、Veo 2、Kling。 |
| 语音 + 音乐 | 基于 token 的 AR（AudioLM、VALL-E、MusicGen）或流匹配（AudioCraft 2） | 离散 token 扩展成本低。 |
| 3D 场景 | 高斯泼溅拟合、扩散先验 | 3D-GS 用于重建，扩散用于新视角。 |
| 密度估计（无需采样） | 流 | 唯一具有精确 `log p(x)` 的家族。 |
| 模拟 / 物理 | 流匹配、分数 SDE | 直线路径，光滑向量场。 |

## 关键术语

| 术语 | 人们说它是什么 | 它实际意味着什么 |
|------|-----------------|-----------------------|
| 生成模型 | "它生成新东西" | 学习 `p_data(x)` 的采样器，可选地暴露 `log p(x)`。 |
| 显式密度 | "你可以评估它" | 模型提供封闭形式或可处理的 `log p(x)`。 |
| 隐式密度 | "GAN 风格" | 只有采样器——无法评估给定点的 `p(x)`。 |
| ELBO | "证据下界" | `log p(x)` 的可处理下界；VAE 和扩散优化它。 |
| 分数 | "对数密度的梯度" | `∇_x log p(x)`；扩散和 SDE 模型学习这个场。 |
| 流形假设 | "数据生活在一个表面上" | 高维数据集中在低维流形上；这就是降维有效的原因。 |
| 自回归 | "预测下一个部分" | 将联合分布分解为条件概率的乘积。 |
| 潜在变量 | "压缩编码" | 解码器可以从中重建输入的低维表示。 |

## 生产说明：五个家族，五种推理形态

每个家族映射到不同的推理服务器成本曲线。生产推理文献将 LLM 推理框架化为预填充 + 解码；同样的分解在这里适用：

- **自回归（桶 1 和 5）。** 顺序解码主导延迟；KV 缓存、连续批处理和推测解码都直接适用。
- **VAE / 扩散 / 流匹配（桶 2 和 4）。** 没有 LLM 意义上的解码。成本 = `num_steps × step_cost`，而 `step_cost` 是完整潜在分辨率下的 transformer 或 U-Net 前向传播。生产旋钮是步数（DDIM / DPM-Solver / 蒸馏）、批次大小和精度（bf16 / fp8 / int4）。
- **GAN（桶 3）。** 一次前向传播。没有调度，没有 KV 缓存。TTFT ≈ 总延迟。这就是为什么 StyleGAN 仍然在窄域 UX 上胜出。

当你在论文摘要中看到"比扩散更快"时，将其翻译为"更少的步数 × 相同的步成本"或"相同的步数 × 更便宜的步成本"。其他都是营销。
