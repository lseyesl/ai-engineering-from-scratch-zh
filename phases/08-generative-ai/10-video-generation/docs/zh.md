# 视频生成

> 图像是一个 2-D 张量。视频是一个 3-D 张量。理论是相同的；计算量是 10-100 倍。OpenAI 的 Sora（2024 年 2 月）证明了这是可能的。到 2026 年，Veo 2、Kling 1.5、Runway Gen-3、Pika 2.0 和 WAN 2.2 以 1080p 从文本生产视频——而开放权重技术栈（CogVideoX、HunyuanVideo、Mochi-1、WAN 2.2）落后 12 个月。

**类型：** 构建
**语言：** Python
**前置知识：** 阶段 8 · 07（潜在扩散），阶段 7 · 09（ViT），阶段 8 · 06（DDPM）
**时间：** ~45 分钟

## 问题

一段 10 秒 1080p 视频，24fps，是 240 帧 1920×1080×3 像素。每段大约 1.5 GB 原始数据。像素空间扩散是不可行的。你需要：

1. **时空压缩。** 一个编码视频而不仅仅是帧的 VAE，将其编码为时空补丁序列。
2. **时间连贯性。** 帧需要在持续数秒的时间内共享内容、光照和物体身份。网络必须对运动建模。
3. **计算预算。** 对于相同的模型大小，视频训练比图像贵 10-100 倍。
4. **条件化。** 文本、图像（第一帧）、音频或另一个视频。大多数生产模型接受所有四种输入。

解决这个问题的架构是**扩散 Transformer（DiT）**，应用于时空补丁，在庞大的（提示、描述、视频）数据集上训练。扩散损失与第 6 课相同。

## 概念

![视频扩散：分块、DiT、解码](../assets/video-generation.svg)

### 分块

用 3D VAE（学习到的时空压缩）编码视频。潜在变量的形状是 `[T_latent, H_latent, W_latent, C_latent]`。将其分割成大小为 `[t_p, h_p, w_p]` 的补丁。对于 Sora 风格的模型，`t_p = 1`（每帧补丁）或 `t_p = 2`（每两帧）。一段 10 秒 1080p 视频压缩到约 20,000-100,000 个补丁。

### 时空 DiT

一个 Transformer 处理扁平序列的补丁。每个补丁有一个 3D 位置嵌入（时间 + y + x）。注意力通常被分解：

- **空间注意力** 在每帧的补丁内。
- **时间注意力** 在同一空间位置跨帧。
- **完整的 3D 注意力** 要贵 16-100 倍；仅用于低分辨率或研究中。

### 文本条件化

与大型文本编码器的交叉注意力（Sora 使用 T5-XXL，CogVideoX-5B 使用 T5-XXL）。长提示很重要——Sora 的训练集有 GPT 生成的密集重新描述，平均每段 200 个 token。

### 训练

标准扩散损失（ε 或 v 预测）在时空潜在变量上。数据：网络视频 + 约 1 亿精选片段 + 合成文本描述。计算：即使是一个小型研究运行也需要 10,000+ GPU 小时；Sora 规模需要 100,000+。

## 动手实现

`code/main.py` 模拟核心时空 DiT 思想：将小型合成视频分块，添加每补丁位置嵌入，并使用 Transformer 风格的注意力在整个补丁序列上去噪。无 numpy；纯 Python。我们展示了即使在一维数据中，当相邻帧的补丁共享去噪器和位置嵌入时，时间连贯性也会出现。

### 步骤 1：将合成 1-D "视频"分块

```python
def make_video(T_frames=8, rng=None):
    # "视频"是沿着平滑轨迹的 1-D 值序列
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### 步骤 2：每帧位置嵌入

```python
def pos_embed(t, dim):
    return sinusoidal(t, dim)
```

### 步骤 3：去噪器看到整个序列

不是独立地对每帧去噪，我们的微型网络将所有帧值 + 它们的位置嵌入拼接起来，并联合预测所有帧的噪声。

### 步骤 4：时间连贯性测试

训练后，采样一段视频。测量帧间增量。如果模型已经学习了时间结构，增量会比独立采样每帧更小。

## 陷阱

- **独立的逐帧采样 = 闪烁。** 如果你对每帧单独运行图像扩散，输出会闪烁，因为每帧的噪声是独立的。视频扩散通过注意力或共享噪声耦合帧来解决这个问题。
- **朴素的 3D 注意力 = OOM。** 在 10 秒 1080p 潜在变量上的完整 3D 注意力是数千亿次操作。分解为空间 + 时间。
- **数据描述比大小更重要。** Sora 相比先前工作的主要升级是在约 10x 更详细的描述上训练（GPT-4 重新标记的片段）。OpenAI 的技术报告明确指出了这一点。
- **第一帧条件化。** 大多数生产模型也接受一张图像作为第一帧。这是"图像到视频"模式；训练包括这种变体。
- **物理漂移。** 长片段（>10s）会累积微小的不一致性。滑动窗口生成 + 关键帧锚定有所帮助。

## 关键术语

| 术语 | 人们说它是什么 | 它实际意味着什么 |
|------|-----------------|-----------------------|
| 视频 VAE | "3-D VAE" | 将 `(T, H, W, C)` 压缩为时空潜在变量的编码器。 |
| 补丁 | "token" | 潜在变量的固定大小 3-D 块；DiT 的输入。 |
| 分解注意力 | "空间 + 时间" | 在空间上运行注意力，然后在时间上运行；跳过完整的 3-D 注意力。 |
| 图像到视频（I2V） | "让这张照片动起来" | 模型接受图像 + 文本，输出从该图像开始的视频。 |
| 关键帧条件化 | "锚定帧" | 固定特定帧以控制视频的运动轨迹。 |
| 运动笔刷 | "方向提示" | 用户在图像上绘制运动向量的 UI 输入。 |
| 重新描述 | "密集描述" | 使用 LLM 用详细提示重新标记训练片段。 |
| 闪烁 | "时间伪影" | 帧间不一致性；通过耦合去噪来修复。 |

## 生产说明：视频潜在变量是一个内存带宽问题

一段 10 秒 1080p 视频，24fps，是 240 帧 × 1920 × 1080 × 3 ≈ 1.5 GB 原始像素。经过 4× 视频 VAE 压缩（`2 × 空间 × 2 × 时间`）后，潜在变量约 100 MB 每个请求。通过时空 DiT 以批次 1 运行 30 步，你每步在 HBM 中移动约 3 GB——瓶颈是内存带宽，而非 FLOPs。

三个生产旋钮，全部直接来自生产推理文献：

- **跨 DiT 的张量并行。** 文生视频模型通常 ≥10B 参数。跨 4 个 H100 的 TP=4 是标准配置；对于 405B 级模型使用 PP=2 × TP=2。每步延迟大致随 TP 线性下降，直到 all-reduce 瓶颈。
- **帧批处理 = 连续批处理。** 在生成时，视频概念上是通过注意力连接的一个批处理帧。连续批处理（运行中调度）适用：如果模型架构允许滑动窗口生成，在帧 `t-1` 正在返回的同时开始渲染帧 `t+1`。
- **片段级预填充缓存。** 对于图像到视频，第一帧条件化类似于 LLM 的提示预填充：计算一次，在时间解码器的多次传递中重用。这实际上是针对视频的 KV 缓存。
