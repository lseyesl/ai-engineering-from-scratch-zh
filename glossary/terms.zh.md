# AI 工程术语表

## A

### Agent（智能体）
- **通常以为：** "能独立思考和行动的自主 AI"
- **实际含义：** 一个 while 循环：LLM 决定调用哪个工具、执行它、查看结果、然后重复
- **命名由来：** 借自哲学——"agent"是任何能在世界中行动的事物。在 AI 中，它只是"LLM + 工具 + 循环"

### Attention（注意力）
- **通常以为：** "AI 如何关注重要部分"
- **实际含义：** 一种机制，每个 token 计算所有其他 token 值的加权和，权重由它们的相关性（通过 query 和 key 向量的点积）决定
- **命名由来：** 2017 年的论文《Attention Is All You Need》类比人类选择性注意力而命名

### Alignment（对齐）
- **通常以为：** "让 AI 安全"
- **实际含义：** 使 AI 系统的行为符合人类意图、价值观和偏好的技术挑战，包括设计者未预料到的边缘情况

### Autoregressive（自回归）
- **通常以为：** "AI 一次生成一个词"
- **实际含义：** 一种模型，根据之前的所有 token 预测下一个 token，然后将该预测作为下一步的输入。GPT、LLaMA 和 Claude 都是自回归模型

### Activation Function（激活函数）
- **通常以为：** "层之间的非线性函数"
- **实际含义：** 在每个线性层之后应用的函数，用于引入非线性。没有它，堆叠任意数量的线性层都会坍缩为单个线性变换。最常用的是 ReLU、GELU 和 SiLU。选择直接影响训练时梯度是否能够流动

### Adam（优化器）
- **通常以为：** "默认的优化器"
- **实际含义：** 自适应矩估计（Adaptive Moment Estimation）。结合了动量（一阶矩）和每个参数的自适应学习率（二阶矩）。对早期步骤有偏差修正。在大多数任务上无需太多调参就能良好运行

### AdamW
- **通常以为：** "比 Adam 更好"
- **实际含义：** 带解耦权重衰减的 Adam。在标准 Adam 中，L2 正则化会被每个参数的自适应学习率缩放，这不是你想要的。AdamW 直接将权重衰减应用到权重上，独立于梯度统计。训练 transformer 的默认优化器

### Autograd（自动梯度）
- **通常以为：** "自动计算梯度"
- **实际含义：** 一个系统，记录对张量的操作并通过反向模式微分自动计算梯度。PyTorch 的 autograd 即时构建计算图（动态图），而 JAX 使用函数变换（grad）。这就是使反向传播变得实用的原因——你写前向传播，框架计算所有导数

## B

### Batch Size（批次大小）
- **通常以为：** "一次处理多少个样本"
- **实际含义：** 一次前向/反向传播中处理的训练样本数量，然后更新权重。更大的批次给出更稳定的梯度估计但使用更多内存。典型值：训练 32-512，推理时可以更大。批次大小与学习率相关——批次翻倍，学习率也翻倍（线性缩放规则）

### Backpropagation（反向传播）
- **通常以为：** "神经网络如何学习"
- **实际含义：** 一种算法，通过链式法则在网络中反向传播，计算每个权重对误差的贡献大小，然后按比例调整权重
- **命名由来：** 误差从输出层向输入层逐层反向传播

## C

### Context Window（上下文窗口）
- **通常以为：** "AI 能记住多少"
- **实际含义：** 单次 API 调用中能容纳的最大 token 数量（输入 + 输出）。不是记忆——它是一个固定大小的缓冲区，每次调用都会重置

### Chain of Thought / CoT（思维链）
- **通常以为：** "让 AI 一步步思考"
- **实际含义：** 一种提示技术，要求模型展示其推理步骤，从而在多步问题上提高准确性，因为每一步都为下一个 token 的生成提供了条件

### CNN（卷积神经网络）
- **通常以为：** "图像 AI"
- **实际含义：** 使用卷积操作（在输入上滑动滤波器）检测局部模式的神经网络。堆叠卷积层可以检测越来越复杂的特征：边缘、纹理、物体

### CUDA
- **通常以为：** "GPU 编程"
- **实际含义：** NVIDIA 的并行计算平台。让你在数千个 GPU 核心上同时运行矩阵运算。PyTorch 和 TensorFlow 底层都使用 CUDA

### Chunking（分块）
- **通常以为：** "把文档拆成小块"
- **实际含义：** 在嵌入之前将文本分成片段以便检索。块大小决定搜索结果的粒度。太小：丢失上下文。太大：稀释相关性。常见策略：固定大小重叠分块、基于句子的分块、或语义分割。典型块大小：256-512 个 token，10-20% 重叠

### Contrastive Learning（对比学习）
- **通常以为：** "通过比较学习"
- **实际含义：** 通过将相似对拉近、将不相似对推远来进行训练。CLIP 就使用这个：匹配图像-文本对 vs 不匹配的对

### Cosine Similarity（余弦相似度）
- **通常以为：** "两个向量有多相似"
- **实际含义：** 两个向量之间夹角的余弦值：dot(a, b) / (||a|| * ||b||)。范围从 -1（相反方向）到 1（完全相同方向）。忽略大小，只关心方向。embedding 和语义搜索的标准相似度度量

### Cross-Entropy（交叉熵）
- **通常以为：** "分类损失"
- **实际含义：** 衡量两个概率分布之间的差异。用于分类：-sum(y_true * log(y_pred))。用于语言模型：正确下一个 token 的负对数概率。越低越好。困惑度就是 exp(交叉熵)

## D

### Data Augmentation（数据增强）
- **通常以为：** "制造更多训练数据"
- **实际含义：** 创建已有数据的修改副本（旋转图像、添加噪声、改写文本），在不收集新数据的情况下增加训练集多样性。减少过拟合

### Decoder（解码器）
- **通常以为：** "输出部分"
- **实际含义：** 在 transformer 中，解码器使用因果（掩码）自注意力，使每个位置只能关注之前的位置。GPT 是纯解码器。BERT 是纯编码器。T5 是编码器-解码器

### Diffusion Model（扩散模型）
- **通常以为：** "从噪声生成图像的 AI"
- **实际含义：** 一种模型，训练来逆转逐步加噪的过程——它学习预测和去除噪声，在生成时从纯噪声开始，迭代去噪

### DPO（直接偏好优化）
- **通常以为：** "更简单的 RLHF"
- **实际含义：** 一种完全跳过奖励模型的训练方法——它直接优化语言模型，使其在人类偏好对中偏好更好的回答

### Dropout
- **通常以为：** "随机关闭神经元"
- **实际含义：** 在训练期间，随机将一部分激活值设为零。迫使网络不依赖任何单个神经元。推理时关闭。简单但有效的正则化方法

## E

### Eigenvalue（特征值）
- **通常以为：** "某种用于 PCA 的数学概念"
- **实际含义：** 对于矩阵 A，特征值 lambda 满足 Av = lambda*v，其中 v 是某个向量。它告诉你矩阵在哪个方向上如何缩放向量。大的特征值 = 数据中方差大的方向

### Embedding
- **通常以为：** "某种把词变成数字的 AI 魔法"
- **实际含义：** 从离散项目（词、图像、用户）到连续空间中稠密向量的学习映射，相似的项目最终会彼此靠近
- **命名由来：** 项目被"嵌入"到一个距离有意义的几何空间中

### Encoder（编码器）
- **通常以为：** "输入部分"
- **实际含义：** 在 transformer 中，编码器使用双向自注意力，使每个位置可以关注所有位置。BERT 是纯编码器。适合理解类任务（分类、NER），但不适合生成

### Epoch
- **通常以为：** "数据过一遍"
- **实际含义：** 正是如此。完整地遍历训练集中每个样本一次。多个 epoch = 多次看到数据。更多 epoch 可以提高学习效果，但有过度拟合的风险

## F

### Feature（特征）
- **通常以为：** "数据中的一列"
- **实际含义：** 数据的单个可测量属性。在经典 ML 中，你手动设计特征。在深度学习中，网络从原始数据中自动学习特征

### Few-Shot（少样本）
- **通常以为：** "先给 AI 一些例子"
- **实际含义：** 在提示中包含少量输入-输出示例，然后再让模型执行任务。通常 3-5 个示例。模型通过模式匹配这些示例来理解所需的格式和行为。对比零样本（无示例）和微调（数千个示例融入权重）

### Fine-tuning（微调）
- **通常以为：** "在你的数据上训练 AI"
- **实际含义：** 从预训练模型的权重开始，在一个较小的、任务特定的数据集上继续训练。只更新已有权重，不从头添加新知识

### Function Calling（函数调用）
- **通常以为：** "能使用工具的 AI"
- **实际含义：** LLM 请求执行外部函数的结构化方式。你用 JSON Schema 描述定义工具，模型输出结构化 JSON 对象指定调用哪个函数及参数，你的代码执行它，结果返回给模型。与智能体不同——函数调用是机制，智能体是循环

## G

### Guardrails（护栏）
- **通常以为：** "AI 的安全过滤器"
- **实际含义：** LLM 周围的输入/输出验证层，用于检测和阻止有害内容、提示注入攻击、PII 泄露或偏离主题的回复。通常是流水线：输入过滤 -> LLM -> 输出过滤。可以基于规则（正则、关键词列表）或基于模型（分类器评分安全性）

### GPT
- **通常以为：** "ChatGPT"或"The AI"
- **实际含义：** 生成式预训练 Transformer（Generative Pre-trained Transformer）——一种特定架构，使用纯解码器 transformer 在大规模文本语料库上训练，预测下一个 token
- **命名由来：** 生成式（生成文本）、预训练（在大数据上训练一次，然后适配）、Transformer（架构）

### GAN（生成对抗网络）
- **通常以为：** "两个 AI 互相打架"
- **实际含义：** 一个生成器网络试图创建真实数据，同时一个判别器网络试图区分真假。它们一起训练：生成器越来越擅长欺骗判别器，判别器也越来越擅长检测伪造

### Gradient（梯度）
- **通常以为：** "斜率"
- **实际含义：** 一个偏导数向量，指向最陡峭上升的方向。在 ML 中，你沿梯度的反方向（梯度下降）来最小化损失

### Gradient Descent（梯度下降）
- **通常以为：** "AI 如何改进"
- **实际含义：** 一种优化算法，沿最陡峭地减少损失函数的方向调整参数，就像在高维地形中下山

## H

### Hyperparameter（超参数）
- **通常以为：** "你需要调节的设置"
- **实际含义：** 训练之前设置的数值，控制训练过程本身：学习率、批次大小、层数、丢弃率。与模型参数（权重）不同，这些不是从数据中学习来的

### Hallucination（幻觉）
- **通常以为：** "AI 在撒谎"或"编造东西"
- **实际含义：** 模型生成听起来合理但未基于其训练数据或给定上下文的文本——它在完成模式，而不是检索事实

## I

### Inference（推理）
- **通常以为：** "运行 AI"
- **实际含义：** 使用训练好的模型对新数据进行预测。没有权重更新。这就是你在生产中所做的：发送输入，获取输出

### Inductive Bias（归纳偏置）
- **通常以为：** 没听说过
- **实际含义：** 模型架构中内置的假设。CNN 假设局部模式重要（卷积）。RNN 假设顺序重要（序列处理）。Transformer 假设一切可能与其他一切相关（注意力）。正确的偏置帮助模型用更少的数据学得更快

### JAX
- **通常以为：** "Google 的 ML 框架"
- **实际含义：** 一个与 NumPy 兼容的库，增加了自动微分（grad）、JIT 编译（jit）、自动向量化（vmap）和多设备并行（pmap）。与 PyTorch 的面向对象风格不同，JAX 是纯函数式的——没有隐藏状态，没有原地修改。被 Google DeepMind 用于 AlphaFold、Gemini 和大规模研究

## K

### KV Cache（键值缓存）
- **通常以为：** "让推理更快"
- **实际含义：** 在自回归生成过程中，缓存先前 token 的键和值矩阵，这样就不用在每一步重新计算。以内存换速度。对快速 LLM 推理至关重要

## L

### Latent Space（潜空间）
- **通常以为：** "隐藏表示"
- **实际含义：** 一个压缩的、学习到的表示空间，其中相似的输入映射到附近的点。自编码器、VAE 和扩散模型都在潜空间中工作。它比输入维度低，但捕捉了重要的结构

### Learning Rate（学习率）
- **通常以为：** "AI 学得有多快"
- **实际含义：** 控制梯度下降步长的标量。太高：越过最小值而发散。太低：收敛太慢或陷入停滞。最重要的超参数

### LLM（大型语言模型）
- **通常以为：** "AI"或"大脑"
- **实际含义：** 基于 transformer 的神经网络，训练来预测序列中的下一个 token，具有数十亿参数，在互联网规模的文本数据上训练

### LoRA（低秩适配）
- **通常以为：** "高效的微调"
- **实际含义：** 不更新所有权重，而是在原始权重旁边插入小型低秩矩阵。只训练这些小矩阵，将内存减少 10-100 倍

### Loss Function（损失函数）
- **通常以为：** "AI 有多错"
- **实际含义：** 衡量预测输出与实际输出之间差距的函数。训练就是最小化这个函数。MSE 用于回归，交叉熵用于分类，对比损失用于 embedding。损失函数的选择定义了"好"对模型意味着什么

## M

### Mixed Precision（混合精度）
- **通常以为：** "加速训练的技巧"
- **实际含义：** 前向传播和大多数操作使用 float16（更快、更省内存），但梯度累积和权重更新保留 float32（更精确）。获得 2 倍加速且精度损失可忽略

### MoE（混合专家）
- **通常以为：** "只有部分模型运行"
- **实际含义：** 一种由许多"专家"子网络组成的模型，路由机制将每个输入只发送给少数专家。完整模型很大，但每次前向传播成本低，因为大多数专家被跳过。Mixtral 和 GPT-4 使用这种架构

### MCP（模型上下文协议）
- **通常以为：** "AI 使用工具的方式"
- **实际含义：** 一个开放协议（基于 stdio/HTTP 的 JSON-RPC），标准化 AI 应用如何连接到外部数据源和工具，包含工具、资源和提示的类型化模式

## N

### NaN（非数字）
- **通常以为：** "训练崩溃了"
- **实际含义：** 表示未定义结果的浮点值（0/0、inf-inf）。训练中 NaN 损失通常意味着：学习率太高、梯度爆炸、对数零值或除以零。训练失败时首先要检查的东西

### Normalization（归一化）
- **通常以为：** "缩放数据"
- **实际含义：** 将数值调整到标准范围。批归一化在批次维度上归一化。层归一化在特征维度上归一化。两者都稳定训练并允许更高的学习率

## O

### Overfitting（过拟合）
- **通常以为：** "模型记住了数据"
- **实际含义：** 模型在训练数据上表现良好，但在未见过的数据上表现差。它学到了噪声，而不是信号。解决方法：更多数据、正则化（丢弃法、权重衰减）、早停、数据增强、更简单的模型

### Optimizer（优化器）
- **通常以为：** "更新权重的那个东西"
- **实际含义：** 使用梯度更新模型参数的算法。SGD 最简单。Adam 最常见。每种优化器都有不同的特性：收敛速度、内存使用、对超参数的敏感性

## P

### Parameter（参数）
- **通常以为：** "模型大小"
- **实际含义：** 模型中可学习的值，通常是权重或偏置。"7B 参数"意味着 70 亿个可学习数字。每个 float32 参数占用 4 字节，所以 7B 参数 = 仅权重的内存就需要 28GB

### Perplexity（困惑度）
- **通常以为：** "模型有多困惑"
- **实际含义：** 平均交叉熵损失的指数。越低越好。困惑度为 10 意味着模型的不确定性相当于每一步在 10 个 token 中均匀选择

### Precision & Recall（精确率与召回率）
- **通常以为：** "准确率指标"
- **实际含义：** 精确率 = 你标记的项目中，有多少是正确的。召回率 = 所有正确的项目中，你找出了多少。它们互相权衡：捕捉每封垃圾邮件（高召回率）意味着更多误报（低精确率）。F1 分数是它们的调和平均数。当假阳性代价高时用精确率，当假阴性代价高时用召回率

### Prompt Engineering（提示工程）
- **通常以为：** "用正确的方式和 AI 说话"
- **实际含义：** 设计输入文本以可靠地产生期望输出——包括系统提示、少样本示例、格式指令和思维链触发词

### Prompt Injection（提示注入）
- **通常以为：** "用言语黑进 AI"
- **实际含义：** 一种攻击，输入中的恶意文本覆盖系统提示或指令。直接注入：用户输入"忽略之前的指令"。间接注入：检索到的文档包含隐藏指令。相当于 LLM 领域的 SQL 注入。没有完整的解决方案——防御是输入验证、输出过滤和权限分离的多层组合

## Q

### QLoRA
- **通常以为：** "比 LoRA 更便宜"
- **实际含义：** 量化 LoRA。将冻结的基础模型权重保持在 4 位精度（NF4 格式），同时以 16 位训练 LoRA 适配器。与标准 LoRA 相比，内存再减少 3-4 倍。需要 14GB 的 LoRA 7B 模型用 QLoRA 只需 4-6GB。在大多数基准测试上质量在完整微调的 1% 以内

## R

### RAG（检索增强生成）
- **通常以为：** "能搜索的 AI"
- **实际含义：** 一种模式：从知识库中检索相关文档（使用 embedding 相似度），将它们塞入提示，让 LLM 基于该上下文回答
- **命名由来：** 检索（Retrieval）→ 找到文档 + 增强（Augmented）→ 加入提示 + 生成（Generation）→ LLM 写答案

### RLHF（基于人类反馈的强化学习）
- **通常以为：** "他们如何让 AI 变得有用"
- **实际含义：** 一个训练流水线：（1）收集人类对模型输出的偏好，（2）基于这些偏好训练奖励模型，（3）使用 PPO 优化 LLM 以产生更高奖励的输出

### Quantization（量化）
- **通常以为：** "让模型变小"
- **实际含义：** 将模型权重的精度从 float32（4 字节）降低到 int8（1 字节）或 int4（0.5 字节）。用少量精度换取 4-8 倍更少的内存和更快的推理。GPTQ、AWQ 和 GGUF 是常见格式

### ReLU
- **通常以为：** "激活函数"
- **实际含义：** 修正线性单元（Rectified Linear Unit）：f(x) = max(0, x)。最简单的非线性激活。计算快速，正值不会饱和。被广泛使用因为它有效且廉价。变体有 LeakyReLU、GELU、SiLU

### ROUGE
- **通常以为：** "摘要评估指标"
- **实际含义：** 面向召回的摘要评估替代指标。衡量生成文本与参考文本之间的重叠。ROUGE-1 统计一元匹配，ROUGE-2 统计二元匹配，ROUGE-L 找最长公共子序列。计算成本低，但只衡量表面相似度——两个意思相同但用词不同的句子得分会很低

## S

### Semantic Search（语义搜索）
- **通常以为：** "理解含义的智能搜索"
- **实际含义：** 通过含义而非关键词匹配来查找文档。将查询和所有文档嵌入到同一向量空间，然后返回其 embedding 最接近查询 embedding 的文档。"payment failed"能找到"transaction declined"，即使它们没有共同词汇。由 embedding 模型 + 向量数据库驱动

### Streaming（流式传输）
- **通常以为：** "看见回复一个字一个字地出现"
- **实际含义：** LLM 在生成 token 时立即发送，而不是等待完整回复。使用服务器发送事件（SSE）或 WebSocket 协议。将首 token 感知延迟从秒级降到毫秒级。对生产级聊天界面至关重要。每个块包含一个增量（部分 token 或词）

### Self-Attention（自注意力）
- **通常以为：** "模型如何决定关注什么"
- **实际含义：** 每个 token 计算 query、key 和 value 向量。两个 token 之间的注意力权重 = 它们 query 和 key 的点积，经缩放和 softmax 处理。输出 = value 向量的加权和。让每个 token 都能看到其他所有 token

### SFT（监督微调）
- **通常以为：** "教模型遵循指令"
- **实际含义：** 在（指令，回复）对上微调预训练模型。模型学会在给定指令时生成回复。这就是把基础模型变成对话模型的方法

### Softmax
- **通常以为：** "把数字变成概率"
- **实际含义：** softmax(x_i) = exp(x_i) / sum(exp(x_j))。将任意实数的向量转换为概率分布（全部为正，总和为 1）。用于分类头、注意力权重，以及任何需要概率的地方

### Swarm（群体）
- **通常以为：** "一群 AI 智能体像蜜蜂一样协同工作"
- **实际含义：** 多个智能体通过消息传递共享状态并协调，从简单的个体规则中产生涌现行为，而非集中控制

## T

### System Prompt（系统提示）
- **通常以为：** "AI 的指令"
- **实际含义：** 对话开始时的一条特殊消息，设置模型的行为、角色和约束。在用户消息之前处理。在大多数 UI 中对用户不可见。定义模型应该和不应该做什么、语气、格式偏好和领域重点。与用户提示不同——系统提示由开发者设置

### Tensor（张量）
- **通常以为：** "多维数组"
- **实际含义：** 深度学习框架中的基本数据结构。0 维张量是标量，1 维是向量，2 维是矩阵，3 维及以上是张量。在 PyTorch 和 JAX 中，张量跟踪其计算历史以进行自动微分，可以位于 CPU 或 GPU 上。所有神经网络的输入、输出、权重和梯度都是张量

### Token
- **通常以为：** "一个词"
- **实际含义：** 由分词器（如 BPE）生成的子词单元（英语中通常 3-4 个字符）。"unbelievable"可能是 3 个 token："un" + "believ" + "able"

### Temperature（温度）
- **通常以为：** "创造力设置"
- **实际含义：** 在 softmax 之前缩放 logits 的标量。Temperature=1 是默认值。越高 = 分布越平坦 = 输出越随机。越低 = 分布越尖锐 = 结果越确定。Temperature=0 相当于 argmax（总选最可能的 token）

### Transfer Learning（迁移学习）
- **通常以为：** "使用预训练模型"
- **实际含义：** 将一个任务上训练好的模型拿来，适配到不同的任务。早期层学习可以迁移的通用特征（边缘、句法模式）。只有后期层需要任务特定训练。这就是为什么你可以为任何 NLP 任务微调 BERT

### Transformer
- **通常以为：** "现代 AI 背后的架构"
- **实际含义：** 一种使用自注意力（让每个位置都能关注其他所有位置）而非循环来处理序列的神经网络架构，实现了大规模并行化
- **命名由来：** 它通过注意力层将输入表示转换（transform）为输出表示

## U

### Underfitting（欠拟合）
- **通常以为：** "模型没在学习"
- **实际含义：** 模型太简单，无法捕捉数据中的模式。训练损失居高不下。解决方法：更多参数、更多层、更长的训练、更少的正则化、更好的特征

## V

### VAE（变分自编码器）
- **通常以为：** "一种生成模型"
- **实际含义：** 一种自编码器，通过强制编码器输出服从高斯分布来学习平滑的潜空间。你可以从这个分布中采样，解码生成新数据。重参数化技巧使其可以通过反向传播训练

### Vector Database（向量数据库）
- **通常以为：** "AI 的专用数据库"
- **实际含义：** 专为存储向量（浮点数的稠密数组）和快速近似最近邻搜索而优化的数据库。是相似度搜索、RAG 和推荐系统中的核心操作

## W

### Weight（权重）
- **通常以为：** "模型学到的东西"
- **实际含义：** 模型参数矩阵中的单个数字。输入大小 768、输出大小 3072 的线性层有 768*3072 = 2,359,296 个权重。训练会调整每个权重以最小化损失函数

### Weight Decay（权重衰减）
- **通常以为：** "正则化"
- **实际含义：** 在损失函数中加入与权重大小成比例的惩罚。相当于 L2 正则化。防止权重过大。典型值：0.01-0.1

## Z

### Zero-Shot（零样本）
- **通常以为：** "无需训练"
- **实际含义：** 在一个模型没有明确训练过的任务上使用它，提示中没有任务特定的示例。模型从预训练中泛化。之所以有效，是因为大型模型见过足够多的多样性，能够处理新的任务格式
